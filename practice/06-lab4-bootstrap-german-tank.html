📚 Part 1 — Dungeons & Dragons

背景：D&D 投 20 面骰，加 modifier，比阈值 17 算成功。Ashley 的角色 Saruman modifier = 11。这一节练模拟，然后假装不知道 modifier，从观察值估计。

Q 1.1单次模拟：roll_result / modified_result / action_succeeded 
possible_rolls = np.arange(1, 21)
roll_result      = np.random.choice(possible_rolls)
modified_result  = roll_result + 11
action_succeeded = modified_result >= 17

💡 np.arange(1, 21) 是 [1..20]（不含 21）。autograder 提示 "What values are possible when you roll a 20-sided die?"——就是怕你写成 np.arange(20)（=[0..19]）。

💡 action_succeeded 是 bool——>= 17 直接生成布尔值。不是 0/1 int。

Q 1.210 次模拟 → ten_reps

"Do not use a for-loop here! Call np.random.choice with multiple arguments."

ten_reps = np.random.choice(possible_rolls, 10) + 11 >= 17
rough_success_chance = np.mean(ten_reps)

💡 np.random.choice(arr, n) 一次抽 n 个——返回 array。array + 标量 = 逐元素相加。>= 17 也是逐元素 → bool array。np.mean(bool_array) = True 比例。

💡 为什么不用 for：vectorized 速度比 for 快几十倍。lab 想让你养成"先想 vectorized 再想 for"的习惯。

Q 1.3理论概率 theoretical_probability 
theoretical_probability = 15 / 20   # 0.75

💡 推导：modifier = 11，需 roll + 11 ≥ 17 → roll ≥ 6。可能值 {6, 7, …, 20}，共 15 个。20 面骰每个等概率 → P(成功) = 15/20。

Q 1.4simulate_modified_rolls() 函数 
modifier = 11
num_rolls = 10

def simulate_modified_rolls():
    return np.random.choice(possible_rolls, num_rolls) + modifier

modified_rolls = simulate_modified_rolls()

💡 返回 array of 10 modified rolls——每个数都是 12 到 31 之间（1+11 到 20+11）。注意函数体只一行，简单 reuse。

Q 1.5从 modified_rolls 估 modifier（min-based）

"If we see that the smallest modified roll was 1, we know modifier = 0. If it's 13, we'll guess modifier = 13 − 1 = 12."

min_based_estimate = modified_rolls.min() - 1

💡 直觉：min(roll) 最小可能 = 1（骰子）。如果观察到的 min(modified) 是 m，那么有可能 roll = 1 出现过，对应 modifier = m − 1。这是下界估计。

💡 偏差：min-based estimate 总是偏低（除非真的运气好抽到 1）。因为 min(roll) 通常 > 1，所以估出来的 modifier > 真值。注意：少 10 次抽样的话很可能见不到 roll=1。

Q 1.6estimate_modifier(nums)（mean-based） 
def estimate_modifier(nums):
    '''Estimate the roll modifier based on observed modified rolls.'''
    return np.mean(nums) - 10.5

mean_based_estimate = estimate_modifier(modified_rolls)

💡 推导：1..20 的均值 = 10.5。所以 1+m, 2+m, …, 20+m 的均值 = 10.5 + m。已知样本均值 ≈ 10.5 + m → m ≈ sample_mean − 10.5。这就是 mean-based estimator。

💡 比 min-based 更稳：用所有 10 个数（不只是 min），收敛快。但 mean-based 可能给出非整数（modifier 真值是 11，估出来 11.3 不奇怪），所以 lab 提示 "It's okay if your mean_based_estimate is not an integer."

📚 Part 2 — German Tank Problem

背景：二战盟军想估算德国坦克总产量 N。坦克 serial number 是 1..N。盟军看到的是 SRS 17 个 serial numbers。本节用同样的"max-based vs mean-based"对比，但这次主角变成了 N。

Q 2.1Parameter vs Statistic：q2_1

4 选项：1=N parameter, 估 parameter；2=N parameter, 估 statistic；3=N statistic, 估 parameter；4=N statistic, 估 statistic。

q2_1 = 2

💡 必背定义：

  • Parameter：population 的某个数值（fixed but unknown）。N = 总坦克数 = parameter。
  • Statistic：从 sample 算出来的数（random，随 sample 而变）。我们的估计 = statistic。
口诀："parameter ↔ population，statistic ↔ sample"。每年 Quiz 3 都至少一题考这个区分。

Q 2.2plot_serial_numbers(df) 
def plot_serial_numbers(df):
    df.plot(kind='hist', y='serial_number', bins=np.arange(1, 202), ec='w')

plot_serial_numbers(observations)

💡 bin 宽度 = 1：题目说 "bins of width 1 ranging from 1 to 200 inclusive" → np.arange(1, 202)（含 1，含 201 作为最后 bin 的右端）。不是 np.arange(1, 200)（不含 200）。

💡 直方图应该"参差不齐"——17 个 serial number 散落在 1–200，所以 bin 高度大多是 0/density，少数是 ~0.06。

Q 2.3Max-based estimate 
def calculate_max_based_estimate(nums):
    return nums.max()

max_based_estimate = calculate_max_based_estimate(observations.get('serial_number'))
# = 135

💡 直觉：sample max ≤ N（任何坦克的 serial 都 ≤ 总产量）。所以 sample max 是 N 的保守下界。这次观察到的 17 个里最大是 135，所以 N ≥ 135。

💡 偏差：max-based 总是偏低（≤ N），bias 大小取决于 sample 抽到接近 N 的元素的概率。这是 lab 想让你看到的：max-based 有偏。

Q 2.4Mean-based estimate（= 2 × sample mean） 
def calculate_mean_based_estimate(nums):
    return 2 * np.mean(nums)
# 或 return nums.mean() * 2

mean_based_estimate = calculate_mean_based_estimate(observations.get('serial_number'))
# ≈ 122.47

💡 推导：population 是 uniform 1..N，mean = (N+1)/2 ≈ N/2（N 大时）。所以 sample_mean ≈ N/2 → N ≈ 2 × sample_mean。

💡 验证：(47+42+57+79+26)/5 × 2 = 50.2 × 2 = 100.4。autograder 给了这个测试点 ✓

💡 vs max-based：mean-based 用所有 17 个数，理论上更稳；但这一次抽样运气不好，122.47 < max_based = 135，相当于估出来 N < 135，跟我们已知 "N ≥ 135" 矛盾！这是 Q2.5 的关键。

Q 2.5q2_5 mean_based_estimate 与 N 的关系

5 个选项：1/2 是各种"≥ N"；3 是"definitely lower by at least 12"；4/5 是其他。

q2_5 = 3   # definitely lower by at least 12

💡 推理链：

  1. max-based estimate = 135 → N ≥ 135（任何 sample max 都是 N 的下界）。
  2. mean-based estimate ≈ 122.47 → 122.47 比 135 低 12+。
  3. 所以 mean-based 必然 比 N 低 至少 12（因为 N ≥ 135 但 mean-based ≈ 122.47）。

💡 教学意图：lab 想让你看到——单一 sample 上 mean-based 不一定优于 max-based。需要多次 sampling 才能判断哪个 estimator 平均更好。这就引出 bootstrap：用 resampling 模拟"如果再抽一遍会怎样"。

📚 Part 3 — Resampling

Q 3.1simulate_resample()：bootstrap 必备模板 
def simulate_resample():
    return observations.sample(num_observations, replace=True)

one_resample = simulate_resample()

💡 必背模板：bootstrap = 从原 sample 有放回抽 n（=原 sample 大小）个。这里 num_observations = 17。autograder 提示 "Make sure to sample with replacement!"——必须 replace=True。

💡 为什么大小要相同：bootstrap 的精神是"模拟一次平行宇宙的 SRS"。每次 SRS 的大小就是 n，所以 resample 也要 n。如果不同大小，就不再是"模拟同样过程"了。

Q 3.2画 resample 直方图（reuse Q2.2 函数） 
plot_serial_numbers(one_resample)
plot_serial_numbers(observations)

💡 观察：resample 的 histogram 跟 original 类似——同样的 17 个 serial number 框架，但因为有放回，有些重复（柱子比原 sample 高），有些缺失。

Q 3.3关于 resample 的 true statements

4 个选项：1=resample 元素都来自原 sample；2=resample 总有重复；3=resample max 总等于 original max；4=original 无重复 serial。

true_statements = [1, 4]

💡 选项 1 ✓：bootstrap 是从原 sample 抽——元素肯定都来自原 sample。

💡 选项 2 ✗：resample 可能没有重复（概率小但非 0）。比如 17 次抽样恰好抽到 17 个不同的——概率是 17!/(17^17) × C(17,17) 约 1.6×10⁻⁶（很小但不是 0）。所以"总有重复"错。

💡 选项 3 ✗：resample max 可能小于 original max（如果没抽到原 sample 的最大元素）。如 original max = 135，resample 没抽到 135 → resample max ≤ 108（次大）。

💡 选项 4 ✓：original observations 是 SRS without replacement 从 1..N，所以 17 个 serial 都不同。

Q 3.4两次 resample + 两个 estimate（for loop） 
for i in np.arange(2):
    resample = simulate_resample()
    plot_serial_numbers(resample)
    plt.show()
    mean_est = calculate_mean_based_estimate(resample.get('serial_number'))
    max_est  = calculate_max_based_estimate(resample.get('serial_number'))
    print(f"Mean-based estimate for resample {i+1}:", mean_est)
    print(f"Max-based estimate for resample {i+1}:",  max_est)
    print('\n')

💡 观察：两次 resample 的 mean-based 数值不同（因 randomness），但 max-based 大概率相同（都 = 135，除非那次没抽到 135）。这暗示：mean-based 的variance 更大但无偏，max-based 的 variance 小但有偏（永远 ≤ N）。

Q 3.5P(resample max = 135) 精确概率

"What's the chance that a resample's max is exactly 135?"

q3_5 = 1 - (16/17) ** 17   # ≈ 0.6432

💡 推导：resample max = 135 ⟺ 至少一次抽到 135。所以 P(=135) = 1 − P(从来没抽到 135) = 1 − (16/17)17。
每次抽样有 1/17 概率抽到 135，所以 17 次都不抽到的概率是 (16/17)17。

💡 数值 ≈ 0.64：约 64% 的 resample 会包含原 max。这意味着 max-based estimator 的 bootstrap 分布有明显峰在 135，剩下 36% 散落在更小的值上。

📚 Part 4 — Resampling via Simulation

Q 4.1通用 simulate_estimates(df, size, statistic, reps) 
def simulate_estimates(original_df, sample_size, statistic, repetitions):
    estimates = np.array([])
    for i in np.arange(repetitions):
        sample = original_df.sample(sample_size, replace=True)
        estimates = np.append(estimates, statistic(sample.get('serial_number')))
    return estimates

💡 高阶函数：statistic 参数本身是个函数（如 calculate_mean_based_estimate）。这是 Python 的 first-class function——函数可以当参数。lab 想让你练这个"工厂模式"。

💡 注意参数顺序：original_df, sample_size, statistic, repetitions。如果传错顺序，autograder 抓不出（参数类型不同会让 TypeError 出现）。

Q 4.25000 次 bootstrap → bootstrap_estimates 
bootstrap_estimates = simulate_estimates(
    observations, num_observations, calculate_mean_based_estimate, 5000
)

💡 一行调用：把通用函数填上参数即可。observations（原数据），num_observations（=17，原大小），mean-based 函数，5000 次重抽。

Q 4.395% bootstrap CI（必背：2.5 / 97.5！） 
left_end  = np.percentile(bootstrap_estimates, 2.5)
right_end = np.percentile(bootstrap_estimates, 97.5)

💡 必背公式：c% CI 的两个 percentile = (100−c)/2 和 (100+c)/2。

  • 95% → 2.5 / 97.5
  • 90% → 5 / 95
  • 99% → 0.5 / 99.5
  • 80% → 10 / 90

⚠️ 高频陷阱：95% CI 写成 np.percentile(arr, 5) 和 np.percentile(arr, 95) 是90% CI，不是 95%。autograder 这里测 left_end ∈ (90, 100) 和 right_end ∈ (145, 155)，如果用 5/95 数值会偏窄。
Q 4.4假设 N = 150，整套流程再来一遍 
population = bpd.DataFrame().assign(serial_number=np.arange(1, 150 + 1))
new_observations = population.sample(70, replace=False)   # SRS without replacement

new_bootstrap_estimates = simulate_estimates(
    new_observations, 70, calculate_mean_based_estimate, 5000
)

new_left_end  = np.percentile(new_bootstrap_estimates, 2.5)
new_right_end = np.percentile(new_bootstrap_estimates, 97.5)

💡 注意两个 sample 的区别：

  • "原 sample" = SRS without replacement from population（这是现实抽 tanks 的方式）。
  • "resample" = SRS with replacement from "原 sample"（bootstrap 的特点）。
Q4.4 两步都要：先 sample(70, replace=False) 得到 new_observations；再 bootstrap (replace=True) 5000 次。

Q 4.5CI 覆盖 N 的比例 q4_5（CI 解读 ⭐⭐⭐） 
q4_5 = 3   # 95%

💡 CI 95% 的正确解读：如果重复"从 population 抽 SRS → bootstrap → 取 95% CI"这个过程 1000 次，约 95% 的 CI 会包含真 N。这里只算了 1 个 CI，但 "if we repeated this many times, 95% of intervals would cover" 是 95% CI 的定义。

⚠️ 错误解读（必避免）：
  • ❌ "95% 的概率 N 在这个 CI 里" —— N 是固定的，CI 也是固定的，要么在要么不在，概率不是 95%。
  • ❌ "95% 的 N 落在 CI 里" —— N 只有一个，谈不上 95%。
  • ✅ "重复该过程多次时，约 95% 的intervals 包含 truth"。必背。

📚 Part 5 — With or Without Replacement?

Q 5.1不放回重抽 5000 次 → estimates_without_replacement

"Don't use simulate_estimates (which is with replacement). Write your own loop with replace=False."

estimates_without_replacement = np.array([])
for i in np.arange(5000):
    sample = original_sample.sample(70, replace=False)
    est = calculate_mean_based_estimate(sample.get('serial_number'))
    estimates_without_replacement = np.append(estimates_without_replacement, est)

💡 关键观察：original_sample.shape[0] = 70，sample(70, replace=False) → 取出全部 70 个，每次完全相同。所以 mean 也完全相同。np.all(np.isclose(estimates, 149.514...)) 永远 True——5000 个值一模一样。

Q 5.2为什么 bootstrap 必须 with replacement？

💡 核心理由：bootstrap 是要模拟从 population 反复抽 SRS 这件事。如果不放回抽 n（=原 sample 大小）个，必然 抽出原样本本身——根本没有"variability"，无法生成不同 estimate 的分布。
必须 replace=True：有放回让 resample 之间真的不同，从而 estimate 分布有 spread，才能做 percentile CI。

⚠️ 本作业典型陷阱

陷阱 1：bootstrap 必须 replace=True，size = 原 sample size
Q3.1 写错的两种：① 漏了 replace=True（默认 False，结果跟原样本一样）；② size 写错了不等于 num_observations。两个错误都让 bootstrap 失效。
陷阱 2：95% CI 用 2.5 / 97.5 percentile，不是 5 / 95
Q4.3。c% CI 公式：(100−c)/2 和 (100+c)/2。考试反复考。
陷阱 3：CI 解读（Q4.5）
"95% intervals from this process contain truth"才对。"95% probability truth is in this interval" 错（truth 和 interval 都是固定的）。
陷阱 4：parameter vs statistic（Q2.1）
N（population 总数）= parameter；从 sample 算的估计 = statistic。考试考这区分时常常加干扰项（如把 "estimate" 说成"true value"）。
陷阱 5：估计 N 时 max-based 是 lower bound
Q2.5：sample max ≤ N 永远成立。所以"max-based 估计是 N 的下界"——如果你的另一个 estimator（如 mean-based）算出来比 max-based 还小，那它必然低估了 N。
陷阱 6：mean-based estimator 公式 = 2 × sample mean
Q2.4。基于 uniform 1..N 的 mean ≈ N/2。不是 sample mean × 1，不是 sample mean − 0.5。这跟 Q1.6 的 mean − 10.5 是同一思想：用 sample mean 反推 population 参数。
