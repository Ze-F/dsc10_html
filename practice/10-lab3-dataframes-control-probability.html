📚 Section 1 — California National Parks

Q 1.1species_counts：每个公园的物种数 
species_counts = (species.groupby('Park Name').count()
    .assign(Count=species.groupby('Park Name').count().get('Category'))
    .get(['Count']))

💡 "rename" 一列：groupby().count() 后所有列名等于原列名（如 'Category'、'Order' 等），值都相同（= 行数）。babypandas 没有 .rename()，所以 .assign(Count=...) 复制出 'Count' 列，再 .get(['Count']) 只保留它。

💡 7 parks → 7 rows ✓，sum = 17785 ✓。

Q 1.2parks_with_species：merge index ↔ column 
parks_with_species = parks.merge(species_counts, left_on='Park Name', right_index=True)

💡 right_index=True：因为 species_counts 是用 'Park Name' 当 index 而不是普通列。这是 merge index 的标准技巧。
alternative：先 species_counts.reset_index() 再普通 merge。

Q 1.3多列 groupby + rename → 76 rows 
species_category = (species.groupby(['Park Name', 'Category']).count()
    .reset_index()
    .assign(Count=species.groupby(['Park Name', 'Category']).count().reset_index().get('Order'))
    .get(['Park Name', 'Category', 'Count']))

💡 多列 groupby + count：每个 (Park, Category) 组合一行。reset_index 把这两列从 multi-index 拿回普通列。同 Q1.1 套路 rename。

⚠️ babypandas .assign(Count=1) 不能传 scalar——必须传 array / Series。如果要"全部填 1"得自己造 array。

📚 Section 2 — Coffee Shop

Q 2.1数 'Medium' 的个数 
number_medium = np.count_nonzero(ten_coffees == 'Medium')

💡 标准模板：array == val 生成 bool array，count_nonzero 数 True。任何"数符合条件的个数"都是这个套路。

Q 2.2coffee_price(size) 
def coffee_price(size):
    if   size == 'Small':  return 2.99
    elif size == 'Medium': return 3.99
    elif size == 'Large':  return 4.79
    else:                  return 4.99

💡 字符串比较：用 == 比较字符串。不是 =。

Q 2.3用 .apply 加 Price 列 
ten_coffees_df = ten_coffees_df.assign(
    Price=ten_coffees_df.get('Size').apply(coffee_price)
)

💡 .apply(函数名，不带括号)：传函数对象。不是 apply(coffee_price())（那是调用结果）。

Q 2.4数 Price > 4 
over_4_dollars = np.count_nonzero(ten_coffees_df.get('Price') > 4)
Q 2.5small_or_xl(coffee_df)—— 通用函数 
def small_or_xl(coffee_df):
    coffees = coffee_df.get('Size')
    n_s  = np.count_nonzero(coffees == 'Small')
    n_xl = np.count_nonzero(coffees == 'Extra Large')
    if   n_s > n_xl: return 'More Small coffees'
    elif n_xl > n_s: return 'More Extra Large coffees'
    else:             return 'Same amount'
⚠️ 用参数 coffee_df，不是全局 ten_coffees_df。函数要可复用——传任何 coffee DataFrame 都能跑。一旦在函数里硬编码全局名就破了通用性。

📚 Section 3 — Iteration

Q 3.1飞镖累加：1000 投的总分 
possible_point_values = np.arange(1, 11)
tosses = 1000

total_score = 0
for i in np.arange(tosses):
    total_score = total_score + np.random.choice(possible_point_values)

💡 累加器初值 = 0（加法 identity）。每轮添加一个随机值。期望总分 = 1000 × 5.5 = 5500。

Q 3.2平均分 
average_score = total_score / tosses

longer_than_8 = 0

Q 3.4收集所有 > 8 字母的词（array 累加器） 
long_words = np.array([])
for word in gatsby_words:
    if len(word) > 8:
        long_words = np.append(long_words, word)
⚠️ np.append 必须重新赋值。np.append(arr, x) 返回新 array，不修改原 array。如果只写 np.append(long_words, word) 不赋值，长度永远是 0。

📚 Section 4 — Hungry Billy / Marbles

Q 4.1wings_prob：能吃到的概率 
wings_prob = 1 - 0.1   # = 0.9

💡 P(available) = 1 − P(out)。别错填 0.1（那是 sold out 的概率，是相反方向）。

Q 4.24 项全有的概率（独立相乘） 
all_prob = 0.9 ** 4   # = 0.6561

💡 4 个 independent → P(all) = 0.9 × 0.9 × 0.9 × 0.9。

Q 4.3至少一项 sold out（complement） 
something_is_out = 1 - 0.9 ** 4   # = 0.3439

💡 "至少一个" 模板：P(at least 1) = 1 − P(none) = 1 − P(all available)。不用容斥（那会复杂）。这是高频陷阱：直觉是想用 + 但实际用 complement。

Q 4.43 色弹珠无放回 
winning_prob = (6 / 8) * (3 / 7)   # = 9/28 ≈ 0.3214

💡 推导：

  • 第 1 颗：任意（P = 9/9 = 1）。
  • 第 2 颗：必须跟第 1 颗不同色，剩 8 颗里 6 颗不同色 → P = 6/8。
  • 第 3 颗：必须是第 3 种颜色，剩 7 颗里 3 颗那色 → P = 3/7。
乘起来 = 1 × 6/8 × 3/7 = 18/56 = 9/28。

⚠️ 典型陷阱

陷阱 1：babypandas 没 .rename()——用 assign + drop。Q1.1, 1.3 反复用。
陷阱 2：assign 不能传 scalar（Count=1）——必须传 array/Series。
陷阱 3：np.append 必须重新赋值（Q3.4）。
陷阱 4：累加器初值——计数用 0，乘法用 1，array 用 np.array([])。
陷阱 5：apply 传函数名，不带 ()。
陷阱 6：函数用参数，不要在函数体里硬编码全局变量（Q2.5）。
陷阱 7："至少一个" 用 complement：1 − P(none)。
