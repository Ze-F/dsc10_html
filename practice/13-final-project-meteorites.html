📚 Section 2 — The View From Space（q2.1–2.9）

背景：假设你是国际空间站的宇航员能看见所有陨石。陨石落在某洲的概率 ∝ 该洲"夜灯亮度足够"的网格数 (= populated_cells)。只有 6 个洲（没有 Antarctica）。

📐 关键约定（贯穿整个 Section 2）
• probabilities = Series, 索引为 6 个 continent, 数值 ∈ [0,1], 和 = 1
• 每颗陨石独立同分布 → 用乘法规则
• 不会看到 Antarctica，因为它不在 probabilities 里
Q 2.1probabilities Series 
probabilities = populated_cells.get('Cells') / populated_cells.get('Cells').sum()
✅ 期望：isinstance(probabilities, bpd.Series) and len(probabilities) == 6 ｜ np.isclose(probabilities.sum(), 1) ｜ np.isclose(probabilities.loc['Asia'], 0.384311)

💡 数值速记：Asia ≈ 0.384、Africa ≈ 0.212、Europe ≈ 0.175、N.America ≈ 0.121、S.America ≈ 0.101、Australia ≈ 0.0067。Asia 最大、Australia 最小——这决定了后面 Q2.8 的"最高概率组合"。

Q 2.23 颗都落 Europe 
prob_all_europe = probabilities.loc['Europe'] ** 3

💡 乘法规则：3 颗独立 → P(A∩B∩C) = P(A)·P(B)·P(C) = p³。值 ≈ 0.175³ ≈ 0.00536。

Q 2.33 颗都不落 Europe 或 Asia 
prob_none_europe_asia = (1 - probabilities.loc['Europe'] - probabilities.loc['Asia']) ** 3

💡 拆解：单颗"不落 E 或 A" = 1 − P(E) − P(A)（互斥洲可以直接减）。3 颗都不落 → 立方。值 ≈ (1 − 0.175 − 0.384)³ ≈ 0.441³ ≈ 0.0856。

⚠️ 坑：互斥（mutually exclusive）洲的"或"才能直接相加。这里因为一颗陨石只落一个洲，所以 P(E ∪ A) = P(E) + P(A)。

Q 2.4prob_none(n, continents) 函数

"n 颗陨石都不落入给定 continents 中的任何一个的概率"

def prob_none(n, continents):
    p_in = sum(probabilities.loc[c] for c in continents)
    return (1 - p_in) ** n

💭 思路：continents 是 list / array。一颗"落入给定洲" 的概率 = 给定洲概率之和（互斥）。"都不落入" = (1 − 这个和)n。

💡 测试用例：prob_none(6, ['Europe', 'North America', 'Asia', 'Australia']) ≈ ?。Africa + S.America = 0.212 + 0.101 ≈ 0.313, 6 次方 ≈ 0.000932。

Q 2.5prob_all(n, continents) 函数

"n 颗陨石都落入给定 continents 中的某一个（可以是同一个）的概率"

def prob_all(n, continents):
    p_in = sum(probabilities.loc[c] for c in continents)
    return p_in ** n

💡 对照 Q2.4：prob_none = (1 − S)n，prob_all = Sn。S = 给定洲概率之和。"都落到给定洲群"≠"都落到同一个洲"——一颗 Asia、一颗 Africa 也算"都落到 [Asia, Africa]"。

Q 2.6用 prob_none 实现 prob_all_again

"prob_all_again should equal prob_all on any input."

def prob_all_again(n, continents):
    all_six = ['Asia', 'Africa', 'Europe', 'North America', 'South America', 'Australia']
    some_continents = [c for c in all_six if c not in continents]
    return prob_none(n, some_continents)

💭 关键洞察："n 颗都落入 X" ≡ "n 颗都不落入 X 的补集"（补集 = 6 大洲减去 X）。所以 prob_all(n, X) = prob_none(n, 6洲 − X)。

💡 边界情况：如果 X 就是 6 个全部，补集 = [] → (1 − sum([])) ** n = 1 ** n = 1，对（必落入 6 洲）。如果 X = [] → 补集 = 6 洲 → (1 − 1) ** n = 0，对。

⚠️ 坑：用 probabilities.index 替代硬编码也行：some_continents = [c for c in probabilities.index if c not in continents]。千万别把 Antarctica 加进列表——它不在 probabilities 里，会 KeyError。

Q 2.7order_probability(continents)

"看到 1 颗落 c₁、再 1 颗落 c₂、…的概率，按指定顺序"

def order_probability(continents):
    return np.prod([probabilities.loc[c] for c in continents])

💡 思路：每颗独立 → 顺序事件的联合概率 = 单个概率的连乘。np.prod([]) 默认返回 1.0（数学约定空积 = 1）。

Q 2.811 颗的最高概率组合 → highest_prob_combination

"6¹¹ 个 length-11 list 中，哪个 order_probability 最大？"

highest_prob_combination = ['Asia'] * 11

💭 思路（不写 for-loop）：连乘的每个因子都 ≤ Asia 的概率（0.384，最大）。要最大化连乘，每个因子都取最大 → 全 Asia。值 ≈ 0.384¹¹ ≈ 4.6 × 10⁻⁵。

💡 为什么不写 for-loop：6¹¹ ≈ 3.6 亿种组合，遍历会非常慢。题目 hint 已经说了"conceptual question"。

Q 2.9prob_one(n, continent) "恰好一颗"

"n 颗中恰好一颗落 continent 的概率。提示：n 种互斥情况（第 i 颗落入，其余 n-1 颗不落入）"

def prob_one(n, continent):
    p = probabilities.loc[continent]
    return n * p * (1 - p) ** (n - 1)

💭 公式拆解：单个 case "第 i 颗落入、其余 n−1 颗不落入" 的概率 = p · (1−p)n−1。共 n 个互斥 case → 总概率 = n · p · (1−p)n−1。

💡 测试用例：prob_one(3, 'Europe') ≈ 3 · 0.175 · 0.825² ≈ 0.358。autograder 期望 ≈ 0.358。选项 0.119 是 case 总差了——那是 1·p·(1-p)² 没乘 n。

📚 Section 3 — Long-Distance Relationship（q3.1–3.9）

背景：probabilities 是理论分布；自 1980 年以来真实看到的陨石分布 = observed。两者一致吗？假设检验！

📐 假设
H0: observed 是从 probabilities 抽样产生的，差异 = 随机抽样误差
HA: observed 不是从 probabilities 抽样的
统计量：TVD（categorical 分布距离，不带方向）
p 值方向：右尾（TVD 永远非负，距离越大越奇怪）
Q 3.11980 后看见的总数 + 观察分布 
since_1980 = saw_fall[saw_fall.get('year') >= 1980]
total_since_1980 = since_1980.shape[0]                            # = 179
observed_proportions = (
    since_1980.groupby('continent').count().get('name') / total_since_1980
)
✅ 期望：len(observed_proportions) == 6 and np.isclose(observed_proportions.sum(), 1)。179 颗是后续模拟次数的依据。

💡 为什么 1980：题目里说陨石记录方法在 1980 年后较稳定（无技术变迁偏差）——这是后面 Section 5 Q5.7 选 "2" 的伏笔。

Q 3.2generate_proportions() — H0 下的模拟 
def generate_proportions():
    return np.random.multinomial(179, probabilities) / 179

💭 思路：multinomial(n, p) 一次抽 179 颗、按 probabilities 分到 6 洲、返回长 6 的计数 array。除 179 → 比例。

💡 比 for-loop 快几十倍：等价于 179 次 np.random.choice(6, p=probabilities) 然后数频次，但 multinomial 一次搞定。

⚠️ 坑：返回是 array（不是 Series）。后面 tvd 要兼容两种类型。

Q 3.3tvd(a, b) + 10,000 次模拟 tvd_stats 
def tvd(a, b):
    return np.sum(np.abs(np.array(a) - np.array(b))) / 2

tvd_stats = np.array([])
for i in np.arange(10000):
    tvd_stats = np.append(tvd_stats, tvd(generate_proportions(), probabilities))
✅ 期望：len(tvd_stats) == 10000 ｜ np.isclose(tvd(np.array([0.5, 0.5]), np.array([0.4, 0.6])), 0.1)

💡 为什么 ÷ 2：直觉——每"多在 A 的"必然对应"少在 B 的"，所以绝对差被算了两次，÷ 2 修正。例：[0.5, 0.5] vs [0.4, 0.6] → |0.1| + |0.1| = 0.2，÷ 2 = 0.1（"10% 的 mass 错位"）。

💡 为什么 np.array(...) 转换：兼容 Series 输入。如果直接 a - b 在 Series-Series 上会按 index 对齐，但传 array 时不对齐——统一转 array 后按位置对齐，要求两边顺序一致。

⚠️ 顺序对齐：generate_proportions() 按 probabilities 的原始顺序返回。observed_proportions 是 groupby 出来按字母排的。两者顺序不同！但因为 probabilities 来自 populated_cells（设置 index='Continent' 后未排序），其实可能也是字母序——具体看 CSV。安全做法：把 observed_proportions .loc[probabilities.index] 重排一次。autograder 没明确测，但稳妥起见可以加这一行。

Q 3.4tvd_p_value 
observed_tvd = tvd(observed_proportions, probabilities)
tvd_p_value = np.count_nonzero(tvd_stats >= observed_tvd) / 10000

💡 永远右尾：TVD ≥ 0，"距离越大越像 HA"。所以用 >= 而不是 <= 或绝对差。

💡 典型值：观察 TVD 通常 ≈ 0.15–0.25；模拟 TVD 分布峰值 ≈ 0.04–0.07。所以 p-value 一般很小（接近 0），强烈 reject。

Q 3.5结论 q3_5 
q3_5 = 3   # We reject the null hypothesis

⚠️ 永远不要选 1："accept the null" 在统计学里不合法。我们最多说"fail to reject"。4 也错："null hypothesis is consistent with our observation" 意思也是接受 H0，错。2 错：p-value < 0.05 已经足够拒绝，不需要更多数据。

Q 3.6最大绝对差 maximum_distance 
def maximum_distance(a, b):
    return np.max(np.abs(np.array(a) - np.array(b)))

💡 跟 TVD 区别：TVD 是和除 2；MD 是取最大。例：[0.3, 0.4, 0.2, 0.1] vs [0.25, 0.25, 0.25, 0.25] → 绝对差 = [0.05, 0.15, 0.05, 0.15] → max = 0.15，TVD = sum/2 = 0.40/2 = 0.20。

⚠️ MD 不除 2。

Q 3.7通用 categorical_test(distance_fn, show_hist) 
def categorical_test(distance_fn, show_hist):
    observed_dist = distance_fn(observed_proportions, probabilities)
    sim_dists = np.array([])
    for i in np.arange(10000):
        sim_dists = np.append(sim_dists, distance_fn(generate_proportions(), probabilities))
    if show_hist:
        bpd.DataFrame().assign(simulated_dists=sim_dists).plot(
            kind='hist', density=True, bins=30, ec='w'
        )
        plt.axvline(x=observed_dist, color='black', label='observed')
        plt.legend()
    return np.count_nonzero(sim_dists >= observed_dist) / 10000

💡 "函数作为参数"：distance_fn 是个函数对象。调用 distance_fn(x, y) 时传入 tvd 或 maximum_distance 都行。这跟 Lec 8 讲的 "functions as first-class objects" 一脉相承。

⚠️ 坑：show_hist 是 True 才画图，否则只返回 p-value。不要把 plot 写在 if 外面。

Q 3.8MD 是否给出同 TVD 一致的结论？ 
md_same_as_tvd = True

💭 思路：实际数据中 observed_proportions 与 probabilities 差距很大（多在 Antarctica 周围），两种距离都会给出极小 p-value → 都 reject。所以 True。

⚠️ 一般性反例：MD 和 TVD不总一致——两个分布只在很多类别都有小差距时，TVD 大但 MD 小；只在一两类有大差距时，MD 大但 TVD 中等。这道题里恰好都 reject。

Q 3.9valid_distance — 哪些是合法距离函数？

合法 = (a) d(a, a) = 0，(b) d(a, b) > 0 当 a ≠ b

valid_distance = [1, 3]

💭 逐项分析

  • option1: np.mean((a-b)**2) — 平方非负，等时为 0，不等时 > 0 ✅
  • option2: np.mean((a-b)**3) — 三次方可能负（[0.3, 0.4]−[0.4, 0.3] = [−0.1, 0.1]，mean 0），❌
  • option3: len(a) * tvd(a, b) — len(a) > 0 是正常数，tvd 是合法距离，正缩放仍合法 ✅
  • option4: np.min((a+b)/2 - 1) — 对概率分布 a+b ≤ 2，(a+b)/2 − 1 ≤ 0，min 也 ≤ 0；且 d(a, a) ≠ 0 ❌

📚 Section 4 — Around the World（q4.1–4.5）

背景：Asia 看见的陨石质量中位数 > North America 的。是真有差异还是抽样波动？两组数值 → permutation test。

📐 假设
H0: Asia 和 N. America 的质量来自同一分布
HA: Asia 的中位数 > N. America 的
统计量：median(Asia) − median(NA)
p 值方向：右尾（"Asia 更大" 越大越奇怪）
Q 4.1传统 permutation（1000 次） 
asia_na = saw_fall[
    (saw_fall.get('continent') == 'Asia') | (saw_fall.get('continent') == 'North America')
].get(['mass', 'continent'])

asia_med = asia_na[asia_na.get('continent') == 'Asia'].get('mass').median()
na_med   = asia_na[asia_na.get('continent') == 'North America'].get('mass').median()

✅ 期望：asia_na 是 DataFrame、列为 {'mass', 'continent'} ｜ np.isclose(observed_diff, 488.13) ｜ len(diff_medians) == 1000 ｜ 0 ≤ mass_p_value ≤ 1 ｜ mean(diff_medians) ∈ [−100, 70]（均匀洗后差应在 0 附近）

💡 shuffle 哪一列都行：洗 continent 列 ≡ 洗 mass 列。这是 permutation test 的对称性。

💡 1000 次能跑出来吗：传统 permutation（用 .assign + 双过滤）每次 ≈ 几十毫秒，1000 次需要几十秒——边界但可接受。10,000 次会非常慢，这就是 Q4.2 要换"faster perm"的动机。

Q 4.2faster permutation: single_simulated_group_diff

"Shuffle mass 一次，前 n_asia 个分给 Asia，其余给 NA。计算 func(asia) − func(na)。"

def single_simulated_group_diff(input_df, func):
    n_asia = (input_df.get('continent') == 'Asia').sum()
    shuf   = np.random.permutation(input_df.get('mass'))
    asia_vals = shuf.take(np.arange(n_asia))
    na_vals   = shuf.take(np.arange(n_asia, len(shuf)))
    return func(asia_vals) - func(na_vals)

💭 关键技巧：跳过 .assign 和 groupby——这两个在 babypandas 里特别慢。直接对 numpy 数组切片，10 倍以上提速。

💡 为什么洗 mass 也行：本质上 permutation 就是"切断 mass 和 continent 的关联"。洗哪边都切断了。

💡 arr.take(np.arange(a)) = arr[:a]：题目 hint 给的是 take 法，因为对 Series 来说 .iloc 切片更安全。这里 shuf 已经是 numpy array，两种都行。

⚠️ func 必须接受 array：func(asia_vals) 直接传 numpy 数组。不要把它转 Series——babypandas 的 Series 不支持 axis 参数。

Q 4.3many_simulated_group_diffs 
def many_simulated_group_diffs(input_df, func, repetitions):
    diffs = np.array([])
    for i in np.arange(repetitions):
        diffs = np.append(diffs, single_simulated_group_diff(input_df, func))
    return diffs

💡 没什么花活：就是把 Q4.2 包在循环里。但因为 Q4.2 已经很快了，现在 10,000 次只要几秒。

Q 5.2通用 continent_median_ci(continent, p) 
def continent_median_ci(continent, p):
    sub = saw_fall[saw_fall.get('continent') == continent]
    boot = np.array([])

Q 5.4North America 均值的 95% CLT CI 
na_mass = saw_fall[saw_fall.get('continent') == 'North America'].get('mass')
mean = na_mass.mean()
sd   = np.std(na_mass)
n    = na_mass.count()

✅ 期望：na_mean_ci[0] < 12300

💡 必背 CLT 95% CI 公式：[mean − 2·SD/√n, mean + 2·SD/√n]。SD 是 sample 的 SD，不是 sample mean 的 SD。√n 是把"个体 SD"换算成"均值 SD"。

💡 注意 vs bootstrap：bootstrap 算中位数的 CI（CLT 只对均值有效）；CLT 算均值的 CI（不需要重采样）。

⚠️ np.std vs .std()：np.std 默认 ddof=0（除 N，population SD），Series.std() 默认 ddof=1（除 N−1，sample SD）。课程基本用 np.std，对大 n 来说差别可忽略。

Q 5.5find_z(p) — 反查 z

"p% 值落在 −z 到 +z 间。find_z(68) ≈ 0.99；find_z(95) ≈ 1.96"

def find_z(p):
    return norm.ppf((1 + p / 100) / 2)

💭 公式推导：p% 在 [−z, z] 内 → cdf(z) − cdf(−z) = p/100。由对称性 cdf(−z) = 1 − cdf(z)，所以 2·cdf(z) − 1 = p/100 → cdf(z) = (1 + p/100)/2。z = norm.ppf((1 + p/100)/2)。

💡 验证：find_z(95) = norm.ppf(0.975) ≈ 1.959964 → 与 95%-CI 经验 z=2 吻合。find_z(99) ≈ 2.576。find_z(68) ≈ 0.994 ≈ 1。

Q 5.6通用 continent_mean_ci(continent, p) 
def continent_mean_ci(continent, p):
    masses = saw_fall[saw_fall.get('continent') == continent].get('mass')
    mean = masses.mean()
    sd   = np.std(masses)
    n    = masses.count()
    z    = find_z(p)
    margin = z * sd / np.sqrt(n)
    return np.array([mean - margin, mean + margin])

np.random.multinomial(n, p) 永远返回长度 = len(p) 的 array。在 tvd 里要兼容 array 和 Series，所以 np.array(a) - np.array(b)。

一个是字母序，一个是 CSV 原序。稳妥做法：observed_proportions = observed_proportions.loc[probabilities.index] 重排一次。

陷阱 16：faster perm 不要用 .assign 或 groupby

陷阱 17：func 接收 array 不接收 Series
np.median(series) 在 babypandas 报 "axis" 错。要么转 np.array(series)，要么 func 处理 array。
陷阱 18：Q6.3 必须 replace=False

📐 CI 类（Section 5）

陷阱 20：bootstrap CI 只能算 中位数，CLT CI 只能算 均值
bootstrap 不挑统计量（中位数、均值、任何都行）；CLT 只对均值有效。Q5.1–5.3 是 median 必须 bootstrap，Q5.4–5.6 是 mean 必须 CLT。

[(100−p)/2, (100+p)/2]。95% → [2.5, 97.5]，80% → [10, 90]，90% → [5, 95]。

错误：TypeError: agg function failed [how->median, dtype->object] / Cannot convert ['Achiras' 'Alexandrovsky' ...] to numeric

    a = tmp[tmp.get('g') == 'A'].get('val').median()
    b = tmp[tmp.get('g') == 'B'].get('val').median()
    diffs = np.append(diffs, a - b)

模板 E — Faster permutation（快 10x+）

def single_diff(input_df, func):
    n_a = (input_df.get('group') == 'A').sum()
    shuf = np.random.permutation(input_df.get('val'))
    return func(shuf.take(np.arange(n_a))) - func(shuf.take(np.arange(n_a, len(shuf))))

sub = df[df.get('group') == 'A']

ci_p  = np.percentile(boot, [(100-p)/2, (100+p)/2])

模板 G — CLT-based CI（只能算均值）

vals = df[...].get('val')
mean = vals.mean()
sd   = np.std(vals)
n    = vals.count()

# 95% 经验法
ci_95 = np.array([mean - 2*sd/np.sqrt(n), mean + 2*sd/np.sqrt(n)])

z = find_z(p)
ci_p = np.array([mean - z*sd/np.sqrt(n), mean + z*sd/np.sqrt(n)])
