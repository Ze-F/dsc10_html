📚 Section 1 — How faithful is Old Faithful?（Q1.1–1.6）

Old Faithful 数据集每一行一次喷发，两列：duration（喷发时长 / 分钟）和 wait（到下一次喷发的等待时间 / 分钟）。共 272 行。目标：用 duration 预测 wait。

Q 1.1Scatter plot of the data

"Make a scatter plot of the data. It's conventional to put the column we will try to predict on the vertical axis and the other column on the horizontal axis."

💭 思路：要预测 wait → wait 上 y 轴，duration 上 x 轴。

faithful.plot(kind='scatter', x='duration', y='wait')

💡 为什么：scatter 的目的是 先肉眼看 relationship，再决定要不要用 linear regression。如果根本不是 roughly linear 的形状，硬套 regression line 就是错的（见 r-linear 章节）。

Q 1.2Roughly linear? Positive or negative?

选项：1 = 不线性；2 = 线性 + 负相关；3 = 线性 + 正相关。

💭 思路：散点是从左下到右上（duration 越长，等待越久）→ 大致 linear + positive。

faith_q2 = 3

💡 为什么：在做 regression 之前肉眼判断 association direction 是好习惯——之后算 r 出来，符号要和你预测一致；不一致就要回头查代码。

Q 1.3Means, standard deviations, & faithful_su

"Compute the mean and standard deviation of the eruption durations and waiting times. Then create a DataFrame called faithful_su containing the eruption durations and waiting times in standard units. You must use np.std to calculate the standard deviation."

💭 思路：标准单位 = (x − mean) / SD。每列分别做。最后用 bpd.DataFrame().assign(...) 拼出两列。

duration_mean = faithful.get('duration').mean()
duration_std  = np.std(faithful.get('duration'))   # 注意是 np.std，不是 .std()
wait_mean     = faithful.get('wait').mean()
wait_std      = np.std(faithful.get('wait'))

faithful_su = bpd.DataFrame().assign(
    duration_su = (faithful.get('duration') - duration_mean) / duration_std,
    wait_su     = (faithful.get('wait')     - wait_mean)     / wait_std
)
faithful_su

💡 为什么 np.std vs .std()：np.std(arr) 默认除以 n（population SD），DSC10 全课都用这个。而 pandas 的 Series.std() 默认除以 n−1（sample SD，Bessel's correction），结果不一样！autograder 卡死的就是这个。

⚠️ 高频陷阱：把 np.std(faithful.get('duration')) 写成 faithful.get('duration').std()，duration_std 就会从 1.1393 变成 1.1414，整个 lab 的所有数字全错。题目里说"You must use np.std"是真的必须用。
✅ autograder 反推数值：duration_std ≈ 1.139271210225768、wait_std ≈ 13.569960017586371。

"Create a scatter plot of the data again, but this time in standard units."

faithful_su.plot(kind='scatter', x='duration_su', y='wait_su')

Q 1.5Correlation guess

"Which would you guess is closest to the correlation: −1, 0, or 1?"

correlation_guess = 1

💡 为什么：散点强烈正相关，接近一条 line。所以 r 接近 1（精确值是 0.9008，但 lab 在此只让你三选一）。

Q 1.6Implement correlation and compute r

"Complete the implementation of the function correlation, which takes in a DataFrame df and two column names x and y and returns the correlation between the two columns."

💭 思路：定义 r ≡ mean of (su(x) · su(y))。lab 已经预先给了 standard_units(any_numbers) 函数，可以直接用。

def correlation(df, x, y):
    x_su = standard_units(df.get(x))
    y_su = standard_units(df.get(y))
    return (x_su * y_su).mean()

r = correlation(faithful, 'duration', 'wait')
r

💡 为什么 r 和 SU 是否独立无关：题目 hint 提到 "Does it matter if we compute the correlation between 'duration_su'/'wait_su' or 'duration'/'wait'?" 答：不影响。correlation 函数内部第一步就是把列转成 SU，再把 SU 又转一次 SU 还是原样。所以传 original 或 standardized 进去结果一样。

✅ 期望值：r ≈ 0.9008111683218132。

📚 Section 2 — The regression line（Q2.1–2.2）

Lec 23 教过：在 standard units 下，regression line 是 wait_su = r × duration_su（slope=r，过原点）。Section 2 的目标是把这条 line 换算回 original units，拿到 slope 和 intercept。

Q 2.1Slope in original units

题干给出"stretching"直觉：把 SU 的 x 横向乘 duration_std 还原；横拉变缓 → slope 除以 duration_std。把 SU 的 y 纵向乘 wait_std 还原；纵拉变陡 → slope 乘以 wait_std。

💭 思路：起点 slope = r（SU 下的）。横拉、竖拉 → slope_original = r × wait_std / duration_std。

slope = r * wait_std / duration_std
slope

💡 为什么这个公式叫"必背 5 件套"之一：original units 下 slope 的符号由 r 决定（SD 永远非负），数值则由 SD 比值放大。这也是为什么 r 接近 ±1 时 slope 不一定大——还要看 SD_y/SD_x。

✅ autograder 反推：(slope * 13 - 100) / 98 ≈ 0.40291... → 解出 slope ≈ 10.7296。验算：0.9008 × 13.5700 / 1.1393 ≈ 10.7296 ✓
对比角度：见 final/02-regression §unit-transforms，那里讲了改单位（cm→m、°F→°C）对 slope/intercept/r 的影响。
Q 2.2Intercept

"The regression line passes through the point (duration_mean, wait_mean). Rearrange waiting time − wait_mean = slope × (eruption duration − duration_mean) to find the intercept."

💭 思路：把上式写成 y = mx + b 的形式 → b = wait_mean − slope × duration_mean。

intercept = wait_mean - slope * duration_mean
intercept

💡 为什么 line 必过 (mean_x, mean_y)：这是 least squares 解的必然性质——见 final/02-regression §least-squares。考试反推 intercept 的题型几乎都基于这一性质。

✅ autograder 反推：intercept ≈ 33.4744。

📚 Section 3 — Investigating the regression line（Q3.1–3.3）

有了 slope & intercept，wait = slope × duration + intercept 就是 regression line。这一节练用它做预测，并定义 residual。

Q 3.1Predict for duration = 2 and 5 minutes 
two_minute_predicted_waiting_time  = slope * 2 + intercept
five_minute_predicted_waiting_time = slope * 5 + intercept
✅ 期望：
• 2 分钟 → ≈ 54.93（autograder: 54.93 < … < 54.94）
• 5 分钟 → ≈ 87.12（autograder: 87.12 < … < 87.13）
Q 3.2Predictions for every row → faithful_predictions

"Make predictions for the waiting time after each eruption in the faithful DataFrame. There is no need for a for-loop or even .apply; use Series arithmetic instead."

💭 思路：Series 乘标量、加标量都是 broadcasting，一行搞定。然后 .assign 接一列。

faithful_predictions = faithful.assign(
    predicted_wait = slope * faithful.get('duration') + intercept
)
faithful_predictions

💡 为什么不用 .apply：Series 算术（slope * series + intercept）天然向量化，速度比 .apply 快几十倍，代码也短。这正是 numpy/pandas 设计初衷。

✅ 验证第一行：duration=3.600 → 10.7296×3.6+33.4744 ≈ 72.101 ✓ （题干表格给的就是这个值）
Q 3.3Residuals → faithful_residuals

"Residuals are defined as follows (note that there is no absolute value): residual = actual wait − predicted wait."

faithful_residuals = faithful_predictions.assign(
    residual = faithful_predictions.get('wait') - faithful_predictions.get('predicted_wait')
)
faithful_residuals

💡 为什么没有绝对值：residual 有正负——正残差表示 line 低估了真实值，负残差表示 高估。如果取绝对值，就丢失了过/欠预测的方向信息，也无法验证"residuals 之和 = 0"（least squares 的关键性质）。

⚠️ 极常见陷阱：方向写反——residual = predicted − actual 是错的（虽然绝对大小一样，但符号反了，所有后续残差图的正负、上下都会反过来）。题目里的"no absolute value"加粗强调，就是怕你这两个都写错。
对比角度：见 final/02-regression §errors 和 final/03-residuals §residuals。

📚 Section 4 — How accurate are different predictions?（Q4.1–4.2）

Q 4.1Predict for 0 / 2.5 / 60 minutes 
zero_minute_predicted_waiting_time           = 0    * slope + intercept
two_point_five_minute_predicted_waiting_time = 2.5  * slope + intercept
hour_predicted_waiting_time                  = 60   * slope + intercept

💡 为什么这三个值这么挑：lab 故意挑了 三种危险性递增的输入——0（物理上不可能）、2.5（在数据范围内但没有精确值）、60（远远超出数据范围）。下一题就要你判断哪些预测可信。

Q 4.2true_predictions — 哪些预测可信？

5 个选项中选所有正确的（list 形式）：1=0 min 可信；2=2.5 min 可信；3=60 min 可信；4=我们有所有被预测 duration 的数据；5=我们有所有被预测 duration 上下的数据。

💭 思路：先查数据范围。faithful 的 duration 范围 ≈ 1.6–5.1 分钟。

  • 0 → 远低于范围，且物理无意义 → 不可信。
  • 2.5 → 在 1.6–5.1 内，可信。虽然数据中没有 恰好 2.5 的点，但上下都有点。
  • 60 → 远远超出范围（5.1 分钟以上一片空白）→ 不可信，典型 extrapolation。
  • 4 → "have data for all of the durations we predicted for"——0 和 60 都没有数据，错。
  • 5 → "have data surrounding (above and below) all of the durations"——0 没有下方数据，60 没有上方数据，错。

true_predictions = [2]

💡 为什么这是 Final 高频题型：Extrapolation 是 regression 的核心警告——line 在数据范围内可信，超出范围就是赌博。期末考"哪个预测可信"的题目几乎每年都有，关键点都是检查 input 是否在 样本 的 x 范围内。

⚠️ 常见错答：把 [1, 2, 3]（"all reliable"）或 [2, 5] 之类选项打包。错。autograder 明确 set(true_predictions) == {2}。

📚 Section 5 — Divide and conquer（Q5.1–5.5）

散点图显示 duration 有两簇：一簇在 ~2 分钟，一簇在 3.5–5 分钟。3 分钟可以划开两簇。这节用 divide-and-conquer 给每簇单独拟合一条 line，看预测是否更准。

Q 5.1Split + 各自 correlation

"Create two DataFrames, below_3 and above_3. below_3 should contain all rows in faithful in which the duration is below 3, and above_3 should contain all rows in faithful in which the duration is greater than or equal to 3. Then call your correlation function on both."

below_3 = faithful[faithful.get('duration') < 3]
above_3 = faithful[faithful.get('duration') >= 3]

below_3_r = correlation(below_3, 'duration', 'wait')
above_3_r = correlation(above_3, 'duration', 'wait')

💡 为什么两个子集的 r 反而比整体小：直觉反常。整体 r ≈ 0.90 之所以高，部分原因是两簇之间有大间隔，让"整体来看高度线性"。一旦把两簇拆开，每一簇内部的 r 就只反映该簇内部的 association ——而每一簇内点的 spread 都不太线性。所以 below_3 r ≈ 0.29，above_3 r ≈ 0.37。

✅ 期望：below_3.shape[0] = 97、above_3.shape[0] = 175、[below_3_r, above_3_r] ≈ [0.2902, 0.3728]。
⚠️ babypandas 提醒：faithful[boolean_series] 不是普通方括号取列——这是布尔过滤。要点是布尔 Series 必须用 &/|（不是 and/or），且每段比较外加括号。这里只有一个条件，没那么复杂。
Q 5.2slope_of / intercept_of

"Both functions should take in a DataFrame df that contains a duration column and a wait column (both in original units, not standard units)."

def slope_of(df):
    '''Return the slope of the regression line for the given DataFrame in original units.'''
    r = correlation(df, 'duration', 'wait')
    return r * np.std(df.get('wait')) / np.std(df.get('duration'))

def intercept_of(df):
    '''Return the intercept of the regression line for the given DataFrame in original units.'''
    return df.get('wait').mean() - slope_of(df) * df.get('duration').mean()

below_3_a = slope_of(below_3)
below_3_b = intercept_of(below_3)
above_3_a = slope_of(above_3)
above_3_b = intercept_of(above_3)

def wait_below_3(duration):
    return below_3_a * duration + below_3_b

def wait_above_3(duration):
    return above_3_a * duration + above_3_b

💡 为什么把 slope 抽成函数：之前 Section 2 是对整体 faithful 算一次 slope/intercept；现在要对每一簇（below_3 / above_3）各算一次。把公式封装进函数比复制粘贴四遍干净，也是 Lab 7 想让你练的 abstraction。

⚠️ 题干强调"original units"：函数内 np.std(df.get('wait'))，不是 standard units 下的列。如果你不小心传了 faithful_su，得到的 SD 全是 1.0，公式就退化成 slope = r 了（这就是 SU 下的公式，不是 original units）。
✅ 期望：[wait_below_3(1), wait_below_3(3), wait_above_3(3), wait_above_3(6)] ≈ [47.90, 60.60, 72.97, 89.28]。
Q 5.3predict_wait：分段函数 
def predict_wait(duration):
    '''Return the wait predicted by the appropriate one of the two regression lines above.'''
    if duration < 3:
        return wait_below_3(duration)
    else:
        return wait_above_3(duration)

💡 为什么用 < 3（含等号在 else）：与 Q5.1 划分一致——below_3 用 < 3，above_3 用 >= 3，所以 duration == 3 走 above_3 那条 line。如果你的 if 写成 <= 3，duration=3 就会走错 line（用 below 的参数），整张图会有 ε 错位。

✅ 期望：[predict_wait(1.5), predict_wait(2.5), predict_wait(3.5), predict_wait(4.5)] ≈ [51.08, 57.43, 75.68, 81.12]。
Q 5.4New residuals + residual plot

前一个 cell 已经给了 new_faithful = faithful.assign(predicted = faithful.get('duration').apply(predict_wait))。这里继续加 new_residuals 列，并画残差图。

new_faithful = new_faithful.assign(
    new_residuals = new_faithful.get('wait') - new_faithful.get('predicted')
)

new_faithful.plot(kind='scatter', x='duration', y='new_residuals',
                  color='purple', figsize=(10, 5));

💡 为什么这里用 .apply 是合理的：predict_wait 内含 if 分支，不是简单的向量运算，Series 算术做不了。所以预先 cell 用 .apply 把 predict_wait 套在每一行的 duration 上，是必须的——这也是 .apply 真正的用场（向量化做不到时）。

对比 Q3.2：那里能纯 Series 算术（slope * series + intercept），就 不要 用 .apply；这里有 if 必须 .apply。区分这两种情况是 babypandas 的功夫。
Q 5.5new_predict — 新预测是否更准？

选项：1=new 残差的 max/min 都更小，所以更准；2=new 残差的 spread 更小，所以更准；3=新预测更差（不能预测 duration=3 处）；4=两组残差看起来相似，无法判断。

💭 思路：肉眼比较两张残差图。divide-and-conquer 把两簇分别拟合，每一簇内的残差更紧密贴近 0 line → spread 更小 → 选项 2。

new_predict = 2

💡 为什么不是选项 1：max/min 只看 极端残差，可能两组的极值都差不多——但平均 spread（多数点偏离 0 的程度）变小才是 regression 改善的标志。这对应 RMSE 的概念：RMSE = √(mean(residual²))，spread 小 → RMSE 小 → 预测平均更准。见 final/02-regression §rmse。

💡 为什么不是选项 4：两张图肉眼明显不同。新残差图基本是"无 pattern 的小 blob"（divide and conquer 后），旧残差图则在 duration=2 附近有明显的左下→右上倾斜（残差有 pattern → 单条 line 不够 fit）。

对比角度：residual plot 4 种 pattern 在 final/03-residuals §patterns。Lab 7 此处看到的就是经典的"原模型 underfitting，拆开后变 patternless blob"案例。

⚠️ 本作业典型陷阱（横切多题）

陷阱 1：np.std 与 .std() 不等价
np.std(arr) 默认 ÷ n（population，DSC10 用这个）；Series.std() 默认 ÷ n−1（sample，pandas/babypandas 用这个）。Lab 7 题干明示要用 np.std。一旦写错，全 lab 数值都偏 ε。必背：DSC10 课程内的 SD 永远是 np.std。
陷阱 2：residual 方向写反
residual = actual − predicted ✅，不是 predicted − actual。前者的正负含义是"实际比预测高多少"——line 低估时 residual 为正。后者绝对值相同但符号相反，残差图全部上下翻转，所有性质（mean = 0，line 上方点对应正残差）都反了。
陷阱 3：Extrapolation 不可信
任何 duration 在 [1.6, 5.1] 之外的预测都不可靠。即便 line 在那里"看起来很正常"，line 是从数据拟合来的，外推就是用没见过的 region 做赌博。Final 考试 100% 会问"哪些预测可信"——锁定在数据范围内的输入即可。
陷阱 4：r 接近 1 ≠ slope 大
Q2.1 的 slope = r × SD_y / SD_x：slope 受 SD 比值放大。Old Faithful 的 r=0.9008 但 slope=10.73，因为 SD_wait/SD_duration ≈ 11.9。r 只决定 slope 的 符号 和"在 SU 下的大小"，不决定 original units 的大小。
陷阱 5：判断模型质量看 spread 不是 max/min
Q5.5 选项 1（"max/min 都更小"）是错的诱饵。模型好坏看 residuals 的整体 spread（RMSE 的体现），不是单个极端残差。题干故意把 max/min 写得很具体来骗人。
陷阱 6：什么时候用 .apply、什么时候不用
Q3.2、Q3.3 用 Series 算术（向量化、快）：slope * series + intercept。Q5.4 必须 .apply，因为 predict_wait 内含 if 分支，单步向量化做不到。考试时如果函数内有条件分支，就该 .apply；如果是简单算术，禁用 .apply（写出来会被扣"不优雅"分）。
陷阱 7：subset 的 r 反而比整体小
Q5.1 看到的反直觉现象：整体 r=0.90，但 below_3 r=0.29、above_3 r=0.37。原因是整体的高 r 部分来源于两簇之间的间隔；一旦在簇内看，association 没那么强。这也解释了为什么 divide and conquer 反而能提高预测准度——不是因为 r 高，而是因为每个 line 专心拟合一个 cluster 的 mean，避开"用一条线穿过两簇"的妥协。
