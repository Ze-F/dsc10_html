📚 Section 1 — Secret Smiski（Q1.1–1.6）

背景：每个 Smiski 是 secret 的概率官方说是 1/144。DSC 10 tutors 收集了 903 个，发现 10 个是 secret（observed 比例 ≈ 10/903 ≈ 0.01107，比 1/144 ≈ 0.00694 高）。问：真实概率是不是更高？

📐 假设：H0: p = 1/144 ｜ HA: p > 1/144（单侧！）
统计量：sample proportion − 1/144（保留方向，越大越支持 HA）
Simulate under H0：np.random.multinomial(903, [1/144, 143/144])
Q 1.1one_simulation()

"Should randomly generate 903 Smiskis under the assumptions of the null hypothesis and return the proportion of Smiskis that are secret. Use np.random.multinomial, no for-loop."

💭 思路：np.random.multinomial(n, [p₁, p₂]) 返回长度为 2 的 array，是每类的计数。第 0 位是 secret 数量，除以 903 就是比例。

def one_simulation():
    return np.random.multinomial(903, [1/144, 143/144])[0] / 903

💡 为什么 multinomial 不要 for-loop：multinomial(903, p) 一次抽 903 个独立伯努利，比"for 903 次 random.choice"快几十倍。这是 babypandas 课程里"避免显式 for 循环"理念的体现。

💡 返回是 array 不是 int：multinomial 永远返回 array（即使只有两类），所以要 [0] 取第一类。返回 [0]/903 是 Python float，是题目想要的比例。

Q 1.2观测统计量 smiski_observed

"Calculate the observed value of the test statistic. test statistic = proportion of secret Smiskis in sample − 1/144."

smiski_observed = 10 / 903 - 1 / 144
✅ 值：≈ 0.01107 − 0.00694 = 0.00413（autograder 提示 "very small number"，< 0.01）。

💡 为什么带方向（不取绝对值）：HA 是 "p > 1/144"，单侧——只有"sample 比例偏高"才支持 HA。带方向保留了"高/低"信息，所以 p 值用右尾。如果题目改成 "p ≠ 1/144"（双侧），就用 |obs − 1/144|，p 值用右尾的绝对值。

Q 1.310,000 次模拟 → proportion_diffs 
proportion_diffs = np.array([])
for i in np.arange(10000):
    proportion_diffs = np.append(proportion_diffs, one_simulation() - 1/144)

💡 为什么这里要 for 循环：每次 one_simulation() 是一次独立实验，10,000 次需要 10,000 个不同的随机结果。multinomial 本身只一次抽 903 个；要 10,000 个 trial 就是 10,000 次 multinomial。这跟"避免 for"不矛盾——避免的是在数据中用 for 遍历行，不是避免 simulation 的 for。

✅ 测试：isinstance(proportion_diffs, np.ndarray) and len(proportion_diffs) == 10000
Q 1.4p 值 smiski_p

"Do large values of our test statistic favor the alternative hypothesis, or do small values?"

💭 思路：HA: p > 1/144 → 大值（sample 比例 > 1/144 → 差为正）支持 HA。p 值 = 模拟中 ≥ observed 的比例 →右尾。

smiski_p = np.count_nonzero(proportion_diffs >= smiski_observed) / 10000

💡 经典口诀："多大算太大" 看 HA 方向。HA 说"实际更高" → 越大越奇怪 → 右尾（≥）。如果 HA 说"实际更低" → 左尾（≤）。

Q 1.5结论 smiski_conclusion

在 0.05 cutoff 下选 1/2/3/4。

💭 思路：H0 下期望 secret 数 = 903 × 1/144 ≈ 6.27，SD ≈ 2.50，observed = 10 个 → z ≈ 1.49 → 单边 p ≈ 0.068。p > 0.05 → fail to reject。

smiski_conclusion = 4
⚠️ 用语必背：
  • p < cutoff → "reject the null" + "data is inconsistent with the model"。不要说 "accept the alternative"。
  • p ≥ cutoff → "fail to reject the null" + "not enough evidence to say data is inconsistent"。永远不要说 "We accept the null"（选项 3 是陷阱）。
所以选项 4 是唯一对的；2 错在 "reject"；1 错在 "draw a conclusion" 太弱；3 错在 "accept"。
见 quiz4/03-pvalues-decisions §concepts 的"三种结论"。
Q 1.6哪些统计量不合适？bad_choices

"From the options below, choose the test statistics that would not have worked. Hint: we want a statistic that helps us determine whether we got secret Smiskis more often than expected."

bad_choices = [2, 3]

💡 为什么 2、3 不行：两者都是绝对值差（|10 − count|、|1/144 − prop|）。绝对值丢失方向——"高很多"和"低很多"会得到同样大的统计量，无法区分 "secret 更多" vs "secret 更少"。题目想测单侧"more often"，必须保留方向。

💡 为什么 1、4、5 都行：

  • 1：number of secret（903 张中的 secret 数）—— 越大越支持 HA（右尾）。
  • 4：1/144 − proportion —— 越负越支持 HA（左尾）。p 值方向反一下即可。
  • 5：proportion of secret —— 越大越支持 HA（右尾）。
关键是单调——同一方向越极端越可疑。绝对差不是单调的。

📚 Section 2 — Gotta Catch 'Em All / Pokémon TCGP（Q2.1–2.5）

背景：Eric 提出 8 类卡牌的概率分布（proposed_dist）。1000 张抽样得到 observed_dist。问：observed 是不是真的从 proposed 分布抽出来的？

📐 假设：H0: Pokémon TCGP 牌从 proposed_dist 抽 ｜ HA: 不是
统计量：TVD = 0.5 × Σ|pi − qi|
Simulate under H0：从 proposed_dist 抽 1000 张，算与 proposed 的 TVD
p 值：右尾（≥），因为 TVD ≥ 0 且越大越偏
Q 2.1哪个不合理 unreasonable_test_statistic

三个选项：1=各类绝对差之和；2=两 distribution 概率之和的绝对差；3=最大绝对差。

unreasonable_test_statistic = 2

💡 为什么选项 2 是傻的：两个 probability distribution，每个内部加起来都 = 1（题干都强调 sum = 1）。所以 |Σproposed − Σobserved| = |1 − 1| = 恒为 0。没区分能力。

💡 1 和 3 都合理：1 是 2×TVD（差个常数因子）；3 是 L∞ 距离。都能反映"偏离 proposed 多远"。

Q 2.2total_variation_distance & observed_tvd 
def total_variation_distance(first_distrib, second_distrib):
    '''Computes the total variation distance between two distributions.'''
    return np.sum(np.abs(first_distrib - second_distrib)) / 2

observed_tvd = total_variation_distance(proposed_dist, observed_dist)

💡 为什么 ÷ 2：两个 distribution 的正偏差和负偏差大小相等（都从 1 重新分配），所以 Σ|差| 把每个错配的概率算了两次——一次在它本来该去的类，一次在它实际去的类。÷ 2 才是"总偏离量"。

✅ 反推：差值 array ≈ [−0.014, −0.024, −0.004, −0.005, 0.002, 0.012, 0.008, 0.025]，|差|和 = 0.094 → TVD = 0.047。autograder 验：int(0.047 × 10⁷ % 8³) == 496 ✓
⚠️ TVD 必须 ≤ 1 且 ≥ 0。如果你的结果是 0.094（忘了 ÷ 2），autograder 会过 0 ≤ tvd ≤ 1 但过不了具体数值检查。
Q 2.35000 次模拟 TVDs 
simulated_tvds = np.array([])
for i in np.arange(5000):
    simulated = np.random.multinomial(1000, proposed_dist) / 1000
    simulated_tvds = np.append(simulated_tvds,
                               total_variation_distance(proposed_dist, simulated))

💡 为什么 ÷ 1000：multinomial 返回计数，TVD 比的是分布（概率），所以要除以 n=1000 转回比例。如果直接比 counts，TVD 会膨胀 1000 倍。

Q 2.4p 值 pokemon_p 
pokemon_p = np.count_nonzero(simulated_tvds >= observed_tvd) / 5000

💡 为什么 TVD 永远用右尾：TVD ≥ 0 是设计上的下界，越偏离越大。所以"模拟中比 observed 还大的比例"才是 p 值。永远不会有左尾 TVD test。

Q 2.5结论 pokemon_conclusion（cutoff 0.01！）

💭 思路：观察 observed_tvd ≈ 0.047；H0 下 TVD 期望 ≈ 0.032，SD ≈ 0.009，z ≈ 1.7 → p ≈ 0.04–0.05。p > 0.01 → fail to reject。

pokemon_conclusion = 4
⚠️ cutoff 是 0.01 不是 0.05！如果按 0.05 的直觉就会选 3（reject）。仔细看题。
⚠️ 实际跑出来 p 值有 noise：5000 次模拟有随机性，你的 p 可能 0.03 也可能 0.06。如果偏巧 < 0.01，conclusion 改成 3（reject）。建议跑两次取众数。

📚 Section 3 — Chocolate（Q3.1–3.9）— Permutation 测试经典模板

背景：2693 块巧克力，每块有 Rating（1–5）和 Characteristics 文本。我们把含 "sweet"（独立词）的标 Sweet，其余 Not Sweet。问：Sweet 的 Rating 是不是比 Not Sweet 低？

📐 假设：H0: 两组 Rating 来自同一分布 ｜ HA: Sweet 的 Rating 平均更低
统计量：mean(Sweet) − mean(Not Sweet)
Simulate under H0：洗 Sweetness 列（permutation）
p 值：左尾（≤），因为 HA 说 Sweet 更低 → 差越负越极端
Q 3.1label_sweet：识别独立 "sweet" 词

"'sweet' appears in the string of characteristics, but not if it appears only as part of a longer word, such as 'bittersweet'."

def label_sweet(characteristics):
    for word in characteristics.replace(',', ' ').split():
        if word == 'sweet':
            return 'Sweet'
    return 'Not Sweet'

💡 为什么先替换逗号：原始格式是逗号 + 空格分隔（如 "milk brownie, macadamia, chewy"）。split() 默认按空白切，"chewy," 会带个逗号——比较时 "chewy," == "sweet" 永远 False。先把逗号变空格，再 split 就干净了。

💡 为什么不能用 'sweet' in characteristics：那会把 'bittersweet' 也算上（子串匹配）。题目要求独立词——必须按 token 比较。

💡 测试样例：label_sweet('sweet, woody') → 'Sweet'；label_sweet('creamy, too sweet, sour') → 'Sweet'；label_sweet('nutty, bittersweet, chalky') → 'Not Sweet'。

Q 3.2chocolate DataFrame 
chocolate = choco.assign(
    Sweetness = choco.get('Characteristics').apply(label_sweet)
).get(['Rating', 'Sweetness'])

💡 链式：先 .assign 新增 Sweetness 列（在 choco 全部 9 列基础上），再 .get([...]) 只取需要的两列。不要先 .get(['Characteristics']) 再 assign，那样会丢 Rating。

✅ 期望：shape (2693, 2)，列 {'Sweet', 'Not Sweet'} 各 339 / 2354 行。
Q 3.3observed_difference 
sweet_mean     = chocolate[chocolate.get('Sweetness') == 'Sweet'    ].get('Rating').mean()
not_sweet_mean = chocolate[chocolate.get('Sweetness') == 'Not Sweet'].get('Rating').mean()
observed_difference = sweet_mean - not_sweet_mean
✅ 反推：autograder 验 int(obs × 10⁶ % 10⁴) == 8645 且 obs < 0 → observed_difference ≈ −0.171355（Sweet 比 Not Sweet 平均评分低 0.17）。
Q 3.4interpretation

6 个选项中选对的，看你能不能读 observed_difference ≈ −0.17 的实际含义。

interpretation = 6

💡 关键：单位是 rating points（评分点），不是 percent！1、2、4、5 全是 "percent" 错；3 方向反了（说 sweet 高，实际低）；只有 6 正确："sweet bars 比 non-sweet 低 0.16 个 rating points"。

Q 3.5Permutation simulation → differences 
differences = np.array([])
for i in np.arange(1000):
    shuffled_sweetness = np.random.permutation(chocolate.get('Sweetness'))
    shuffled = chocolate.assign(Sweetness = shuffled_sweetness)
    s_m  = shuffled[shuffled.get('Sweetness') == 'Sweet'    ].get('Rating').mean()
    ns_m = shuffled[shuffled.get('Sweetness') == 'Not Sweet'].get('Rating').mean()
    differences = np.append(differences, s_m - ns_m)

💡 为什么洗 Sweetness 就模拟了 H0：H0 说两组评分来自同分布——意味着"贴上 Sweet 还是 Not Sweet 的标签"对评分没影响。洗标签后，每次 shuffled 的 group 分配是随机的——这正是 H0 下"标签不重要"的可视化。

💡 注意：保持 shuffled 列长度 = 339 + 2354 不变。np.random.permutation 只重排顺序，不增不减，所以 Sweet 还是 339 个、Not Sweet 还是 2354 个——只是配对的 Rating 变了。

Q 3.6chocolate_p

"Whether to use <= or >=: think about whether larger values or smaller values of our test statistic favor the alternative hypothesis."

chocolate_p = np.count_nonzero(differences <= observed_difference) / 1000

💡 为什么 <=（左尾）：HA 说 Sweet 的 Rating 更低 → 差 (sweet − not sweet) 越负越极端。observed = −0.17。模拟中 ≤ −0.17 的比例 = p 值。

⚠️ 经典陷阱：把 <= 写成 >= → p 值会变成 1 − p。如果不确定方向，把 HA 抄一遍："Sweet 的 mean 更低" → 差更负 → 用 <=。
Q 3.7chocolate_conclusion & sweeter_is_worse

从 6 个选项里选所有真的（list 形式）+ 设 sweeter_is_worse 为 True/False。

chocolate_conclusion = [2, 5]
sweeter_is_worse     = True

💡 为什么 reject 两个 level 都过：观察差 −0.17，2693 行的样本足够大，置换分布的 SD 很小（绝大多数 shuffled 的 mean 差都接近 0）。p 值往往 ≈ 0 或非常小，所以 0.01 和 0.05 两个 cutoff 都 reject。
选项 1、4 错在 "accept"（永远不用！）；选项 3、6 错在 "fail to reject"。

Q 3.8如果洗 Rating 而不是 Sweetness？shuffled_rating 
shuffled_rating = 4

💡 Permutation test 的对称性：洗 A 列保 B 列 ⟺ 洗 B 列保 A 列。本质上都是"把 A、B 之间的配对打乱"。所以洗 Rating 列的结果跟洗 Sweetness分布一样，p 值一样，结论一样。这是 Quiz 4 常考的概念题。

💡 反驳其他选项：

  • 1（p 变成 1 − p）：错，是同一个 test。
  • 2（要改 null）：错，null 没变。
  • 3（要改 test stat）：错，统计量定义可以不变。
  • 5（Rating 是数字不能 shuffle）：错，np.random.permutation 可以洗任何 array。

Q 3.9why_shuffle 
why_shuffle = 3

💡 Shuffle 的根本目的："generate new data under the null hypothesis"。H0 说两组来自同分布——洗标签后的虚拟世界就是 H0 真的成立的世界。我们看实际 observed 在这个虚拟世界里有多奇怪。

💡 反驳：选项 1（mitigates noise）—— shuffle 增加 randomness 不是减；2（shuffling = bootstrapping）—— 不！shuffle 是不放回重排，bootstrap 是有放回采样；4（under the alternative）—— 完全反了，shuffle 是 under null。

📚 Section 4 — Battery Life（Q4.1–4.5）— CI 法做 hypothesis test

背景：80 条耳机电池数据（40 AirPods + 40 Galaxy Buds）。问：两个 brand 的平均电池寿命有没有差异？

📐 假设：H0: μGalaxy − μAirPods = 0 ｜ HA: ≠ 0（双侧！）
方法：不做 permutation 模拟。而是 bootstrap 估计差值的 95% CI，看 0 是否在 CI 内。
Q 4.1取两组 & 算 observed 差 
galaxy   = battery_life_data[battery_life_data.get('Brand') == 'Samsung Galaxy Buds Pro']
airpods  = battery_life_data[battery_life_data.get('Brand') == 'Apple AirPods']
galaxy_mean  = galaxy.get('BatteryLife').mean()
airpods_mean = airpods.get('BatteryLife').mean()
observed_diff_mean = galaxy_mean - airpods_mean
✅ autograder 反推：observed_diff_mean > 0 且 int(obs × 100 % 100) == 79 → Galaxy 比 AirPods 平均多 X.79 分钟。结合题干你看到的 4h41m vs 4h18m（差 23 分钟），所以大约 22–24 分钟。
Q 4.2Bootstrap → difference_means（1000 次）

"Generate your Galaxy Buds resamples by sampling from galaxy, and your AirPods resamples by sampling from airpods. Do not use the combined dataset."

difference_means = np.array([])
for i in np.arange(1000):
    g_sample = galaxy.sample(galaxy.shape[0],   replace=True)
    a_sample = airpods.sample(airpods.shape[0], replace=True)
    diff = g_sample.get('BatteryLife').mean() - a_sample.get('BatteryLife').mean()
    difference_means = np.append(difference_means, diff)

💡 为什么分开采样：每个 sample 必须保留原样本结构（40 Galaxy + 40 AirPods），否则 g_sample 可能采到 38 个 Galaxy + 2 个 AirPods，平均差就失真。要保证"两个 brand 的 sample 大小固定"，必须分别从 galaxy / airpods 子集 resample。

⚠️ Lab/HW 经典陷阱：从合并 battery_life_data 抽 80 个 → 每次抽到的两 brand 数量随机变化 → bootstrap 的 SE 不是"两 brand 各 40 时的"SE。题目加粗强调 "Do not use the combined dataset"。

💡 为什么这不是 permutation test：permutation 是 shuffle group labels（保留 80 个但乱配 brand）来模拟 H0；bootstrap 是从原 sample 有放回采样来估计真实差值的不确定度。这里走第二条路，最终用 CI 是否含 0 来判 H0。

Q 4.395% CI 
left_endpoint  = np.percentile(difference_means, 2.5)
right_endpoint = np.percentile(difference_means, 97.5)

💡 公式：c% CI 的左右 percentile = (100−c)/2 和 (100+c)/2。95% → 2.5 / 97.5。90% → 5 / 95。这是必背 8 件套之一（见 Quiz 3 笔记）。

Q 4.4CI 法判 H0 → reject_null 
reject_null = True

💡 CI 法核心规则：双侧 H0: μ = c。如果 c 不在 95% CI 内 → 在 0.05 水平 reject H0；如果 c 在 CI 内 → fail to reject。这里 c = 0。Galaxy − AirPods 平均 ≈ +23 分钟，bootstrap 几乎所有 sample 都正，CI 大概是 [+15, +30] 之类，0 不在 CI 内 → reject。

见 quiz4/05-ci-as-hypothesis-test §concepts 的对称关系："(100−p)% CI 配 p% 显著性"。95% CI ↔ 0.05 significance。
Q 4.5单位换算 → same_conclusion

"What if all the people who responded had provided their battery lives in hours instead of minutes?"

same_conclusion = True

💡 为什么不变：所有数据 × 1/60 → 所有 mean × 1/60 → CI 端点 × 1/60。CI 整体被缩放，但0 的位置不变（0 × 1/60 = 0）。所以"0 在 CI 内吗？"这个判断不变，结论不变。

💡 推广：任何 线性 单位换算（cm↔m、°C↔K、min↔hour）都不影响 hypothesis test 结论。这跟"线性变换不改变 r、不改变 p 值"是同一类性质。

⚠️ 本作业典型陷阱（横切多题）

陷阱 1：用语红线 —— 永远不要写 "We accept"
Quiz 4 / Final 高频送分题：H0 永远不被 "accept"，只能 "reject" 或 "fail to reject"。HW 6 的 Q1.5、Q2.5、Q3.7 选项里全埋了 "accept" 陷阱。
陷阱 2：单侧 vs 双侧 / 用哪个尾巴算 p 值
  • Q1.4（Smiski）HA: p > 1/144 → 右尾 >=。
  • Q2.4（Pokémon TVD）TVD 永远右尾 >=（TVD 已经是非负的"偏离度"）。
  • Q3.6（Chocolate）HA: Sweet 更低 → 差更负 → 左尾 <=。
  • Q4.4（Battery）双侧 HA: ≠ 0 → CI 法判 0 是否在 CI 内。
记忆法：把统计量看成"越远离 H0 越极端"——找出 HA 方向，写到 HA 那边的尾巴去。
陷阱 3：TVD 必须 ÷ 2
Q2.2 公式 = Σ|p − q| / 2。少 ÷ 2 → TVD = 0.094 而不是 0.047。autograder 会用 int(tvd × 10⁷ % 8³) == 496 检测到。
陷阱 4：Permutation 是不放回重排，Bootstrap 是有放回采样
Q3.5 用 np.random.permutation（每个元素出现 恰好 1 次，只是顺序变）；Q4.2 用 .sample(n, replace=True)（有的多次有的不出现）。Q3.9 选项 2 故意混淆这两者，错。
陷阱 5：Bootstrap 两样本要分开采
Q4.2 强调"don't use combined dataset"。两组各 40 必须保留——分别 galaxy.sample(40, replace=True) 和 airpods.sample(40, replace=True)。
陷阱 6：CI 法的对应关系
95% CI ↔ 0.05 significance（双侧）；99% CI ↔ 0.01。如果题目说 "0.05 significance"，你需要的是 95% CI，不是 99%。
陷阱 7：cutoff 是题目说的那个
Q1.5 cutoff = 0.05；Q2.5 cutoff = 0.01。题目从不同 cutoff 切，结论可能完全相反。看清题目。
陷阱 8：interpretation 单位—— rating point 不是 percent
Q3.4 是干扰项最大的题。差值 −0.171 意思是"评分点低 0.17"，不是"百分比"。选项 1、2、4、5 全是 percent，全错。

🧰 四类检验代码模板速查

模板 A — 已知 p₀ 的单侧 proportion 检验（Smiski 型）

def one_simulation():
    return np.random.multinomial(n, [p0, 1-p0])[0] / n

stats = np.array([one_simulation() - p0 for _ in np.arange(reps)])
# 或 for-loop + np.append
p_value = np.count_nonzero(stats >= observed_diff) / reps   # H_A: p > p0

模板 B — Categorical TVD 拟合度检验（Pokémon 型）

def tvd(a, b): return np.sum(np.abs(a - b)) / 2

simulated_tvds = np.array([])
for _ in np.arange(reps):
    sim = np.random.multinomial(n, proposed_dist) / n
    simulated_tvds = np.append(simulated_tvds, tvd(proposed_dist, sim))

p_value = np.count_nonzero(simulated_tvds >= observed_tvd) / reps   # TVD 永远右尾

模板 C — Permutation 两均值检验（Chocolate 型）

diffs = np.array([])
for _ in np.arange(reps):
    shuf = np.random.permutation(df.get('group'))
    tmp  = df.assign(group = shuf)
    mA = tmp[tmp.get('group') == 'A'].get('value').mean()
    mB = tmp[tmp.get('group') == 'B'].get('value').mean()
    diffs = np.append(diffs, mA - mB)

# H_A: A < B → 左尾；H_A: A > B → 右尾；H_A: A ≠ B → 双侧（用 |·|）
p_value = np.count_nonzero(diffs <= observed_diff) / reps

模板 D — Bootstrap CI 法（Battery 型）

diffs = np.array([])
for _ in np.arange(reps):
    sA = groupA.sample(groupA.shape[0], replace=True)
    sB = groupB.sample(groupB.shape[0], replace=True)
    diffs = np.append(diffs, sA.get('val').mean() - sB.get('val').mean())

left, right = np.percentile(diffs, [2.5, 97.5])   # 95% CI
reject_null = not (left <= 0 <= right)            # 0 在 CI 内 → fail to reject
