📚 Part 1 — Comparing Grades Using Standard Units

背景：Kathleen 在 BILD 1 拿了 87，Eric 在 MATH 18 拿了 92。两个班的难度不同（hist 显示 BILD 中位 ≈ 65，MATH 中位 ≈ 90）。用 SU 比较"相对"成绩。

Q 1.1compute_variance(data)（手动，不用 np.std） 
def compute_variance(data):
    data = np.array(data)
    n = len(data)
    average = data.mean()           # 也可以 np.mean(data)
    sum_squared_diff = ((data - average) ** 2).sum()
    return sum_squared_diff / n

bild_midterm_var = compute_variance(bild_midterm.get('Score'))
math_midterm_var = compute_variance(math_midterm.get('Score'))

💡 variance 公式：variance = mean((x − mean)²)。注意分母是 n（population），不是 n−1（sample，Bessel）。DSC 10 全用 population SD。

💡 broadcasting：(data - average) 自动让每个元素减 mean。** 2 逐元素平方。.sum() 求和。整个表达式向量化、无 for loop。

✅ 期望：bild_midterm_var ≈ 173.205，math_midterm_var ≈ 84.266。验证 compute_variance([1, 2, 3, 4, 5]) == 2.0 ✓
Q 1.2compute_sd(data)—— 一行

Hint: "Your implementation of compute_sd should only take one line, that involves both the return keyword and the function compute_variance."

def compute_sd(data):
    return np.sqrt(compute_variance(data))

💡 SD = √variance。复用 Q1.1，简洁。这也是 lab 想让你练的"function composition"。

✅ 期望：BILD SD ≈ 13.16，MATH SD ≈ 9.18。注意 BILD 数据散得更开（histogram 宽），SD 更大。
Q 1.3compute_su(score, avg, sd)

"Warning: Be careful with the order of operations!"

def compute_su(score, avg, sd):
    return (score - avg) / sd

kathleen_su = compute_su(kathleen_score, bild_midterm.get('Score').mean(), compute_sd(bild_midterm.get('Score')))
eric_su     = compute_su(eric_score,     math_midterm.get('Score').mean(), compute_sd(math_midterm.get('Score')))

💡 注意减法的顺序：(score − avg) / sd，不是 (avg − score) / sd。warning 提示就是怕你写反。SU 是"score 比 mean 高几个 SD"。

✅ 期望：kathleen_su ≈ 1.668（比班级均高 1.67 个 SD），eric_su ≈ 0.383（比班级均高 0.38 个 SD）。
→ Kathleen 在自己班相对表现远好于 Eric，即使绝对分数低 5 分。这就是 SU 的力量：把不同尺度的成绩放到同一坐标系。
Q 1.4Kathleen / Eric 的 proportion

"For both Kathleen and Eric individually, calculate the proportion of students they scored higher than (or the same as)."

kathleen_proportion = (bild_midterm.get('Score') <= kathleen_score).mean()
eric_proportion     = (math_midterm.get('Score') <= eric_score    ).mean()

💡 经典 babypandas 套路：(series <= val).mean() = bool Series 的均值 = True 的比例 = "≤ val 的占比"。
也可以写成 np.count_nonzero(series <= val) / series.shape[0]，等价但啰嗦。

✅ 期望：kathleen_proportion == 0.954，eric_proportion == 0.582。Kathleen 比 95.4% 同学高分，Eric 只比 58.2% 高。结论与 SU 一致：Kathleen 相对表现更好。

📚 Part 2 — Chebyshev's Inequality & Normal Curves

Q 2.1哪个直方图大致正态？

BILD 1（钟形对称）vs MATH 18（左偏 / 右尾被截）。Normal 必备特征：对称 + 单峰 + 钟形。

q2_1 = 1   # 只有 BILD 1 大致 normal

💡 MATH 18 不 normal：histogram 显示分数集中在 80–100，不对称（左侧拖尾、右侧被 100 上限截断）。Normal 要求两侧对称延伸。

Q 2.2MATH 18：mean / median / mode 排序（最小→最大）

"Don't compute any of these values manually!" — 看直方图判断。

q2_2 = [1, 2, 3]   # mean < median < mode

💡 左偏分布：MATH 18 的 histogram 是左偏（long left tail）。规则：左偏 → mean < median < mode（mean 被左尾拉低）。右偏反过来：mean > median > mode。
记忆口诀："mean 跟着尾巴走"——尾巴在哪边，mean 就在哪边的极端。

Q 2.3无假设（Chebyshev）：top 5% 要 ≥ 几个 SD？

4 选项：1=4.5 SDs（用 z=√20）；2=2 SDs；3=略小于 2 SDs；4=无法保证。

q2_3 = 1   # 4.5 SDs (z = √20)

💡 推导：要在任何分布下保证 top 5%。Chebyshev：1 − 1/k² ≥ 0.95 → k² ≥ 20 → k ≥ √20 ≈ 4.47 ≈ 4.5 SDs。
这告诉你：在 mean ± 4.5·SD 内至少 95% 的数据。如果 Kathleen 得分高于 mean + 4.5·SD，她必然在 top 5%（实际：top 2.5% 至少）。但 4.5 SDs 是个非常高的门槛——很难达到。

💡 为什么不是 2 SDs（选项 2）：2 SDs 只保证 75% 在 ±2·SD 内（Chebyshev: 1 − 1/4 = 75%），不到 95%。所以高于 mean + 2·SD 不一定保证 top 5%——可能两边各有 12.5% 还在 ±2·SD 外。

Q 2.4假设 normal 分布：top 5% 要 ≥ 几个 SD？

同 4 选项。这次假设正态，结果应该不同。

q2_4 = 3   # 略小于 2 SDs

💡 推导：normal 下 95% 数据在 mean ± 2·SD 之间，所以每侧各 2.5%。高于 mean + 2·SD 的只有 2.5%（< 5%）。所以"略小于 2 SDs"就够进 top 5%（具体是 ≈ 1.645 SDs，norm.cdf(1.645) ≈ 0.95）。

💡 选项 3 的推理："50% 在 mean 以下，2 SDs 以上是 2.5% → mean + 2·SD 把你拍进 top 2.5%（自然 ⊂ top 5%）"。所以略小于 2 SDs 就能进 top 5%。
对比 Chebyshev 的 4.5 SDs：相同的问题，假设 normal 比"任何分布"宽松很多——只需 2 SDs 而不是 4.5。这就是为什么正态假设非常有用（也是为什么 CLT 重要）。

⚠️ 经典对比：Q2.3 用 Chebyshev 得 z = √20 ≈ 4.47；Q2.4 用 Normal 得 z ≈ 1.645。同一个 top-5% 问题，假设不同 → 阈值差几倍。考试会考这个对比，要能讲清"Chebyshev 是 lower bound (for any distribution)，Normal 是 exact (for normal only)"。

📚 Part 3 — Choosing Sample Size

背景：UCSD 一门新课，900 个学生符合先修要求，但不知道实际有多少会选。给定的是整个 population 数据（0 = 不上、1 = 上），让你模拟抽样过程。最终目标：95% CI 宽度 ≤ 10 个学生。

💡 proportion 等于 mean of 0/1 列：再次出现的 "proportion as mean" trick。Q3.2 会用图确认这个 sample 比例分布近似 normal（CLT）。

Q 3.2sample proportion 像 normal 还是像原 population？ 
q3_2 = 1   # 更像 normal

💡 CLT 直接结论：原 population 是 0/1（两个柱），但 sample mean (= proportion) 的分布近似 normal——这正是 CLT 的核心声明。

Q 3.3把宽度 10 个学生 → proportion 
num_eligible_students = 900
width_as_proportion = 10 / num_eligible_students   # ≈ 0.01111

💡 转换：我们要的统计量是 proportion（0–1 之间），CI 宽度也要换成 proportion 单位。10 / 900 ≈ 0.01111。

Q 3.4需要的 SD = ? → target_sd

"95% of the data lies between plus and minus 2 SDs of the mean."

target_sd = width_as_proportion / 4   # ≈ 0.002778

💡 关键推导：95% CI = mean ± 2·SD → CI 宽度 = 4·SD。所以 SD = width / 4。这是 Quiz 3 必背的"width 公式"。

💡 不是 ÷ 2：常见错答。CI 宽度是总长（右端 − 左端），不是半径。半径才是 2·SD；总长是 4·SD。

Q 3.5population_sd（用 Part 1 的 compute_sd） 
population_sd = compute_sd(population.get('Planning on taking'))
# ≈ 0.4583

💡 总体是 0/1 数据，900 个学生中 ≈ 70% 会上（看 histogram）。所以 SD ≈ √(0.7 × 0.3) ≈ 0.458。compute_sd 算出来就是这个数。

💡 babypandas 限制提醒：题目要求用你自己写的 compute_sd（Part 1），不允许直接 np.std。但实际上 np.std 也行（数值一样）——题目只是想让你 reuse 自己的函数。

Q 3.6req_sample_size—— 终极样本量公式 
req_sample_size = (population_sd / target_sd) ** 2
# = (0.4583 / 0.002778)² ≈ (165)² ≈ 27,216

💡 推导：从 CLT SD_sample_mean = pop_SD / √n，求 n：n = (pop_SD / target_sd)²。
这就是 "Quiz 3 必背公式" n ≥ (4·SD/w)² 的等价形式——把 target_sd = w/4 代入即得。

✅ 期望：req_sample_size ≈ 27216（autograder 验 np.isclose(req_sample_size, 27216)）。
Q 3.74 个 T/F 陈述

req_sample_size = 27216 但 population = 900。这是不可能的（无放回抽样不能取多于 population）。怎么办？

statement_1 = True    # 退而求其次：放宽 CI 宽度
statement_2 = False   # "Sampling with replacement" 不能解决（采样独立性问题不同）
statement_3 = True    # 退而求其次：降低 confidence 水平（如 80% 而不是 95%）
statement_4 = False   # 不能"扩大 population"——总体就是 900 个

💡 statement_1 True：宽度变宽 → 需 SD 上限放宽 → 需 n 变小。可行。

💡 statement_2 False：有放回采样跟无放回在大样本时差别不大，但不能突破"独立信息量"上限——900 个学生最多 900 份独立信息。重复采样得到的额外样本是冗余的，不真正减小不确定度。

💡 statement_3 True：降低 confidence（如从 95% 到 68%）→ SD 系数从 2 变 1 → 同样 SD 给出更窄 CI → 需要的 n 缩小。可行（但 confidence 没那么高）。

💡 statement_4 False：population 是客观存在（900 个符合 prerequisites 的学生），不是你能"做大"的。荒谬。

⚠️ 真实 takeaway：当 req_sample_size > population，意味着原 95% / width 10 这个组合本质不可能。必须放弃一个目标（confidence 或 precision），或者承认无法保证。

⚠️ 本作业典型陷阱

陷阱 1：variance 分母是 n，不是 n − 1
DSC 10 永远用 population variance（÷ n），与 numpy 的默认 np.std 一致。不是 sample variance（÷ n−1）。如果你写了 / (n - 1)，所有结果会偏大。
陷阱 2：Chebyshev vs Normal 用错
Q2.3（"无假设"）必须用 Chebyshev → 4.5 SDs。Q2.4（"normal"）用 68-95-99.7 → 略 < 2 SDs。如果反了，差几倍。
陷阱 3：CI 宽度 = 4·SD 不是 2·SD
Q3.4 关键公式：target_sd = width / 4。宽度是左右两端之差，半径才是 2·SD。把 /4 写成 /2 → target_sd 大一倍 → req_sample_size 小 4 倍 → 答案完全错。
陷阱 4：左偏 / 右偏 mean-median-mode 顺序
Q2.2：左偏（tail 在左）→ mean < median < mode；右偏（tail 在右）→ mean > median > mode。口诀："mean 跟着尾巴走"——尾在哪 mean 偏到哪。
陷阱 5：Q3.7 误以为"有放回"能解决
Statement 2 是干扰项。直觉是"sample with replacement 可以抽无数次"——但额外抽到的是重复信息，不增加独立观测数。population 数量是真正的"信息上限"。
