📋 项目概览

数据：Spotify Weekly Charts，每行 = 某周某地区某 track 的排名 + 14 个音频特征 + 元信息。规模数十万行。

逻辑链：

  1. Sec 1–2：把"原始 charts"压缩成"唯一歌曲"（按 track + artist groupby + median）。处理重复 uri、多 artist 的歌等等。
  2. Sec 3：tempo 分类 + 各类音频特征均值——综合 groupby([a,b])、merge、自定义排序。
  3. Sec 4：字符串过滤 + 把"含某词 vs 不含"对比抽象成 word_analysis(word) 函数。
  4. Sec 5：处理 release_date 的三种格式（YYYY / YYYY-MM / YYYY-MM-DD），算 weeks_between。
  5. Sec 6：megahits 判定——top 1/2 + 长留榜时间 + 长 top 10 连击。考累加器模式找最长 streak。
  6. Sec 7：把 megahits 的音频特征 vs 全部歌的均值对比，画水平条形图。

Tier 速览（高频考点 + babypandas 限制）：

  • ⭐⭐⭐ babypandas 没有 .rename() ——用 .assign(新=旧).drop(columns=[旧])。
  • ⭐⭐⭐ babypandas 没有 .reindex() ——用辅助 _order 列 + sort + drop。
  • ⭐⭐⭐ babypandas .drop() 只能 columns=，不能按 index 删行——用布尔过滤代替。
  • ⭐⭐⭐ groupby().median()/mean()/min() 不能处理非数值列——必须先 get([...])。
  • ⭐⭐⭐ 多列 groupby：groupby(['a', 'b']) 在 Sec 1.4、1.7、1.10 反复出现。
  • ⭐⭐⭐ 累加器找最长 streak（Q6.8）—— for + current/longest 双变量。
  • ⭐⭐ .str.contains('word', regex=False)（Q4.6 强调 regex=False）。
  • ⭐⭐ merge 不传 on 自动按所有共有列 join（Q3.5）。

def tempo_to_name(t):
    if   t < 60:  return 'Lento'
    elif t < 90:  return 'Adagio'
    elif t < 110: return 'Andante'

Q 4.6word_analysis(word, draw_plot=False)：抽象成函数 
def word_analysis(word, draw_plot=False):
    feats = ['danceability', 'energy', 'speechiness', 'acousticness',
             'instrumentalness', 'liveness', 'valence']
    has_word = songs.get('track_name').str.lower().str.contains(word.lower(), regex=False)\
        .apply(lambda b: 'True' if b else 'False')
    means = songs.assign(has_word=has_word).get(feats + ['has_word']).groupby('has_word').mean().T
    means = means.assign(AbsDiff=(means.get('False') - means.get('True')).abs())
    if draw_plot:
        title = "What's So Special About " + word[0].upper() + word[1:].lower() + "?"
        means.sort_values(by='AbsDiff').plot(kind='barh', y=['False', 'True'], title=title)
    return means

💡 regex=False 必须：题目强调。否则 .、* 等会被当 regex 解析。

💡 标题首字母大写：word[0].upper() + word[1:].lower()。考字符串切片 + 拼接。

Q 5.4weeks_between：跨年逻辑 
def weeks_between(date1, date2):
    year1, month1, day1 = date1[0], date1[1], date1[2]
    year2, month2, day2 = date2[0], date2[1], date2[2]
    total_days = 0
    if year1 == year2:
        total_days = days_between(month1, day1, month2, day2)
    else:
        total_days = (year2 - year1 - 1) * 365
        total_days = total_days + days_between(month1, day1, 12, 31)
        total_days = total_days + days_between(1, 1, month2, day2)
        total_days = total_days + 1
    return int(total_days / 7)

💡 跨年分三段：① 当年从 (m1, d1) 到 12/31；② 中间整年（year2−year1−1 年）；③ 第 year2 年从 1/1 到 (m2, d2)；最后 +1（衔接的那天）。除以 7 取整 = 整周数。

Q 6.7calculate_rank_array(track)：长度 num_weeks 的 rank 数组 
def calculate_rank_array(track_name):
    song_rows = possibly_mega[possibly_mega.get('track_name') == track_name]
    weeks_in = np.array(song_rows.get('week_num'))
    ranks_in = np.array(song_rows.get('rank'))
    rank_array = np.array([])
    for w in np.arange(1, num_weeks + 1):
        if w in weeks_in:
            rank_array = np.append(rank_array, ranks_in[weeks_in == w][0])
        else:
            rank_array = np.append(rank_array, 201)
    return rank_array

💡 201 是 sentinel："不在榜"用一个不可能值（> 200）代替。后续判断 "rank ≤ 10" 时它永远 False。

Q 6.8longest_streak(track, n=10)：累加器找最长连击 
def longest_streak(track_name, n=10):
    rank_array = calculate_rank_array(track_name)
    longest = 0
    current = 0
    for num in rank_array:
        if num <= n:
            current = current + 1
            longest = max(longest, current)
        else:
            current = 0
    return max(longest, current)

💡 累加器双变量模板：

  • current：当前连击长度
  • longest：迄今最长
  • 满足条件 → current++ 并更新 longest；不满足 → current 归零
期末非常高频。注意 return 用 max(longest, current) 是冗余的（longest 已经 always >= current），但更安全。

Q 6.9加 longest_streak_top_ten 列（用 dict 缓存） 
unique_tracks = np.unique(possibly_mega.get('track_name'))
streak_map = {t: longest_streak(t, 10) for t in unique_tracks}
possibly_mega = possibly_mega.assign(
    longest_streak_top_ten=possibly_mega.get('track_name').apply(lambda t: streak_map[t])
)

💡 性能优化：如果直接 .apply(longest_streak)，会对每行（多行同 track）重复计算同样的 streak。先建 dict 缓存 → 每个 track 只算一次。这是真实场景的必备优化。

⚠️ 典型陷阱

陷阱 1：autograder 类型检查 (int / float)
所有 "shape[0]"、"count"、"average" 都要 int(...) 或 float(...) 包，否则 numpy int64/float64 通不过 isinstance(..., numbers.Integral)。Q1.1, 1.9, 2.2, 6.2, 6.4 等都涉及。
陷阱 2：has_love 是字符串 'True'/'False' 不是 bool
Q4.1。后续 Q4.2、4.3 用 == 'True' 过滤。如果存的是 bool，需要 == True，两者不同。
陷阱 3：.str.contains 必须 regex=False
Q4.1, 4.6 强调。否则 .、* 等正则元字符会被误解。
陷阱 4：merge 不传 on 自动联合多列
Q3.5。共有 'track_name' 和 'artist_names'，automerge 用两列联合。如果你以为只用 track_name，行数会膨胀。
陷阱 5：累加器 streak 模板（Q6.8）
current / longest 双变量。满足条件 → current++，更新 longest；不满足 → current = 0。条件循环结束后 longest 就是答案。
陷阱 6：性能——把 .apply 的重复计算缓存到 dict
Q6.9。对每行同 track 直接 .apply(longest_streak) 会重复计算。先建 dict 一次性算完，apply 改成查 dict。期末 / 真实场景必备技巧。
