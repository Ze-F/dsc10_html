📚 Section 1 — Lucky Triton Lotto, Continued

背景：彩票规则——5 个 1–29 之间的数 + 1 个 1–8 的 "Tritonball"。中奖号码已知。每张 $5。本节模拟买 10 万张票看实际净收益。

Q 1.1simulate_one_ticket() 
def simulate_one_ticket():
    first_five = np.random.choice(np.arange(1, 30), 5, replace=False)
    tritonball = np.random.choice(np.arange(1, 9))
    return np.append(first_five, tritonball)

💡 注意 replace=False：彩票前 5 个数不能重复（同一票里 5 个 distinct 数字）。Tritonball 单抽一个，没"重复"问题，所以不写 replace。

💡 np.arange(1, 30) = [1..29]，np.arange(1, 9) = [1..8]。注意右端开区间。

Q 1.2num_matches（一行） 
num_matches = np.count_nonzero(winning == simulated_ticket)

💡 position-wise 比较：winning == ticket 返回 boolean array（逐位比较）。count_nonzero 数 True 数量。注意：这不是 set 匹配。如果票上有 15 但位置错了，不算 match。这是题目设定（与现实彩票不同）。

Q 1.3count_free_housing：模拟 100,000 票中头奖次数 
winning = np.array([15, 9, 24, 23, 1, 3])
count_free_housing = 0
for i in np.arange(100000):
    ticket = simulate_one_ticket()
    if np.count_nonzero(winning == ticket) == 6:
        count_free_housing = count_free_housing + 1

💡 理论概率超低：6 位全中的概率 ≈ 1/(C(29,5)×8×5!/5!) = 1/(118755 × 8) ≈ 1/950040 ≈ 1.05×10⁻⁶（不计排列）；位置 sensitive 后还要乘 1/120 → ≈ 8.77×10⁻⁹。100,000 票 → 期望中奖 0.000877 次 → 几乎一定 = 0。这正是彩票的特点：理论上可能，实际上别想。

Q 1.4most_matches：100,000 票中最多匹配几位 
most_matches = 0
for i in np.arange(100000):
    ticket = simulate_one_ticket()
    matches = np.count_nonzero(winning == ticket)
    if matches > most_matches:
        most_matches = matches

💡 max 模板：维护 running max。each iteration 比较，更大就更新。每次随机抽，最大值通常 ≤ 3 或 4（更高极罕见）。

Q 1.5net_winnings（关键：必为负！） 
prizes = np.array([0, 10, 25, 100, 1000, 5000, 20000])
net_winnings = -5 * 100000   # 起步：买 100,000 票 = -$500,000
for i in np.arange(100000):
    ticket = simulate_one_ticket()
    matches = np.count_nonzero(winning == ticket)
    net_winnings = net_winnings + prizes[matches]

💡 用 array 索引代替 if/elif：prizes[matches]——matches 是 0/1/.../6，自动取对应奖金。比 6 个 elif 干净。

💡 期望必为负：彩票运营方期望利润 > 0，所以购票人期望必为负。autograder 测 net_winnings < 0。每跑一次值不同（随机），但永远负。

📚 Section 2 — Sampling with NBA Data

背景：NBA 球员数据（含 Points、Salary 等）。题目提供 compute_statistics 函数算样本均值。本节练 convenience vs SRS 的区别。

Q 2.1convenience_sample：NYK / BRK / PHI 三队 
convenience_sample = nba_data[
    (nba_data.get('Team') == 'NYK') |
    (nba_data.get('Team') == 'BRK') |
    (nba_data.get('Team') == 'PHI')
]

💡 | 不是 or：Series 之间逻辑或必须用 |（按位）。or 会触发 ValueError。每段条件外加括号——优先级问题。

Q 2.2convenience_stats 
convenience_stats = compute_statistics(convenience_sample)

💡 直接调用。compute_statistics 是题目预定义的，返回 [mean_points, mean_salary]。convenience sample 这三个东海岸球队的均值可能跟整体差很多——这是 Q2.3 的引子。

Q 2.3sampling_q3：convenience 是否代表 population？ 
sampling_q3 = 2   # 否

💡 为什么 convenience 不行：convenience sample 是非随机挑选（如"挑离你近的"或"挑这三队"）。系统性偏差——这三队的薪资分布、得分文化可能系统不同于其他队，所以即使样本多也无法代表 population。
关键：sample 是否能 estimate population 不取决于大小，而取决于random & representative。

💡 对比 SRS：SRS 每个个体被选概率相同——即使 sample 小（30 个），也无偏 on average。大样本 SRS 既无偏又方差小。convenience 永远有偏。

Q 2.4my_small_stats（SRS 70）+ sampling_q4 
my_small_stats = compute_statistics(nba_data.sample(70), draw=False)

sampling_q4 = 3

💡 .sample(70) 默认无放回。每次跑得到不同结果（randomness），但都接近 population mean。sampling_q4 = 3 是"close but every time slightly different"。

Q 2.5my_large_stats（SRS 180）+ sampling_q5 
my_large_stats = compute_statistics(nba_data.sample(180), draw=False)

sampling_q5 = 1   # 更大 sample → 更小 variance

💡 CLT 暗示：SD_sample_mean = pop_SD / √n。n 从 70 → 180 → SD 减小到 √(70/180) ≈ 0.62 倍。所以大样本的 estimate 更紧贴 population mean，且 estimate 之间 variation 也更小。

📚 Section 3 — Milk Tea, Yippee!

背景：survey 问大家最喜欢哪种 tea（jasmine / black / green / oolong）。原数据 ≈ 200 行。本节用 bootstrap 估计真比例。

Q 3.1jasmine_proportion 
jasmine_proportion = np.count_nonzero(survey.get('tea') == 'jasmine') / survey.shape[0]

💡 套路：count_nonzero(boolean Series) = True 数量，除以总数 = 比例。也可以 (survey.get('tea') == 'jasmine').mean() —— 等价。

Q 3.2boot_jasmine_proportions：1000 次 bootstrap 
boot_jasmine_proportions = np.array([])
for i in np.arange(1000):
    resample = survey.sample(survey.shape[0], replace=True)
    p = np.count_nonzero(resample.get('tea') == 'jasmine') / resample.shape[0]
    boot_jasmine_proportions = np.append(boot_jasmine_proportions, p)
⚠️ autograder 提示："all of your resamples have the same mean — take a closer look at how you're resampling" —— 漏写 replace=True 是最常见错误。详见 Lab 4 Q5。
Q 3.395% CI 
jasmine_lower_bound = np.percentile(boot_jasmine_proportions, 2.5)
jasmine_upper_bound = np.percentile(boot_jasmine_proportions, 97.5)
Q 3.4⭐ CI 解读陷阱 1："95% of people fall in CI"？ 
q3_4 = False

💡 错答的本质："95% 的人喜欢度在 CI 内" —— 把 CI 当成个体分布的范围。
CI 是关于 parameter（population proportion，一个数）的估计区间。它不描述"个体的喜好值"——每个个体的"喜好"是 {0, 1}（喜欢/不喜欢 jasmine），跟 CI 的小数没关系。

Q 3.5⭐ CI 解读陷阱 2："95% probability truth is in CI"？ 
q3_5 = False

💡 必背：truth（真比例）是 fixed，CI 也是 computed（从你的 sample 算出来后 fixed）。两个 fixed 的对象，要么含、要么不含，没"95% 概率"。
正确版：如果重复这个过程（每次抽 sample → bootstrap → CI）很多次，约 95% 的 CIs 会包含 truth。随机的是过程，不是 truth。

Q 3.6how_many：2000 个独立 95% CI 中含 truth 的数量？ 
how_many = 1900   # = 2000 × 0.95

💡 这是 95% CI 正确解读的直接计算：每个 CI 独立 95% 含真值 → 期望 2000 × 0.95 = 1900。本题正面考你 Q3.5 的概念。

Q 3.7ci_90 / ci_96 / ci_99：CI 宽度 vs confidence

三个 CI 给出：[0.195, 0.30] / [0.185, 0.31] / [0.175, 0.325]。哪个对应 90% / 96% / 99%？

ci_90 = 2   # 最窄 (0.105)
ci_96 = 1   # 中等 (0.125)
ci_99 = 3   # 最宽 (0.150)

💡 必背规律：confidence level 越高 → CI 越宽。直觉：要"更自信"地包含 truth，自然 net 撒得更大。99% > 96% > 90% → 对应 CI 宽度依次递减。

💡 数学：c% CI 的 percentile = (100−c)/2 到 (100+c)/2。c 越大，区间越靠两端。bootstrap 分布偏中间集中，所以两端值更极端 → CI 更宽。

Q 3.8differences_in_resamples()：jasmine − black 
def differences_in_resamples():
    differences = np.array([])
    for i in np.arange(1000):
        resample = survey.sample(survey.shape[0], replace=True)
        j = np.count_nonzero(resample.get('tea') == 'jasmine') / resample.shape[0]
        b = np.count_nonzero(resample.get('tea') == 'black')   / resample.shape[0]
        differences = np.append(differences, j - b)
    return differences

boot_differences = differences_in_resamples()

💡 一次 resample 两个比例：从同一个 resample 算 jasmine_prop 和 black_prop，相减。不要分别 resample 然后相减——那会破坏 jasmine 和 black 之间的相关性。

Q 3.9difference 的 95% CI 
diff_lower_bound = np.percentile(boot_differences, 2.5)
diff_upper_bound = np.percentile(boot_differences, 97.5)
Q 3.10⭐ q3_10：difference CI 比 jasmine CI 更宽还是更窄？ 
q3_10 = 2   # 更窄的是 jasmine（即 difference 更宽）

💡 直觉：估计两个未知参数之差 > 估计单一未知参数。jasmine 比例是单一未知 → 不确定度 only from jasmine sampling。difference 涉及 jasmine 和 black 两个 sampling 不确定度 → 总不确定度 = √(var_j + var_b)（假设独立）> var_j。

💡 实际数：jasmine CI 宽度 ≈ 0.1，difference CI 宽度 ≈ 0.15。差距明显。这正是 Q3.7 同一思想（越多 random source → 越宽 CI）的应用。

📚 Section 4 — Need a Lyft?

背景：rideshare DataFrame 含 Uber 和 Lyft 价格 + 文本列（type, source 等）。问：哪家平均更便宜？bootstrap 各自的 99% CI。

Q 4.1uber_lyft_means（先 get([...]) 再 groupby） 
uber_lyft_means = (
    rideshare.get(['app', 'price'])
             .groupby('app')
             .mean()
             .sort_values(by='price')
)
⚠️ 关键 babypandas 陷阱：直接 rideshare.groupby('app').mean() 会报错——因为 rideshare 含字符串列（type、source 等）。pandas 2.x 不再自动忽略非数值列，babypandas 又不支持 numeric_only=True。必须先 .get(['app', 'price']) 去掉非数值列。

💡 sort 后 index 顺序：升序后 Uber 在第一行（更便宜）。autograder 验 index = ['Uber', 'Lyft']。

Q 4.2Uber 99% CI

Hint："Make sure to query before resampling!"

uber_only = rideshare[rideshare.get('app') == 'Uber']   # 先 query

uber_means = np.array([])
for i in np.arange(1000):
    resample = uber_only.sample(uber_only.shape[0], replace=True)   # 再 bootstrap
    uber_means = np.append(uber_means, resample.get('price').mean())

lower_bound = np.percentile(uber_means, 0.5)
upper_bound = np.percentile(uber_means, 99.5)

💡 必须先 query 后 bootstrap：如果先 bootstrap 整个 rideshare 再 query Uber，每次 resample 中 Uber 行数变化——你想要的是"Uber 内部"重抽，不是"混着 Uber+Lyft 抽再筛"。

💡 99% CI percentile = 0.5 / 99.5：套公式 (100−99)/2 = 0.5 和 (100+99)/2 = 99.5。

Q 4.3app_and_hist(app_name)：通用函数 
def app_and_hist(app_name):
    app_data = rideshare[rideshare.get('app') == app_name]
    means = np.array([])
    for i in np.arange(1000):
        resample = app_data.sample(app_data.shape[0], replace=True)
        means = np.append(means, resample.get('price').mean())
    lower = np.percentile(means, 0.5)
    upper = np.percentile(means, 99.5)
    # ... plot ...
    return f'A 99% confidence interval for the average {app_name} ride price is [{lower}, {upper}].'
⚠️ 必须 return，不是 print：题目要求函数返回字符串（外部可以用 uber_string = app_and_hist('Uber')）。如果 print + 无 return，uber_string 会是 None。autograder 提示 "Make sure your function returns a string, and doesn't just print"。

⚠️ 本作业典型陷阱

陷阱 1：CI 解读 4 道连击（Q3.4/3.5/3.6/3.7）
① CI 不是"95% 个体落在 CI"；② CI 不是"truth 有 95% 概率在 CI"；③ 重复多次的话约 95% CI 含 truth；④ 高 confidence → 宽 CI。每一条都是 Quiz 3 / Quiz 4 / Final 高频考点。
陷阱 2：bootstrap 必须 replace=True
Q3.2 / Q4.2 / Q4.3。漏写 → resample 都跟原样本一样 → estimate 都相同。
陷阱 3：先 query 后 bootstrap（不能反）
Q4.2：要算 Uber CI 就先 rideshare[app == 'Uber']，再对子表 bootstrap。先 bootstrap 整个 rideshare 再查 Uber 是错的。
陷阱 4：babypandas groupby 不能用 mean() 含非数值列
Q4.1：rideshare 有 string 列 → 必须先 get(['app', 'price']) 再 groupby。pandas 2.x 不再自动跳过非数值。
陷阱 5：convenience sample 不行（哪怕大）
Q2.3：sample 是否能 estimate population 关键在无偏（random + representative），不在 size。convenience sample 的 bias 是 systematic，n 多大都救不了。
陷阱 6：difference CI 比 single CI 更宽
Q3.10：估两个量之差，不确定度叠加（变方差和）。考试常考"以下哪个 CI 最宽？"——差 CI 几乎总是最宽的。
陷阱 7：Q4.3 必须 return 而不是 print
函数返回字符串，外部才能用变量接住。print 只在屏幕显示，函数返回值是 None。autograder 抓得很严。
