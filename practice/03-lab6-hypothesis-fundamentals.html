📚 Section 1 — Therapeutic Touch（Emily Rosa, 1996）

背景：TT 从业者声称能感知人体能量场（HEF）。Emily 设计了双盲实验：280 次试验，从业者隔板伸手过来，Emily 抛硬币决定把自己的手放在他们的左手还是右手附近，从业者猜哪只。结果 280 次猜对 123 次（≈ 43.9%，比 50% 低）。

📐 假设（双侧！）
H0: 从业者随机猜，每次猜对的真实概率 = 0.5
HA: 真实概率不等于 0.5（任意方向都算 evidence——这里其实低于 0.5，但题目要求测"any effect, in either direction"）

统计量：|观察比例 − 0.5|（带绝对值，因为双侧）
Simulate under H0：从 [0.5, 0.5] 抽 280 次，算"猜对"比例与 0.5 的绝对差
p 值：右尾（≥），因为统计量已经取了 |·|（始终 ≥ 0），越大越极端
Q 1.1写 null & alternative

"Write both of your answers" — Markdown 答案不被 autograder 检查。

H0：The TT practitioner is guessing randomly; the proportion of correct guesses is 0.5.

HA：The proportion of correct guesses is different from 0.5（双侧）。

💡 为什么双侧：题干明示 "We want to test whether or not some treatment has an effect, in either direction. This is not the same as testing whether the treatment actually works as intended."—— Emily 不是想证 TT 有效，而是测 TT 跟乱猜有没有差别。任何方向（明显高于 50% 或明显低于 50%）都是 evidence against H0。

Q 1.2expected_correct 
expected_correct = 0.5

💡 为什么 0.5：H0 等同于"投硬币"——每次猜对概率 = 0.5。autograder 的 message "Don't just copy this answer!" 是看你有没有想清楚 H0 的含义，不是真的禁止填 0.5。

Q 1.3best_stat（双侧→选 |·|）

三个选项：1=expected − actual（带方向）；2=|expected − actual|（绝对差）；3=expected + actual（和，没意义）。

best_stat = 2

💡 为什么不是 1（带方向）：1 是 expected − actual。如果 actual = 0.7（远高于 0.5），差 = −0.2 是负的；如果 actual = 0.3（远低于 0.5），差 = +0.2 是正的。"两种极端"得到相反符号。双侧检验要求"任何方向偏离都算极端"，所以必须取 |·|。

💡 为什么不是 3（和）：expected + actual 在 [0, 2] 区间，跟"偏离 H0"没有任何单调关系，毫无判别力。课件里 lec 19 那个 coin flipping 例子就是反例集。

💡 对比 HW 6 Q1.6：那里是单侧 HA（"p > 1/144"），所以不能用 |·|（绝对差丢失方向，单侧不可用）；这里是双侧，必须用 |·|。这是同一套机制对镜像问题的两种答案。

Q 1.4calculate_test_stat(expected, actual) 
def calculate_test_stat(expected_prop, actual_prop):
    return np.abs(expected_prop - actual_prop)
✅ 测试：
  • calculate_test_stat(0.6, 0.5) + calculate_test_stat(0.1, 0.4) = 0.1 + 0.3 = 0.4 ✓
  • calculate_test_stat(0.99, 0.43) = 0.56 ✓
  • calculate_test_stat(0.43, 0.99) = 0.56 ✓ —— 顺序换了结果一样，证明确实是绝对差。
Q 1.5observed_test_stat 
observed_test_stat = calculate_test_stat(0.5, 123 / 280)
# = |0.5 − 0.4393| = 0.0607 ≈ 17/280

💡 17 怎么出来的：H0 下"应猜对 140 次"（=280×0.5）；实际猜对 123 次；差 17 次 → 比例差 17/280 ≈ 0.0607。autograder 反推 np.isclose(observed_test_stat, 17/280)。

Q 1.6单次模拟 + one_simulated_test_stat 
model_proportions = np.array([0.5, 0.5])
simulation_proportion   = np.random.multinomial(280, model_proportions)[0] / 280
one_simulated_test_stat = np.abs(0.5 - simulation_proportion)

💡 为什么 [0] / 280：multinomial(280, [0.5, 0.5]) 返回 array 如 [142, 138]，第 0 位是"猜对"次数。除以 280 得到该次模拟的"猜对比例"。

⚠️ 高频隐性陷阱（autograder 卡死大军）：如果你写成 
simulation_proportion   = np.random.multinomial(280, model_proportions)[0] / 280
one_simulated_test_stat = calculate_test_stat(0.5, np.random.multinomial(280, model_proportions)[0] / 280)
# ❌ 调用了 multinomial 两次！两个变量基于不同的随机结果
autograder 测试 5 (np.isclose(one_simulated_test_stat, np.abs(0.5 - simulation_proportion))) 就会失败，提示 "Test case failed"。

修复：把 simulation_proportion 保存下来，one_simulated_test_stat 用它计算，不要再次调用 multinomial：np.abs(0.5 - simulation_proportion)。
Q 1.710,000 次模拟 → simulated_test_stats 
num_repetitions = 10000
simulated_test_stats = np.array([])
for i in np.arange(num_repetitions):
    sim_prop = np.random.multinomial(280, model_proportions)[0] / 280
    simulated_test_stats = np.append(simulated_test_stats, np.abs(0.5 - sim_prop))

💡 注意 np.all(simulated_test_stats <= 0.2) 应该 True：H0 下 SD ≈ √(0.5·0.5/280) ≈ 0.0299，4 个 SD ≈ 0.12，几乎不会 > 0.2。如果你 |·| 写漏了，可能出现负值——autograder 测试 2 就过不了。

Q 1.8p_def（p-value 定义）

4 个选项：1=p 是 "exactly = observed" 的概率；2="= observed 或更偏向 alt 方向"；3=under alt...；4=number（不是概率）。

p_def = 2

💡 严格定义：p-value = P( test statistic ≥ observed | H0 )，"≥" 那部分 = "or even further in the direction of the alternative"。

💡 排除其他选项：

  • 1 错在 "equal to"——连续分布的精确等于概率 = 0，没意义；离散也只是单点。
  • 3 错在 "under the alternative"——p 值永远是在 H0 下算的。
  • 4 错在 "number"——p 是 fraction / probability，不是计数。

Q 1.9emily_p_val 
emily_p_val = np.count_nonzero(simulated_test_stats >= observed_test_stat) / 10000

💡 为什么右尾 >=：统计量已经是 |·|（始终非负），越大越偏离 H0。p 值 = 模拟分布中比 observed 还远的比例 = 右尾。

💡 经典数据：observed ≈ 0.0607（约 17/280）。在 H0 下，|sample_prop − 0.5| ≥ 0.0607 大约对应 z ≥ 2.03，双侧 p ≈ 0.042。所以 p ≈ 0.04 < 0.05 但 > 0.01——题干随后给的 cutoff 是 0.01（医学保守标准），按这个 cutoff 还不能拒绝；但按 0.05 就能拒绝。Lab 6 这里题目没让你写结论，所以不卡这点。

💡 真实历史：Emily Rosa 这个实验后来发在 JAMA 上，TT 从业者圈子炸了。原始论文里他们用的就是双侧检验，p < 0.05 → "TT 跟随机猜没显著区别"。

📚 Section 2 — YouTube Advertisements（King Triton）

背景：YouTube 公开说 92% 广告可跳过。King Triton 看了 100 条，发现 18 条不可跳过（只有 82% 可跳过），怀疑 YouTube 的 92% 偏高。

📐 假设（单侧！）
H0: 92% 可跳过、8% 不可跳过（YouTube 的声明正确）
HA: 真实可跳过比例 < 92%（等价：不可跳过比例 > 8%）

统计量：100 条广告中 unskippable 的数量（不取 |·|）
Simulate under H0：从 [0.92, 0.08] 抽 100 个，记录 unskippable 数
p 值：右尾——unskippable 越多越支持 HA（"YouTube 偏高了" ↔ "实际更多不可跳过"）
Q 2.1写 null & alternative

H0: YouTube's claim is accurate — 92% skippable, 8% unskippable.

HA: 真实可跳过比例 小于 92%（即 unskippable 比例 > 8%）。

💡 为什么单侧：King Triton 怀疑 92% 偏高——只在乎"实际比 92% 低"一个方向。"实际比 92% 高"对他没意义（他不是来质疑 YouTube 谦虚了的）。

Q 2.2null_probabilities 
null_probabilities = np.array([0.92, 0.08])
# [0]=skippable, [1]=unskippable

💡 顺序很关键：题目明示 "first element = skippable, second = unskippable"。如果反了，下面 [1] 取出来的就是 skippable 数，结论会完全错。

Q 2.310,000 次模拟 → unskippable_counts 
unskippable_counts = np.array([])
for i in np.arange(10000):
    unskippable_counts = np.append(unskippable_counts,
                                   np.random.multinomial(100, null_probabilities)[1])

💡 [1] 是 unskippable 数：因为 null_probabilities 第 1 位是 unskippable。题目要求记录"unskippable 的数量"——不要除以 100，要的就是 count（0–100 的整数）。

✅ 测试：7.5 < np.mean(unskippable_counts) < 8.5。H0 下期望 = 100 × 0.08 = 8，正好在区间里。
Q 2.4unskippable_p_val 
unskippable_p_val = np.count_nonzero(unskippable_counts >= 18) / 10000

💡 为什么右尾：观察到 unskippable = 18。HA: 真实不可跳过比例 > 8% → unskippable count 越大越支持 HA。p 值 = 模拟中 ≥ 18 的比例。

💡 这个 p 值有多小：H0 下期望 8，SD ≈ √(100·0.08·0.92) ≈ 2.71，observed 18 对应 z = (18−8)/2.71 ≈ 3.7 → 单侧 p ≈ 0.00011。题干 autograder 测试就要求 0 ≤ unskippable_p_val ≤ 0.01——一个数量级别的过界。

💡 结论（题目没让写但你应该知道）：p ≈ 0.0001 ≪ 0.01 → reject YouTube 的 92% 声明。证据非常强。

⚠️ 本作业典型陷阱（横切多题）

陷阱 1：双侧 vs 单侧——统计量取不取 |·| 是分界线
Q1.3（双侧）选 |·|；HW 6 Q1.6（单侧）排除 |·|。同一道题型，alt 方向不同，答案就完全相反。看清题干 "in either direction"（双侧）/ "more often than expected"（单侧）。
陷阱 2：np.random.multinomial 的索引意义
multinomial(n, [p₀, p₁])[k] 是第 k 类的计数。Q1.6 取 [0]（"猜对" = 第一类）；Q2.3 取 [1]（"unskippable" = 第二类）。索引对应你的 probability array 顺序。
陷阱 3：Q1.6 的 "同一次抽样" 陷阱
simulation_proportion 和 one_simulated_test_stat 必须基于同一次 multinomial 调用。如果你两个变量分别调用 multinomial，autograder test 5 会失败。修复：先存 sim_prop，再用它算 abs 差。
陷阱 4：p-value 定义（Q1.8）
选项 2 才对——"= observed or even further in the direction of the alt"。不是 "exactly = observed"（连续分布概率永远 0）。不是 "under the alternative"。不是 "number"。
陷阱 5：counts vs proportions
Q2.3 题目说 "Number of unskippable ads"——不要除以 100。Q1.6 题目说 "proportion"——必须除以 280。要看题目要什么。两边的 axvline 也跟着改：Q2 用 x=18（count），Q1 用 x=observed_test_stat（proportion 的绝对差）。
