📚 Section 1 — Testing the CLT: Coin Flips & Ramen Ratings

Q 1.1simulate_sample_n(n)

"Repeat 5000 times: flip n fair coins, find proportion of heads. Return array of 5000 proportions."

def simulate_sample_n(n):
    proportions = np.array([])
    for i in np.arange(5000):
        sample = np.random.choice([0, 1], n)
        proportions = np.append(proportions, np.mean(sample))
    return proportions

💡 为什么 mean of (0/1) = proportion：1 表示正面、0 表示反面，array 平均 = (正面数 + 0) / n = 正面比例。这是"proportion as mean"——Quiz 3 / Quiz 4 反复用，必须烂熟。

💡 题干图意：随着 n 从 2 → 400，histogram 越来越像钟形 + 越来越窄（per CLT）。Lab 看图就能感受 CLT。

Q 1.2predict_sd(n)（CLT 公式预测，不模拟）

"Should return the predicted SD (according to the CLT) of the sample mean's distribution. Do not use or modify simulate_sample_n. You should not actually take any samples."

def predict_sd(n):
    return ramen_std / np.sqrt(n)

💡 必背 CLT 公式：样本均值分布的 SD = 总体 SD / √n。题干预先算了 ramen_std = np.std(ramen.get('Stars'))，直接用。

💡 为什么"不用采样"：CLT 是定理，不需要模拟就能给出预测值。整道题想让你区分"公式预测"（predict_sd）与"实测"（empirical_sd）。

✅ 反推：predict_sd(75) = 1.0151 / √75 ≈ 0.11722，autograder 验 int(predict_sd(75) × 10¹⁰ % 10⁶) == 175288 ✓
Q 1.3empirical_sd(n)（用模拟验证） 
def empirical_sd(n):
    sample_means = np.array([])
    for i in np.arange(1000):
        sample = ramen.sample(n, replace=True)
        sample_means = np.append(sample_means, sample.get('Stars').mean())
    return np.std(sample_means)

💡 这跟 bootstrap 几乎一样的代码：抽 1000 次、每次大小 n、有放回。区别只是这里用 np.std(sample_means)——bootstrap 用 percentile，empirical_sd 用 std。同一段代码不同后处理。

💡 题目意图：图上散点 (predicted_sd, empirical_sd) 几乎重合，确认 CLT 公式正确。

📚 Section 2 — Til' Death Duo: Duolingo（Q2.1–2.6）

背景：193 个国家的 Duolingo 最热门语言数据。134 个国家最热门 = English（≈ 69.4%）。把 duolingo 当总体，每次 SRS 抽 100 个国家。

Q 2.1props：10000 个 English 比例 
props = np.array([])
for i in np.arange(10000):
    sample = duolingo.sample(100)
    prop = np.count_nonzero(sample.get('most_pop_language') == 'English') / 100
    props = np.append(props, prop)

💡 duolingo.sample(100) 默认无放回（SRS）。题目明确说 "simple random sample"。babypandas 的 .sample(n) 不带参数时是 without replacement。如果要 with replacement，加 replace=True。

💡 为什么用 np.count_nonzero：sample.get('most_pop_language') == 'English' 是 boolean Series。count_nonzero 数 True 的个数。也可以用 .sum()，因为 True 当作 1。

Q 2.2theoretical_props_mean vs actual_props_mean

"You should not access the data in props when calculating theoretical."

theoretical_props_mean = np.count_nonzero(duolingo.get('most_pop_language') == 'English') / duolingo.shape[0]
actual_props_mean      = props.mean()

💡 CLT 关于 mean of sample means：sample mean 分布的均值 = 总体均值（CLT 的 unbiasedness）。这里总体比例 = 134/193 ≈ 0.6943。实际 props.mean() ≈ 0.6943 ± noise。

💡 为什么不能访问 props 算 theoretical：theoretical 是理论预测，必须从总体（duolingo）算出；如果用 props 那就等于 actual 了，没区别。

Q 2.3counts：10000 个 English 数量（不除 100） 
counts = np.array([])
for i in np.arange(10000):
    sample = duolingo.sample(100)
    cnt = np.count_nonzero(sample.get('most_pop_language') == 'English')
    counts = np.append(counts, cnt)

💡 跟 Q2.1 唯一区别：不除以 100。counts 是 0–100 的整数（每个样本中 English 国家数），props 是 0–1 的比例。

Q 2.4theoretical_counts_mean

Hint: "A count is not a mean, so CLT does not directly apply. However, you can use theoretical_props_mean to help."

theoretical_counts_mean = 100 * theoretical_props_mean
actual_counts_mean      = counts.mean()

💡 count = size × proportion：每个 sample 中 count = 100 × proportion；对平均也是 mean(counts) = 100 × mean(props) = 100 × 0.6943 ≈ 69.43。

💡 sample mean 公式不直接给 count 用：CLT 说 sample mean 分布趋近 normal。count = sum，不是 mean。但 sum 跟 mean 差 n 倍线性变换，所以 sum 的分布也是 normal（CLT 的第二种形式）——只是公式不一样。

Q 2.5count_most_pop_language(size) 泛化 
def count_most_pop_language(size):
    counts = np.array([])
    for i in np.arange(10000):
        sample = duolingo.sample(size)
        cnt = np.count_nonzero(sample.get('most_pop_language') == 'English')
        counts = np.append(counts, cnt)
    return counts

💡 把 100 改成 size。然后 Q2.6 看 size = 10, 25, 50, 100 时 histogram 的变化。

Q 2.6观察 counts 分布的 mean / SD 随 size 怎么变？

4 选项：1=mean 增加；2=mean 减小；3=SD 增加；4=SD 减小。选所有 true 的。

q2_6 = [1, 3]

💡 mean 增加（选 1）：mean(counts) = size × p。size 越大 mean 越大。

💡 SD 增加（选 3）：counts 的 SD = √(size × p × (1−p))——size 越大 SD 越大。注意是 √size，不是 size 本身。

⚠️ 跟 proportions 对比：proportions 的 SD = √(p(1−p)) / √size——size 越大 SD 越小。同一个 sample，count 和 proportion 的 SD 趋势相反。这是高频陷阱：count 的 SD 与 √size 成正比；proportion 的 SD 与 1/√size 成正比。

📚 Section 3 — Drake vs Kendrick Lamar（Q3.1–3.5）

背景：UCSD 有 43,533 名学生。Sofia 调查 580 人，203 选 Drake，377 选 Kendrick（≈ 65%）。本节禁止用 kendrick_proportions 数组——只能用 CLT + 0/1 SD 公式。

📐 必备公式
0/1 数据 SD: √(p₀ × p₁) 其中 p₀ 是 0 的比例、p₁ 是 1 的比例
Sample mean SD（CLT）: pop_SD / √n
Normal 95% rule: mean ± 2·SD
样本量（约束 SD ≤ w）: n ≥ pop_SD² / w²
Q 3.1approximate_sd（不许用 kendrick_proportions） 
approximate_sd = np.sqrt(0.35 * 0.65) / np.sqrt(sample_size)
# sample_size = 580
# ≈ 0.4770 / 24.083 ≈ 0.01981

💡 套路：sample proportion 的 SD = 总体 0/1 SD / √n = √(p·(1−p)) / √n。总体 p 未知，用 sample 比例代入（0.35 / 0.65）。

💡 为什么 sample 比例可以替代总体：sample 比例是总体比例的一致估计（unbiased estimate）。当 n 较大（580 ≫ 30），二者非常接近。这是"plug-in"思想。

✅ autograder 反推：int(approximate_sd × 10⁶ % 10⁴) == 9805 → approximate_sd ≈ 0.01980 ✓
Q 3.2exact_sd（实测，与 Q3.1 对比） 
exact_sd = np.std(kendrick_proportions)

💡 跟 Q3.1 数值应该非常接近——这就是 CLT 在实际场景的验证。

Q 3.395% CI（Normal-based，不用 bootstrap）

"Without accessing kendrick_proportions in any way, compute an approximate 95% CI."

lower_limit = 0.65 - 2 * approximate_sd
upper_limit = 0.65 + 2 * approximate_sd
# ≈ [0.6104, 0.6896]

💡 Normal-based CI 公式：95% CI = sample mean ± 2 × SD_sample_mean。这就是 Quiz 3 的"必背 8 件套"之第 3 条。Kendrick 比例的中心是 0.65（样本观测），SD 用 approximate_sd。

💡 跟 Bootstrap CI 的对比：

  • Bootstrap：从 sample 有放回采样 1000 次 → 取 2.5/97.5 percentile（数据驱动，慢）。
  • Normal-based（CLT-based）：直接 mean ± 2·SD（公式驱动，快，但要先估 SD）。
两种 CI 应该非常接近——这是 CLT 的另一层验证。题目专门"不准用 bootstrap"是为了考你 normal 公式。

Q 3.4样本量计算 → new_sample_size

"Sofia wants SD ≤ 0.004. Compute the smallest sample size. Use the SD ≈ sample SD assumption."

new_sample_size = int(np.ceil(0.35 * 0.65 / (0.004 ** 2)))
# = ceil(0.2275 / 0.000016) = ceil(14218.75) = 14219

💡 推导：要 SD = pop_SD / √n ≤ 0.004 → √n ≥ pop_SD / 0.004 → n ≥ pop_SD² / 0.004² = 0.2275 / 1.6×10⁻⁵ = 14218.75。必须向上取整到 14219（取 14218 的话 SD 会略 > 0.004，不满足约束）。

⚠️ 必须 np.ceil 不是 round 或 int：14218.75 round → 14219（碰巧对），但 14218.25 round → 14218（小于约束）。"smallest such that" 永远向上取整。autograder 给出 "Rounded up - 4/4 tests correct" 提示就是这意思。

💡 一般公式："SD ≤ w" 时 n ≥ pop_SD² / w²。对 0/1 数据，pop_SD = √(p·(1−p))，所以 n ≥ p(1-p) / w²。

Q 3.5想要 SD = 0.002（更精）→ q3_5

4 个选项：1=反复重抽直到 SD = 0.002；2=size 减半；3=size 翻 4 倍；4=不可能，超过总人口。

q3_5 = 4

💡 计算：SD ∝ 1/√n。SD 减半 → √n 翻倍 → n 翻 4 倍 = 4 × 14219 = 56876。但 UCSD 总人数 43,533 < 56,876 → 不可能。

💡 排除其他选项：

  • 1 错：反复重抽不会改变单个样本的 SD，只是看到不同样本的随机波动。
  • 2 错：SD 减半要 n × 4，不是 n / 2。SD = 1/√n 的关系。
  • 3 是正确的 n 翻 4 倍，但4 才是更"true"的，因为实际不可能（超总人数）。题目选 "best answer and explanation"，所以选 4 而不是 3。

📚 Section 4 — Key Concepts（Q4.1–4.4）

Q 4.1SU 公式：value = 36, mean = 20, SD = 3

4 选项考 SU 公式 (value − mean) / SD。

q4_1 = 3
# = (36 − 20) / 3

💡 必背：standard units = (value − mean) / SD。Z = (X − μ) / σ。不是 (mean − value)、不除以 √SD、不平方。

Q 4.2Chebyshev: 至少一半数据落在 mean ± k·SD 内，求最小 k

4 个选项：1.35 / 1.40 / 1.45 / 1.50。

q4_2 = 3   # k = 1.45 是最小满足的

💡 Chebyshev 不等式：至少 1 − 1/k² 的数据在 ± k·SD 内。
要 1 − 1/k² ≥ 0.5 → k² ≥ 2 → k ≥ √2 ≈ 1.4142。最小选项 ≥ 1.4142 是 1.45（选项 3）。

💡 常用 Chebyshev 数值（必背）：

  • k=2 → 至少 75%
  • k=3 → 至少 88.8%
  • k=4 → 至少 93.75%
  • k=5 → 至少 96%

Q 4.35 个陈述选所有"始终为真"的 
q4_3 = [1, 5]

💡 逐项分析：

  • 1. 大样本的 sample mean 直方图近似 normal → CLT 直接结论 ✓
  • 2. 大样本的 sample median 直方图近似 normal → CLT 只对 mean/sum 适用，不对 median。✗（高频陷阱！）
  • 3. 知道 mean 和 SD 就能算 68% CI（mean ± 1·SD） → 68-95-99.7 法则只对 normal 分布；任意分布不能这么用。✗
  • 4. 任何分布 95% 数据在 mean ± 2·SD 内 → 这是 normal 的性质（68-95-99.7），不是 Chebyshev。任意分布只能说 ≥ 75%。✗
  • 5. 任何分布 ≥ 95% 数据在 mean ± 5·SD 内 → Chebyshev: 1 − 1/25 = 96% ≥ 95% ✓

⚠️ Normal 性质 vs Chebyshev：
  • Normal 68-95-99.7：精确比例（68 / 95 / 99.7），仅对 normal 分布。
  • Chebyshev：下界（at least），对任何分布。
题目用 "for any distribution" 时只能用 Chebyshev；用 "approximately normal" 才能用 68-95-99.7。对调就错。
Q 4.4SD 要减到 y/4，n 要乘几？

4 选项：2x / 4x / 8x / 16x。

q4_4 = 4   # 16x

💡 推导：SD = pop_SD / √n。当 n = x 时 SD = y。当 SD = y/4 时，pop_SD/√n_new = y/4 = (pop_SD/√x) / 4 → √n_new = 4√x → n_new = 16x。

💡 通用规律：SD 缩小 k 倍 → n 要乘 k²。SD 缩 2x → n 乘 4；缩 4x → 乘 16；缩 10x → 乘 100。这就是 Q3.5 的根：要把 0.004 → 0.002（缩 2x），n 乘 4。

⚠️ 本作业典型陷阱

陷阱 1：CLT 公式 SD 用谁的
SD_sample_mean = pop_SD / √n——分子是总体 SD，不是 sample SD。但实际操作中我们用 sample SD 估计 pop SD（plug-in）。考试如果给你 pop SD，直接用；如果只给 sample，用 sample 当 estimate。
陷阱 2：counts 的 SD 跟 proportions 趋势相反
SD_counts = √(n · p · (1-p))（与 √n 成正比）；SD_props = √(p · (1-p)) / √n（与 √n 成反比）。Q2.6 的两个真陈述都基于此。
陷阱 3：normal 公式 vs Chebyshev
Q4.3 选项 3、4 都把 normal 的精确法则（68-95-99.7）误用到"任何分布"上。要对任何分布说话只能 Chebyshev（下界，approximate / at least）。Q4.3 选项 5 才是真的（5 SD 内 ≥ 96%）。
陷阱 4：CLT 只对 mean / sum 适用
Q4.3 选项 2 说"sample median 也近似 normal"——错。CLT 明确只保证 sample mean / sum 的分布近似 normal。Median、min、max、IQR 等都不保证。
陷阱 5：样本量公式记反
Q3.4 要 SD ≤ w：n ≥ pop_SD² / w²。不是 n ≥ pop_SD / w²，不是 n ≥ (pop_SD / w)²（其实等价，但要注意分子分母位置）。
Quiz 3 标准形式（按 CI 宽度）：n ≥ (4 × SD / w)²（w 是 CI 宽度，95% CI 宽 = 4 × SD/√n）。这两个公式都常考。
陷阱 6：n 要向上取整 (np.ceil)
Q3.4 算出 14218.75 → int(np.ceil(...)) = 14219。不是 int(14218.75) = 14218（这是向下取整），不是 round(14218.75) = 14219（碰巧对，但 14218.4 round → 14218 就错了）。
