Q 1.1'Revenue' 列的类型 
revenue_type = type(raw_fortune_500.get('Revenue').iloc[0])

💡 type(x) 返回对象类型。Revenue 存的是 '$611.29' 这种字符串 → str。

Q 1.2Walmart 的 revenue string 
walmart_revenue_string = raw_fortune_500.get('Revenue').iloc[0]
# = '$611.29'
Q 1.3string → 实际美元数 
walmart_revenue = float(walmart_revenue_string.strip('$')) * 1e9

💡 三步：① .strip('$') 去掉 $ → '611.29'；② float() 转 float → 611.29；③ × 1e9（10 亿）→ 611,290,000,000。注意顺序：先 strip 再 float（float 不能解析 '$' 开头）。

Q 1.4定义 convert_revenue_string_to_number 
def convert_revenue_string_to_number(some_revenue_string):
    return float(some_revenue_string.strip('$')) * 1e9

💡 把 Q1.3 的逻辑封装成函数。这是 Lab 2 的核心思想——发现重复模式 → 抽函数。Q3.2 会用这个函数批量处理。

Q 2.3disemvowel：链式 .replace 
def disemvowel(a_string):
    return a_string.replace('a', '').replace('e', '').replace('i', '')\
                   .replace('o', '').replace('u', '')

💡 .replace(old, new) 字符串方法：返回新字符串（不修改原 string）。链式调用 — 每次替换都返回新 str，下一个 .replace 接着替换。

💡 .replace('a', '')：替换为空字符串 = 删除。

Q 2.4num_non_vowels：函数 composition 
def num_non_vowels(a_string):
    return len(disemvowel(a_string))

💡 一行 = 复用 + 计长度。这是 Lab 2 课程要点："function composition" —— 小函数组成大函数。

Q 2.5print_kth_most_streamed_song(k)：用 print 不 return 
def print_kth_most_streamed_song(k):
    sorted_df = spotify_streams.sort_values(by='Streams (Billions)', ascending=False)
    song_name = sorted_df.index[k - 1]
    print('The song ranked number', k, 'for most all-time Spotify streams is', song_name)

💡 k - 1 因为 Python 0-index。第 1 名 → index[0]，第 2 名 → index[1]…

💡 print vs return：print 在屏幕显示文字，函数返回值 None。如果外部赋值 x = print_kth...()，x 是 None。区分这两种 output 是面试常考点。

📚 Section 3 — Applying Functions

Q 3.13 个 callable 函数的 array 
some_functions = np.array([max, min, len])

💡 函数是 first-class object——可以放进 array 里。不带括号（带括号是调用）。Python 把函数当成普通"对象"传递。

Q 3.2加 'Revenue_Dollars' 列（核心模板） 
fortune_500 = raw_fortune_500.assign(
    Revenue_Dollars=raw_fortune_500.get('Revenue').apply(convert_revenue_string_to_number)
)

💡 .apply 套路：Series.apply(func) 把 func 应用到 Series 的每个元素，返回新 Series。然后 .assign(NewCol=...) 把这个新 Series 加成新列。

💡 apply(func) 传函数名，不带括号。apply(func()) 是调用 func 一次把结果传进来——错。

✅ shape = (500, 11)（原 10 列 + 1 新列）。
Q 3.3average revenue 
average_revenue = fortune_500.get('Revenue_Dollars').mean()
# ≈ 36,288,520,000 ≈ $36 billion
Q 3.4profit / revenue 比例（Profit 也是 string） 
profit_proportion = fortune_500.get('Profit').apply(convert_revenue_string_to_number) / fortune_500.get('Revenue_Dollars')

💡 convert_revenue_string_to_number 跨用于多列——只要列是 '$XX.YY' 这种格式，同一函数就能用。复用得很优雅。

💡 Series ÷ Series = 逐元素，返回 Series。500 行 → 500 个比例。

📚 Section 4 — Last Year's Revenue

Q 4.1filter → 转 % → 算 Previous_Revenue → assign 
def convert_pct_to_number(pct_string):
    return float(pct_string.replace('%', ''))

filtered = fortune_500[fortune_500.get('% Change') != '(No previous year)']
pct_change = filtered.get('% Change').apply(convert_pct_to_number)
previous_revenue = filtered.get('Revenue_Dollars') / (pct_change / 100 + 1)
with_previous_fortune_500 = filtered.assign(Previous_Revenue=previous_revenue)

💡 公式：current = previous × (1 + pct/100) → previous = current / (1 + pct/100)。注意分母是 1 + pct/100，不是 1 + pct。

💡 必须先 filter '(No previous year)'——否则 float('(No previous year)') 会报错。

💡 .replace('%', '')：去掉 % 符号。.strip 也行（'%' 在末尾时），但 .replace 更通用。

✅ 期望：filtered shape = (471, ...)（500 − 29 个无去年数据）。Walmart 2021 ≈ $571.3B。
Q 4.22021 平均 
average_revenue_2021 = with_previous_fortune_500.get('Previous_Revenue').mean()
# ≈ $33.6B
Q 4.3scatter: 去年 vs 今年 
with_previous_fortune_500.plot(kind='scatter', x='Previous_Revenue', y='Revenue_Dollars')

💡 应该看到强线性趋势——大公司去年大、今年也大。这暗示 regression 在这种数据上会很好用（铺垫 Final 内容）。

📚 Section 5 — Distribution of Revenue

Q 5.1density histogram, 宽度 0.5×10¹¹ 
bins = np.arange(0, fortune_500.get('Revenue_Dollars').max() + 0.5 * 10**11, 0.5 * 10**11)
fortune_500.plot(kind='hist', y='Revenue_Dollars', density=True, bins=bins)

💡 bins 公式：start=0，stop=max+step（保证 max 在最后一个 bin 里），step=0.5×10¹¹（500 亿美元）。
Walmart 的 ~6 千亿在最右；大多数公司在最左（< 500 亿）。

Q 5.2从图估计 > $100B 的公司数 
num_companies_more_than_100_billion = 38

💡 视觉估算：从 100B（= 2 个 bin）起到最右的所有 bar 的总面积 × 500（总公司数）≈ 38。
必须是 int——autograder 测 isinstance(..., numbers.Integral)。Q5.3 用代码验证这个估计。

Q 5.3用代码精确数 
num_companies_more_than_100_billion_code = fortune_500[
    fortune_500.get('Revenue_Dollars') > 100 * 10**9
].shape[0]
# = 38

💡 模板：count via shape[0]。filter + .shape[0] = 行数。比 .sum() 或 len() 更标准。

💡 教学意图：Q5.2 估、Q5.3 精确——验证你对 density histogram 的读图能力。一致 → 你读图对了；不一致 → 重新读图（或代码错）。

⚠️ 典型陷阱

陷阱 1：strip('$') 顺序——先去字符再 float。float('$611.29') 会报错。
陷阱 2：函数 .apply 传名不带 ()。带括号是调用。
陷阱 3：% Change 计算公式：previous = current / (1 + pct/100)，不是 current / (1 + pct)。
陷阱 4：先 filter '(No previous year)' 再转 float。否则 float 会报错。
陷阱 5：用 print() 函数返回 None，跟 return 不同。
陷阱 6：Q5.2 必须是 int（用 int() 包），不是 numpy int。
