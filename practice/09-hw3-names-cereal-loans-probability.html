📚 Section 1 — 100 Years of "C" Baby Names

Q 1.1男性 Charles 折线 + 峰值年份 
male_charles = baby[(baby.get('Name') == 'Charles') & (baby.get('Gender') == 'M')]
male_charles.groupby('Year').sum().plot(kind='line', y='Count')

male_charles_peak = 2   # 1950

💡 先 filter 再 groupby：双条件用 &，每条件外加括号（== 优先级低于 &）。groupby + sum 因为同名可能多 state，每年总和才是真实流行度。

Q 1.2num_babies：(State, Gender) 组合 (102 × 3) 
num_babies = baby.groupby(['State', 'Gender']).sum().reset_index().get(['State', 'Gender', 'Count'])

💡 51 state × 2 gender = 102 行。reset_index 把 multi-index 拆回普通列。.get([...]) 选回需要的 3 列（sum 会聚合所有数字列，可能多出 Year 等）。

Q 1.3avg_age：加权平均（高频考点） 
with_age = baby.assign(TotalAge=(2024 - baby.get('Year')) * baby.get('Count'))
grouped  = with_age.groupby(['Gender', 'Name']).sum()
avg_age  = (grouped
    .assign(Average_Age=grouped.get('TotalAge') / grouped.get('Count'))
    .reset_index()
    .get(['Gender', 'Name', 'Average_Age']))

💡 必须加权：每行已聚合（Count 个 baby in Year y）。直接 .mean() 算 Year 会把1 个 baby和1000 个 baby当成同权重，错。
模板：weighted_mean = Σ(x · w) / Σ(w)。这里 x = (2024 − Year)，w = Count。

⚠️ 这是 Midterm 的高频考点——"给定有 Count 列的数据，求每组的加权 mean / median"。任何"avg age"题目都要先想 "weighted by Count?"

📚 Section 2 — Cerealsly?!

Q 2.1merge 不同列名（left_on + right_on） 
states_and_varieties = (states
    .merge(varieties, left_on='Favorite Cereal', right_on='name')
    .drop(columns=['name'])
    .sort_values(by='State')
    .reset_index(drop=True))

💡 列名不同的 merge：用 left_on= 和 right_on= 指定。merge 后两列都保留——把 'name' 重复列 .drop 掉。
states 2 列 + varieties 16 列 − 1 重复 = 17 列 ✓

Q 2.2set difference（merge 之后看哪些没出现） 
states_not_varieties = (len(states.get('Favorite Cereal').unique())
                        - len(states_and_varieties.get('Favorite Cereal').unique()))
varieties_not_states = (len(varieties.get('name').unique())
                        - len(states_and_varieties.get('Favorite Cereal').unique()))

💡 inner join 后剩下的 = 双方都有。所以 "只在 states 不在 varieties" = states 总 distinct − merged distinct。Quiz 3 常考集合的关系。

✅ 反推：states_not_varieties = 2，varieties_not_states = 63。
Q 2.3同名列 merge → _x/_y 后缀 
both = states.merge(states_alt, on='State')
same_cereal = (both.get('Favorite Cereal_x') == both.get('Favorite Cereal_y')).sum()

💡 自动后缀：当两表有同名列 + 用 on='State' merge → 'Favorite Cereal' 列会变成 Favorite Cereal_x（来自 left）和 Favorite Cereal_y（来自 right）。用 == 比对得到 bool Series，.sum() 数 True。

📚 Section 3 — Tritonmobile

Q 3.1calculate_interest(score)：分段 
def calculate_interest(score):
    if   score >= 781: return 0.056
    elif score >= 661: return 0.069
    elif score >= 601: return 0.093
    elif score >= 501: return 0.119
    else:               return 0.142

💡 从高到低排：每个 elif 把剩下的范围检查"是否 ≥ 下一个阈值"。顺序反了会出错——比如把 >= 501 放最前面会让 score=900 也返回 0.119。

Q 3.2calculate_principal(price, down) 
def calculate_principal(price, down_payment):
    pct = down_payment / price
    if   pct < 0.10: discount = 0.02
    elif pct < 0.20: discount = 0.05
    elif pct < 0.30: discount = 0.09
    elif pct < 0.40: discount = 0.14
    else:             discount = 0.20
    return (price - down_payment) * (1 - discount)

💡 这次从低到高排（因为用 <）。等价于"小于 10% → 折扣 2%；落在 [10%, 20%) → 5%"。顺序反了又会错。
验证：(50000−11250) × (1 − 0.09) = 38750 × 0.91 = 35262.5 ✓

Q 3.3函数 composition：monthly_payment(price, down, score) 
def monthly_payment(price, down_payment, score):
    interest  = calculate_interest(score)
    principal = calculate_principal(price, down_payment)
    monthly   = principal * (interest / 12) / (1 - (1 + interest / 12) ** (-60))
    return round(monthly)

💡 annuity 公式：M = P × r / (1 − (1+r)^(-n))，其中 r = monthly interest = interest/12，n = months = 60（5 年）。指数是负——常错写成正。

💡 函数 composition：monthly_payment 复用前两个函数。这是 OOP 的精神——把复杂逻辑拆成小块，组合调用。考试常考"看代码理解 composition"。

📚 Section 4 — Alternating Products

Q 4.1alternating_product(values)：每隔一个相乘 
def alternating_product(values):
    product = 1
    for i in np.arange(0, len(values), 2):
        product = product * values[i]
    return product

💡 np.arange(0, n, 2) = [0, 2, 4, …]。step 参数控制间隔。这是取偶数位的标准模板。

💡 累加器初值 = 1（乘法的 identity）。加法累加器初值 = 0。

Q 4.2alternating_sign_product：符号交替 
def alternating_sign_product(values):
    product = 1
    for i in np.arange(len(values)):
        if i % 2 == 0:
            product = product * values[i]
        else:
            product = product * (-values[i])
    return product

💡 i % 2 == 0：检查偶数位（包括 0）。所有元素都参与（不是每隔一个），但奇数位取负。
验证：[2, 3.5, 1] → 2 × (−3.5) × 1 = −7 ✓

📚 Section 5 — Lucky Triton Lotto

Q 5.1tritonball_chance 
tritonball_chance = 1 / 8

💡 单选：从 1–8 等概率，匹配某固定数字 = 1/8。

Q 5.2first_three_chance：无放回 
first_three_chance = (1 / 29) * (1 / 28) * (1 / 27)

💡 按顺序无放回：第 1 个从 29 选 → 1/29；第 2 个从剩下 28 选 → 1/28；第 3 个 1/27。分母递减是无放回的标志。

Q 5.3free_housing_chance 
free_housing_chance = (1/29) * (1/28) * (1/27) * (1/26) * (1/25) * (1/8)

💡 独立事件相乘：白球抽取（5 个数无放回） + Tritonball（独立的另一组）。两者独立——所以总概率 = 白球全中 × Tritonball 匹配。

Q 5.4"complement"：不中的概率 
no_free_housing_chance = 1 - free_housing_chance

💡 P(not A) = 1 − P(A)。Quiz 3 / Midterm 必备。"至少一次"也常用 1 − P(从来没有)。

⚠️ 典型陷阱

陷阱 1：加权平均不能用 .mean()
Q1.3：如果有 Count 列代表 frequency，直接 mean 会忽略权重。要用 Σ(x·w) / Σw。
陷阱 2：merge 不同列名要 left_on/right_on
Q2.1。merge 后两列都保留，记得 drop 重复的那个。
陷阱 3：merge 同名列产生 _x/_y
Q2.3。访问时要用 'Favorite Cereal_x'、'Favorite Cereal_y'，不能再用原名。
陷阱 4：if/elif 顺序方向（>= vs <）
Q3.1（高分→低分用 >=）vs Q3.2（小折扣→大折扣用 <）。方向反了结果完全错。
陷阱 5：annuity 公式指数是负
Q3.3：(1+r)^(-n)。常错写成正——结果会差好几个数量级。
陷阱 6：乘法累加器初值是 1，不是 0
Q4.1：product = 1。0 会让结果永远是 0。
陷阱 7：无放回的分母递减
Q5.2：1/29 × 1/28 × 1/27（不是 1/29³）。每抽走一个，pool 少 1。
