🚨 考前 30 分钟速看 · Top 10 易记混措辞 ⭐⭐⭐

明早起床 / 进考场前 5 分钟最后扫一遍。每条都是历年错答陷阱。

  1. CI 解读唯一正确版：
    "Approximately X% of the intervals we create through this process will contain the true population parameter."
    错答：① "X% chance true param is in this CI" ❌（CI 固定、truth 固定，无概率）② "X% of sample data fall in CI" ❌ ③ "Definitely contains" ❌
  2. "Yes/No, due to CLT" 三问 checklist：(1) sample 是 random 吗？(2) n 够大吗（一般 ≥ 30）？(3) 我们关心的是 sample mean / proportion 吗？三个全 yes 才能 due to CLT。
  3. residual = actual − predicted。方向绝对不能反。正 residual = 真实值在线上方。
  4. reject / fail to reject / 永不 accept。p < α → reject null；p ≥ α → fail to reject。永远不写 "accept the null"。
  5. Bootstrap 必 replace=True 且 size = 原 sample size。sample.sample(sample.shape[0], replace=True)
  6. Chebyshev 是下界（"at least"），不是精确。z=2 → ≥ 75%；z=3 → ≥ 88.8%；z=4 → ≥ 93.75%。
  7. Sample size 公式里 w 是 CI 总宽度，不是 margin（半宽度）。
  8. Density histogram bin [a, b) 半开区间，最后一个 bin 右端闭合。一个 value 只算进一个 bin。
  9. series.apply(func) 不带括号。.apply(func()) 会报错。
  10. 布尔过滤用 & / | / ~，不是 and / or / not。每个条件外套括号。

1. CI 一切 ⭐⭐⭐

Bootstrap CI 模板（填空版，考试常以 fill-in-the-blank 出现）

boot_stats = np.array([])
for i in np.arange(10000):                              # ← reps
    resample = my_sample.sample(my_sample.shape[0],     # ← 必须 = 原 sample size
                                 replace=True)           # ← 必须 True
    stat = resample.get('col').mean()                         # ← 想估什么就算什么
    boot_stats = np.append(boot_stats, stat)
ci = np.percentile(boot_stats, [2.5, 97.5])         # ← 95% CI

c% CI 的 percentile

low = (100 − c) / 2 ； high = (100 + c) / 2
clowhigh
80%1090
90%595
92%496
95%2.597.5
99%0.599.5

95% CLT-based CI（仅用于 sample mean 或 proportion）

CI = [ sample_mean − 2 · (SD / √n) , sample_mean + 2 · (SD / √n) ]

这里 SD 是 sample SD（充当 pop SD 估计），n 是 sample size。系数 2（课程用 2，不是 1.96）。

通用置信度 z 反推

cdf(z) = (1 + c) / 2 → z = stats.norm.ppf((1 + c) / 2)

例：c = 0.92 → cdf(z) = 0.96 → z ≈ 1.75。CLT CI = [mean ± z · SD/√n]。

CI 解读 ✅/❌ 对照

说法对错原因
"95% of intervals from this process contain truth"✅唯一正确
"95% probability truth is in this CI"❌CI 和 truth 都固定，无概率
"95% of data values fall in CI"❌CI 估 parameter，与数据点无关
"95% of sample means fall in CI"❌CI 是为这一个 sample mean 构造的
"Definitely contains the truth"❌5% 的 CI 是错的

CI 几何性质

  • 更高置信度 c → 更宽 CI
  • 更大 n → 更窄 CI（按 √n 缩小）
  • Bootstrap CI: 用 percentile，不依赖 normal
  • CLT CI: 公式法，只对 mean/proportion 适用

2. Distributions / Sampling / CLT ⭐⭐⭐

三联术语区分（fill-in-the-blank 常考）

术语是什么怎么得
Probability distributionrandom variable 各取值的理论概率从模型推导
Empirical distribution一个 sample 中各值的观测比例从数据频次
Sampling distribution一个 statistic（如 sample mean）的概率分布从重复抽样推导/simulate

大数定律：sample size ↑ → empirical → probability distribution。

parameter vs statistic / population vs sample

populationsample
来源整体抽出的子集
"summary"parameter（固定但未知）statistic（随机，可算出）
例子μ, σ, px̄, s, p̂

SRS（Simple Random Sample）

  • 所有 size-n subset 概率相等
  • without replacement（默认）
  • 注意：with replacement 不算 SRS

Standardization

z = (x − μ) / σ z 无单位 线性变换

z 表示"距均值几个 SD"。z > 0 在均值上方。

Chebyshev 不等式（适用任意分布）

距均值 z 个 SD 内的比例 至少：

z至少占比分数形式
275%3/4
388.8...%8/9
493.75%15/16
596%24/25
通用1 − 1/z²
Chebyshev 给的是下界。题问"至少占多少" → Chebyshev。问"大约占多少且分布近似 normal" → 用 68/95/99.7。

Normal 68/95/99.7（仅当分布近似 normal）

距均值 ±1/±2/±3 SD 内 ≈ 68% / 95% / 99.7%。

反查 z：stats.norm.cdf(z) 给 P(Z ≤ z)；stats.norm.ppf(p) 给 z 使得 P(Z ≤ z) = p。

Square Root Law（核心）

SD of sample mean = (population SD) / √n

n × 4 倍 → SD 变 1/2；n × 100 → SD 变 1/10。

CLT 适用 checklist（"Yes/No, due to CLT" 必看）

  1. Sample 是 random（SRS 或 with replacement 都 OK）
  2. n 足够大（经验上 ≥ 30；二项 np≥10、n(1-p)≥10）
  3. 我们关心的是 sample mean 或 sample proportion

三条全满足 → sample mean 的 sampling distribution 近似 normal，不管 population 是什么分布。

Wi26 陷阱："约 95% 的 sample means 落在 pop mean ± 2 pop SDs" → 官方答 Yes, due to CLT（解读为 2 SEs = 2 SD-of-mean，不是 2 pop SDs；课程在这道题上把 "2 SDs" 当 SEs 算）。

3. Sample size ⭐⭐

公式

n ≥ ( 4 · SD / w )² w = CI 总宽度

来源：95% CLT CI 总宽 = 2 · 2 · SD/√n = 4·SD/√n。令 ≤ w 解出 n。

w 是总宽度（high − low），不是半宽度 margin。题目说 "within ±0.05" → margin = 0.05 → w = 0.10。

0/1 数据特例

  • SD 上界 = 0.5（当 p = 0.5 时取到）
  • 所以 0/1 数据：n ≥ (4 · 0.5 / w)² = (2/w)² 保守版
  • Proportion as mean：0/1 取值的均值 = 1 的比例，所以 proportion 可以用 mean 的全部工具（CLT、bootstrap、sample size）

已知 n, w 反推 SD（少见但 Wi26 考过）

SD ≤ w · √n / 4

4. Hypothesis testing ⭐⭐⭐

5 步流程

  1. 写 Null（"无差异 / 是某模型"）和 Alt（"有差异 / 不是某模型"）
  2. 选 test statistic（一个数字总结 sample 偏离 null 的程度）
  3. 计算 observed test statistic（从真实 sample）
  4. Simulate under null 多次 → 得 null 下 test statistic 的分布
  5. p-value = 在 null 下看到"observed 或更极端"的比例

p-value 决策

p结论常用 α
p ≤ 0.01"highly statistically significant"，reject null0.01
0.01 < p ≤ 0.05"statistically significant"，reject null0.05
p > 0.05"fail to reject null"—
永远不写 "accept the null"。fail to reject ≠ null 为真，只是证据不够。

Test statistic 选 abs vs 不带 abs

Alt 方向statisticp-value 取
单侧（A > B 或 A < B）带方向（如 A − B 或 mean − null_mean）单侧 tail
双侧（A ≠ B）abs（如 |A − B|）右尾（因为 abs ≥ 0）

simulate under null 的常见形态

  • 模型已知：np.random.multinomial(n, [p1, p2, ...])
  • simple yes/no：np.random.choice([0, 1], n, p=[1-p, p])
  • 两组比较：见 §5 permutation

4b. TVD（Total Variation Distance）⭐⭐

公式

TVD(p, q) = 0.5 · Σ | p_i − q_i | = np.abs(p − q).sum() / 2

p, q 是两个 categorical distribution（每个 = 1）。

性质

  • 范围 [0, 1]。0 = 同分布、1 = 不相交支撑
  • 只用于 categorical 数据。数值变量请用 mean / median 差作为统计量
  • 典型用法：observed 分布 vs hypothesized 模型分布

典型场景（Lec 20-21 + Alameda jury）

  • 陪审团种族分布 vs 该县人口分布是否相符 → TVD as test stat
  • Null：陪审团是 SRS from county → simulate multinomial → 算每次 TVD → 比 observed TVD 看 p-value
出现"smoking 与 birth weight 相关吗" → birth weight 是数值 → 不能用 TVD，应用 mean 差 + permutation test。

5. Permutation testing ⭐⭐⭐

什么时候用

比较两组 sample，问"它们是否来自同一分布"。Null = "组别 label 与 outcome 无关"。

流程

  1. 合并两组为一个总池子（保留 outcome 列，丢弃原 label）
  2. Shuffle group labels（np.random.permutation），分配回去
  3. 重新计算 test statistic（如 group A mean − group B mean）
  4. 重复多次 → 比 observed
shuffled_labels = np.random.permutation(df.get('group'))
shuffled = df.assign(shuffled_group=shuffled_labels)
group_means = shuffled.groupby('shuffled_group').mean()
stat = group_means.get('outcome').iloc[0] - group_means.get('outcome').iloc[1]

Permutation vs Bootstrap（必会区分）

PermutationBootstrap
用途hypothesis test（比较两组）estimate 参数 + CI
抽样without replacement（shuffle）with replacement
对象合并后的整池原 sample
结果p-valueCI（percentile）

典型例题

  • Smoking & baby weight：母亲吸烟 vs 不吸烟两组婴儿体重 → mean 差作 stat
  • Deflategate：Patriots vs Colts 球压差异 → mean 差作 stat
  • A/B test：网站新旧版本转化率比较 → proportion 差作 stat

7. Regression 三件套 ⭐⭐⭐

Correlation r

r = mean( z_x · z_y ) z_x = (x − mean_x) / SD_x
  • r ∈ [−1, 1]，无单位
  • 对称：r(x, y) = r(y, x)
  • 对线性变换不变（换单位、放缩、平移都不变 r）
  • 只衡量线性 association。r = 0 不代表无关（可能 parabola）
  • 对 outlier 极敏感：一个点能把 r 从 0 推到 1

Regression line in standard units

predicted_z_y = r · z_x 过 (0, 0)，slope = r

Regression to the mean：|r| < 1 → predicted z_y 比 z_x 更靠近 0（极端 x 对应不那么极端的 y）。

Regression line in original units（必会公式）

m = r · SD_y / SD_x
b = mean_y − m · mean_x
predicted_y = m · x + b

必过点：(mean_x, mean_y)（代 x = mean_x → predicted_y = mean_y）。

SU ↔ 原单位完整方程

(predicted_y − mean_y) / SD_y = r · (x − mean_x) / SD_x

单位换算速查表

把 x 换单位（× k，+ c）把 y 换单位（× k', + c'）
r：不变r：不变
SD_x：× |k|（平移不变）SD_y：× |k'|
mean_x：× k + cmean_y：× k' + c'
m（slope）：× k' / k
b（intercept）：要从 m, mean_x, mean_y 重算

Single outlier 警告

所有 regression 题第一步问"有 outlier 吗"。outlier 可以把 r 从 0 拉到 1，或反过来。

8. Residuals + Prediction ⭐⭐

核心公式

residual = actual − predicted 方向绝不能反

正 residual = 真实点在回归线上方。

RMSE（least squares 最小化的对象）

RMSE = sqrt( mean( residual² ) )

Regression line 是所有可能直线中 RMSE 最小的那条。

Residual plot 4 类 pattern

residual plot 形状结论
无模式（patternless 散布在 0 周围）✅ 线性模型合适
曲线 pattern（U 形 / 倒 U）真关系是非线性，应换模型
漏斗形（方差随 x 变化）heteroscedasticity：误差不等方差
趋势倾斜（系统偏一边）所选直线斜率/截距错了

关键性质

  • regression line 的 residuals mean = 0
  • regression line 的 residuals 与 x 的 r = 0（无线性相关）
  • Anscombe's quartet / Datasaurus dozen：r、mean、SD、line 完全相同，但散点图截然不同 → 必须画图

Prediction Interval（bootstrap 法）

boot_preds = np.array([])
for i in np.arange(reps):
    resampled = df.sample(df.shape[0], replace=True)   # 重抽 (x, y) 整对
    m = slope(resampled, 'x', 'y')
    b = intercept(resampled, 'x', 'y')
    boot_preds = np.append(boot_preds, m * x_new + b)
ci = np.percentile(boot_preds, [2.5, 97.5])

几何性质：prediction interval 在 x = mean_x 处最窄，离 mean_x 越远越宽（draw_many_lines 几何观察）。

Reverse regression（y on x ≠ x on y）

  • 由 x 预测 y 的直线 ≠ 由 y 预测 x 的直线（除非 |r| = 1）
  • 角色互换后：m_new = r · SD_x / SD_y（公式 SD 反过来）
  • 两条都过 (mean_x, mean_y)

Best line ≠ Best model

RMSE 最小的直线可能仍然是糟糕的拟合（如真关系是 quadratic）。Residual plot 是检查工具。

9. DataFrame 速查（babypandas）⭐⭐⭐

babypandas ≠ pandas：没有 df['col'] 方括号取列、没有 df.size()、没有 inplace=True。所有操作返回新对象。

索引 / 取数

方法语义返回
df.get('col')取一列Series
df.get(['col1', 'col2'])取多列DataFrame
df.loc[label]按 index label 取行Series（一行）/ DataFrame（多行）
df.iloc[pos]按整数位置取行Series（一个）/ DataFrame（多个）
df.iloc[i, j]第 i 行第 j 列单值标量
df.shape[0]行数（替代 size）int
df.shape[1]列数int

重塑 / 列操作

  • df.assign(new=expr)：加新列，必须重新赋值 df = df.assign(...)
  • df.set_index('col') / df.reset_index()
  • df.sort_values('col', ascending=False)：默认升序，加 .iloc[0] 取极值
  • df.drop(columns='col') 或 .drop(index=label)
  • df.take([i1, i2, ...])：按位置取多行

groupby

df.groupby('col').sum()       # 单列分组
df.groupby('col').mean()
df.groupby('col').count()     # 每组多少行
df.groupby(['a', 'b']).mean() # 多列分组 → MultiIndex
# 链式取极值：
df.groupby('city').mean().sort_values('price', ascending=False).iloc[0]

merge（Tier 1 必考：行数预测）

df1.merge(df2, left_on='ka', right_on='kb')

默认 inner join：只保留两边 key 都有的。

行数规则：若 key k 在 df1 中出现 m 次、在 df2 中出现 n 次，则结果中 k 出现 m × n 次。

速查：

  • 两边都 unique → 结果行数 ≤ min(len1, len2)
  • 一边有 dup → 结果会复制行
  • 双方都有 dup → Cartesian product 爆炸
  • 没匹配的 key → 消失（inner 默认）

10. 布尔过滤 + 字符串 ⭐⭐⭐

布尔过滤

df[df.get('age') > 18]                            # 单条件
df[(df.get('age') > 18) & (df.get('city') == 'SD')]   # 两个条件，&、括号

11. Density histogram ⭐⭐

Height × Width = Proportion Σ(Height × Width) = 1

关键性质

  • 所有 bar 面积之和 = 1（不是高度和！）
  • bin [a, b) 半开，最后一个 bin 右端闭合。一个值只算进一个 bin
  • Y 轴单位：per unit（如 per dollar、per year）
  • Rebin：合并两个 bin → 新 width 加倍，新高度按 Σ proportion ÷ 新 width 重算（不是取平均高度！）

典型问法

  • 给一个 bin 的 height 和 width，问 proportion → 直接相乘
  • 给 proportion 和 width，问 height → 相除
  • 问"高于 X 的比例" → 找 bin 边界（注意半开！）

构图 API

df.plot(kind='hist', y='col', density=True,
        bins=np.arange(low, high+step, step),
        ec='w')

12. 概率规则 ⭐⭐⭐

基本规则

规则公式前提
Addition (mutex)P(A ∪ B) = P(A) + P(B)A, B 互斥
Addition (general)P(A ∪ B) = P(A) + P(B) − P(A ∩ B)任意
Multiplication (indep)P(A ∩ B) = P(A) · P(B)A, B 独立
Multiplication (general)P(A ∩ B) = P(A) · P(B | A)任意
ComplementP(A^c) = 1 − P(A)任意
ConditionalP(A | B) = P(A ∩ B) / P(B)P(B) > 0

独立性判断

A, B 独立 ⟺ P(A | B) = P(A) ⟺ P(A ∩ B) = P(A) · P(B)

"At least one" 套路

P(at least one) = 1 − P(none)

例：抛 n 次硬币至少一次正面 = 1 − (1/2)ⁿ。

with vs without replacement

13. Simulation 模板 ⭐⭐⭐

results = np.array([])
for i in np.arange(reps):           # 一般 10_000
    # 模拟一次实验
    outcome = ...                        # 用 np.random.choice 等
    results = np.append(results, outcome)    # ← 必须重新赋值！

# 计数 / 概率估计
estimate = np.count_nonzero(results == target) / reps
必须重新赋值：arr = np.append(arr, x)。np.append(arr, x) 不修改原数组！

常见统计

  • 事件发生概率：np.count_nonzero(results == event) / reps
  • 大于阈值的比例：np.count_nonzero(results > threshold) / reps
  • 样本均值：results.mean()
  • histogram 看分布：bpd.DataFrame().assign(s=results).plot(kind='hist', density=True)

14. np.random 三件套 ⭐⭐

函数返回用途
np.random.choice(arr, size, replace=)抽中的值（array）通用抽样
np.random.multinomial(n, [p1, p2, ...])计数 array从 categorical model 抽 n 个，给每类多少个
np.random.permutation(arr)shuffle 后的 arraypermutation test 打乱 label
df.sample(n, replace=False)DataFrameDataFrame 行抽样（bootstrap）

易混陷阱

  • multinomial 返回的是计数（如 [37, 53, 10]），不是比例。要比例自己除 n
  • choice 默认 replace=True。SRS 要 replace=False
  • df.sample 默认 replace=False。bootstrap 要写 replace=True
  • p= 参数：choice(arr, size, p=probs) 给定每个值的概率

15. .apply 陷阱 ⭐

Series.apply

def categorize(x):
    if x < 18:
        return 'child'
    elif x < 65:
        return 'adult'
    else:
        return 'senior'

df.get('age').apply(categorize)       # ← func 不带括号！
.apply(categorize)，不是 .apply(categorize())。后者会先调用 categorize()（缺参数）报错。

加新列模板

df = df.assign(category=df.get('age').apply(categorize))
