Q1.1 — Wi26

Which column of the pizza DataFrame would be an appropriate index?

(a) "store"   (b) "kind"   (c) "rating"   (d) "neighborhood"   (e) None of these

显示答案 + 解析

答案：(e) None of these

合适的 index 必须是每行唯一的。

  • "store": 同一家店有多个切片，重复 → 不行
  • "kind": 不同店都有 Cheese/Pepperoni → 不行

所以选 None of these。

Q6 — Wi26 (Probability)

At Pizza by Peter, head chefs Bianca and Ella make each pizza, either together or alone. For any given pizza, there's a chance that the chefs mess up. The probabilities are:

  • If Bianca makes a pizza alone: 1/4 chance of not being suitable
  • If Ella makes a pizza alone: 1/3 chance of not being suitable
  • If Bianca tosses dough & Ella bakes: 1/6 chance of not being suitable
  • If Ella tosses dough & Bianca bakes: 1/5 chance of not being suitable

Q6.1: Suppose Ella makes one pizza alone, Bianca makes one pizza alone, and their suitability is independent. What is the probability that at least one of the pizzas is not suitable to serve?

Q6.2: Let A = "Ella worked on the pizza (alone or with Bianca)", B = "the pizza is suitable to serve". Suppose P(A) = 1/4 and P(B) = 3/5. What is P(A or B)?

显示答案 + 解析

Q6.1 答案：1/2

"At least one not suitable" = 1 − P(both suitable)
P(Ella suitable) = 2/3, P(Bianca suitable) = 3/4
P(both suitable) = (2/3)(3/4) = 6/12 = 1/2
P(at least one not suitable) = 1 − 1/2 = 1/2

Q6.2 答案：Not enough information

P(A or B) = P(A) + P(B) − P(A and B)
= 1/4 + 3/5 − P(A and B)
但题目没给 P(A and B)，且 A 和 B 不独立（根据上面 4 个条件概率，P(suitable | Ella involved) 在不同情况下不同），所以无法计算。

陷阱：不要默认事件独立！

Q9 — Wi26 (Simulation — 高频考点)

DoorDash promotion: Ray orders four random pizza slices from pizza. He picks the first two and compares ratings to the other two. He eats only if sum of his two ratings > sum of other two ratings.

Estimate the probability that Ray gets to eat:

repetitions = 1000
count_eats = 0

for i in np.arange(repetitions):
    promo_ratings = np.random.choice(__(a)__, 4, replace=False)
    ray_sum_of_ratings = _____(b)_____
    other_sum_of_ratings = _____(c)_____
    if ray_sum_of_ratings > other_sum_of_ratings:
        count_eats = ____(d)____
prob_eats = _____(e)_____
显示答案 + 解析 
(a) pizza.get("rating")
(b) promo_ratings[0] + promo_ratings[1]   # 或 promo_ratings[:2].sum()
(c) promo_ratings[2] + promo_ratings[3]   # 或 promo_ratings[2:].sum()
(d) count_eats + 1
(e) count_eats / repetitions

这是经典 simulation 模板：

  1. 初始化 counter = 0
  2. for i in np.arange(reps)
  3. 用 np.random.choice 抽样（这里 replace=False 不放回 → 4 个不同 slice）
  4. 判断条件 → counter += 1
  5. 最后除以 reps得到概率估计

注意：(a) 必须是 pizza.get("rating") 而不是 pizza 整个 DataFrame —— np.random.choice 要的是 1D array/Series。

Q10 — Wi26 (One-line expression)

Pizza by Peter awards loyalty points: every third slice you purchase earns 2i points, starting from the 1st slice. The 1st, 4th, 7th and 10th slices earn 2i loyalty points where i is 1, 4, 7, 10 respectively.

Without using +, write a one-line expression that evaluates to the total loyalty points from those four slices.

显示答案 + 解析 
(2 ** np.arange(1, 11, 3)).sum()

解析：

  • np.arange(1, 11, 3) → [1, 4, 7, 10]（步长 3，从 1 到 10）
  • 2 ** ... → numpy broadcasting，逐元素求 2 的幂 → [2, 16, 128, 1024]
  • .sum() → 1170

题目限制不能用 +，所以必须用 .sum()。

🥈 模拟二: Fall 2025 Midterm

来源：Fall 2025 Midterm Exam ｜ 讲师：Janine Tiefenbruck, Peter Chi ｜ 50 分钟

主题：English words 难度数据集

📦 DataFrame schema: words

Indexed by "word". Columns:

  • "freq" (int): Frequency of the word in the texts.
  • "length" (int): Number of letters.
  • "ps" (str): Part of speech, formatted like "('getting', 'VBG')" (string of an ordered pair).
  • "diff" (int): Number of students who marked the word difficult.

First few rows:

word          freq  length  ps                       diff
getting       2     7       ('getting', 'VBG')       0
apathy        1     6       ('apathy', 'JJ')         10
devoured      2     8       ('devoured', 'VBD')      5
identifiable  2     12      ('identifiable', 'JJ')   0
beat          3     4       ('beat', 'NN')           0
Q1 — Fa25 (字符串处理 — Lec 8)

Each value in "ps" is a string formatted like "(word, tag)" with single quotes, e.g. "(getting, VBG)" with the actual format "('getting', 'VBG')". To clean so that "ps" contains only the part of speech tag:

words = words.assign(ps = words.get("ps").apply(fix_ps))

After running, the rows show: VBG, JJ, VBD, JJ, NN.

Which definitions of fix_ps work? Select all that apply.

  1. def fix_ps(x): return x.split("'")[3]
  2. def fix_ps(x): return x.split(",")[1].strip("'")
  3. def fix_ps(x): return x.split(", ")[1].strip("'")
  4. def fix_ps(x): return x.split(", ")[1].strip(")").strip("'")
  5. def fix_ps(x): return x.split(", ")[1].strip("'").strip(")")
显示答案 + 解析

答案：(1), (4), (5) 正确

原始字符串例子："('getting', 'VBG')"

  • (1) ✓ split("'") → ['(', 'getting', ', ', 'VBG', ')']，取 [3] = 'VBG'
  • (2) ✗ split(",") → ["('getting'", " 'VBG')"]，[1] = " 'VBG')"，.strip("'") 只去 '，留下 " 'VBG')" 中的空格和 )
  • (3) ✗ split(", ") → ["('getting'", "'VBG')"]，[1] = "'VBG')"，.strip("'") → "VBG')"（结尾的 ) 还在）
  • (4) ✓ "'VBG')" → strip(")") → "'VBG'" → strip("'") → "VBG"
  • (5) ✓ 顺序换了，"'VBG')" → strip("'") → "VBG')"（注：strip 只从两端去字符，遇到非 ' 就停） → strip(")") → "VBG"

关键：strip(chars) 从字符串两端去除指定字符（不限于一次）。

Q2 — Fa25 (Merge 行数预测)

We have a DataFrame tags indexed 0..N with columns tag, meaning, examples. After running fix_ps, we want to merge words with tags to see the meaning of each word's POS tag.

merged = words.reset_index().merge(__(a)__).set_index("word")

Q2.1: Fill in (a).
Q2.2: Suppose words has 5357 rows and tags has 36 rows. How many rows does merged have?

Q3 — Fa25 (sort_values 可视化推理)

Below is a bar chart and the code:

(words.sort_values(by="diff", ascending=False).take(np.arange(20))
      .plot(kind='barh', y="diff"));

The chart shows 20 words from "scolding" (top, x≈21) to "decades" (bottom, x≈25).

True/False:

3.1: Students found "decades" more difficult than any other word.

3.2: If we removed ascending=False, the chart would show the same info but bars in opposite order top-to-bottom.

3.3: If we changed np.arange(20) to np.arange(15), the chart would show all words that > 15 students found difficult.

显示答案 + 解析

3.1: False（⚠️ 易错）

从 bar chart 可见，有 5 个其他词的 diff 值与 "decades" 相同（最高 diff 值多个词并列）。所以 "decades found more difficult than any other word" 是 False —— 那些并列的词也同样难。

陷阱：看到 "decades" 在 bar chart 底部（diff 最大行）就以为它是唯一最大——但 sort_values 不会因为并列而合并行，"decades" 只是这些并列最大值中恰好排在底部那一行。一定要检查 bar 高度是否真的全部互不相同。

3.2: False

去掉 ascending=False 默认 ascending=True，排出的是不同的 20 个词（最容易的 20 个），不是简单顺序颠倒。

3.3: False

np.arange(15) 取前 15 个最难的词，未必是 diff > 15 的所有词。这个 chart 只显示按排名取的前 N 个，没有按 diff 阈值过滤。

Fill in classify_length_2 to give the same output. Each blank must be a single line.

        __(b)__

  • Way 1: groupby("address").sum() 得到每户的 sum(price) 和 sum(how_many)。avg = sum(price)/sum(how_many) 仅当 price 是该候选的总价才对（题目里 price 是 per piece 的）—— 实际上这里也对，因为分子分母同时被 sum 后比例不变？
    重新审视：题目说 price = cost per piece (单价)。Way 1 算的是 sum(price)/sum(how_many) ≠ total_money/total_pieces. 应该是错的。
    但 Fa24 官方答案给 Way 1 和 Way 5。可能 price 在合并后变成了 piece × price = 总额。需要看 Fa24 上下文。
  • Way 5: x = how_many × price = 总额; sum(x) = total_money; sum(how_many) = total pieces; avg = sum(x)/sum(how_many) ✓

（标准答案：Way 5 是正确的，Way 1 取决于 schema。这题主要练习 groupby 后 .assign 用 ac.get(...).mean() vs 列除法的细节。）

⚡ L9 — Sp24 Q4.1 (Probability — random dialing)

Monkey dials a random 10-digit number (each digit equally likely 0-9). What is P(monkey calls one of your contacts)? Use DataFrame contacts.

显示答案 
contacts.shape[0] / 10**10

解析：总共有 1010 种 10 位数（含 leading zero）。Contacts 中每个 phone number 唯一对应一种。所以 P = (n contacts) / 1010。

⚡ L10 — Sp24 Q8 (.groupby + max via sort)

Find the largest number of people in contacts who share the same birthday (people born on same month-day, regardless of year):

contacts.groupby(__(a)__).__(b)__.get("Phone").__(c)__
显示答案 
(a) ["Month", "Day"]    # 多列 groupby
(b) count()
(c) max()

多列 groupby 把 (Month, Day) 组合作为分组键；count() 数每组人数；最后 max() 找最大组的人数。
