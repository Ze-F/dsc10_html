📋 考点速览

本节是 DSC10 midterm 的核心地基。每一份历年 midterm（6/6）都会反复考察布尔过滤、多条件组合与 groupby 聚合 —— 而且常常和 Lec 4 的 sort_values / iloc / .get 串联成"链式 one-liner"。

本节出现的题型

  • 布尔过滤：df[df.get('col') == value] 选行
  • 多条件组合：&、|、~，每个条件外必须加括号
  • 计数行数 / 求比例：.shape[0] 或 np.count_nonzero(bool_series)
  • groupby 三步法：.get([cols]) → .groupby('cat') → .agg()
  • 找极值：.sort_values(...).get('col').iloc[0] 或 .index[0]
  • 字符串过滤：.str.contains、.str.startswith、.str.split
  • 等价改写：multi-select"哪些写法都对"题型（Wi26、Fa24 反复考）

反复出现的考察模式

  • 给一段 treat[(treat.get('a') == X) & (treat.get('b') == Y)]，问行数 / 列数 / 输出
  • 给一个 groupby('col').count() 结果，问 哪些行被合并了 / row 数何时等于原表行数（→ 当 col 是 index / 唯一 时）
  • 填空 pizza.get([__]).groupby(__).__().sort_values(__).__
  • "select all that apply"：列出 4–5 种链式写法，哪些等价于目标值

📚 知识点

1. 比较运算符产生 Boolean Series（broadcasting）

对 Series 做比较（==、!=、<、<=、>、>=）会逐元素返回一个 bool 类型的 Series，长度与原 Series 一致：

states.get('Region') == 'West'
# State
# Alabama        False
# Alaska          True
# Arizona         True
# ...
# Name: Region, Length: 50, dtype: bool

这种"标量与 Series 比较 → Series"的机制叫 broadcasting（广播）。

2. 布尔过滤（query）

用方括号把一个布尔 Series（或 list / array）传给 DataFrame：保留所有 True 的行。

# 选出 Region 为 West 的所有州
only_west = states[states.get('Region') == 'West']

# 等价改法：先存下条件
mask = states.get('Region') == 'West'
only_west = states[mask]
📌 query 的两步骤心法：
1️⃣ 造一个长度等于行数的 True/False 序列；
2️⃣ 把这个序列传到 df[ ... ] 里。

如果条件没有任何行匹配，结果就是一个 0 行 DataFrame（不会报错）。

3. 多条件 — & / | / ~

符号含义等价（错误！）写法
&AND（两个都为 True）and ❌
|OR（至少一个 True）or ❌
~NOT（取反）not ❌

每个条件外必须加括号，因为 & 和 | 的运算优先级高于 ==：

# ✅ 正确
states[(states.get('Party') == 'Republican') & (states.get('Region') == 'South')]

# ❌ 报错（& 会先和 'Republican'/'South' 结合）
states[states.get('Party') == 'Republican' & states.get('Region') == 'South']

# ❌ 报错（and 不能用于 Series）
# ValueError: The truth value of a Series is ambiguous.
states[(states.get('Party') == 'Republican') and (states.get('Region') == 'South')]

4. .shape — 求行数、求比例

.shape 是属性不是方法，没有括号。返回 (行数, 列数)：

only_rep = states[states.get('Party') == 'Republican']
only_rep.shape       # (31, 6)
only_rep.shape[0]    # 31  —— 行数
only_rep.shape[1]    # 6   —— 列数

# 求 Republican 比例
only_rep.shape[0] / states.shape[0]    # 0.62
统计行数的几种等价写法（select-all-that-apply 常考）：
  • df[mask].shape[0]
  • np.count_nonzero(mask) （True=1, False=0）
  • mask.sum()

5. .index 与 .take([...])

midwest_sorted.index           # Index(['Kansas', 'Minnesota', ...])
midwest_sorted.index[0]        # 'Kansas'  —— 注意 index 不是 column！

# .take 按整数位置选行（类似 Series.iloc）
df.take([0, 1, 2])             # 前 3 行
df.take(np.arange(3))          # 等价

要点：.get('State') 在 State 已是 index 时会报 KeyError，这时用 .index。

6. 列操作三件套：.assign / .drop / .get

# 加一列（关键字必须是合法 Python 标识符，所以列名不能有空格）
states.assign(Pop_in_millions=states.get('Population') / 1_000_000)

# 删列
df.drop(columns=['Population'])

# .get 单列名 → Series；list of 列名 → DataFrame（即使 list 只有一个元素）
states.get('Capital City')          # Series
states.get(['Capital City'])        # DataFrame (1 列)
states.get(['Capital City', 'Party'])  # DataFrame (2 列)

7. groupby 三步工作流

「I-G-A」：Isolate columns → Group → Aggregate。

# 第 1 步：选出 grouping 列 + 要聚合的列
# 第 2 步：.groupby(分组列) —— 该列变成 index
# 第 3 步：调用聚合方法
states.get(['Region', 'Population']).groupby('Region').sum()

# 输出：index=Region, 列=Population
#                Population
# Region
# Midwest          68985454
# Northeast        57609148
# South           125576562
# West             78588572

聚合方法：.sum()、.mean()、.median()、.max()、.min()、.count()。

⚠️ babypandas 没有 .size()！groupby 后数每组行数请用 .count()（逐列数 非空，无缺失时每列都等于组的行数）。
顺带一提：DataFrame 顶层也没有 .size 属性。想要整张表的行数，用 df.shape[0]（详见陷阱 5）。

8. groupby 后的常见模式

# 模式 A：找极值对应的 group label —— 用 .index[0]
(states
   .get(['Region', 'Population'])
   .groupby('Region')
   .sum()
   .sort_values(by='Population', ascending=False)
   .index[0])
# 'South'

# 模式 B：取出一个特定 group 的某个聚合值
df.groupby('cat').mean().get('col').iloc[0]

# 模式 C：reset_index() 把 group 列变回普通列
df.groupby('cat').count().reset_index()

9. Series 字符串方法 .str.*

# 子串包含 → bool Series
treat.get('candy').str.contains('Sour')

# 字符串切分 → Series of list
treat.get('address').str.split(',')

# 前缀 / 后缀
contacts.get('Phone').str.startswith('987')
contacts.get('Phone').str.endswith('6789')   # 注意区分 .endswith

这些方法都返回新 Series，不修改原数据。.str.contains 配合布尔过滤用得最频繁。

10. 链式 one-liner（Lec 5 后半的精髓）

# 多步骤合一行；过长时用括号包起来分行
(states[states.get('Region') == 'Midwest']
   .sort_values(by='Land Area', ascending=False)
   .index[0])
# 'Kansas'

⚠️ 常见陷阱与考点

陷阱 1：& vs and
Series 之间的"逻辑与"必须用 &（按位运算符，逐元素）。and 是 Python 内建短路逻辑，会试图把整个 Series 转成单个 bool，触发 ValueError: The truth value of a Series is ambiguous。| vs or 同理。
陷阱 2：忘记给每个条件加括号
& 优先级 高于 ==。a == 1 & b == 2 会被 Python 解析为 a == (1 & b) == 2，结果错或直接报错。每个 == 表达式外必须裹括号。
陷阱 3：groupby 后 grouping column 跑到 index 去了
states.groupby('Region').sum() 之后 'Region' 不再是普通列，而是 index。要用 .index（不带括号）取它，或先 .reset_index() 把它变回列。
陷阱 4：聚合方法对非数字列的怪异行为
.sum() 在字符串列上会把字符串首尾相连（如 "SpringfieldIndianapolis..."）；.mean() 在字符串列上会报错。babypandas 不再像旧版 pandas 那样默默丢列，所以推荐先 .get([需要的列]) 再 .groupby，避免意外。
陷阱 5：.size 在 babypandas 中不存在 —— 但要分清两个语境
课件那句「avoid using .size() which is similar but not the same as .count()」是针对 groupby 场景说的，那里 gb.size() 和 gb.count() 才真的"相似但不同"（前者是每组行数 Series，后者是每组每列非空数的 DataFrame）。

在 DataFrame 顶层，.size 其实跟 .shape 才是一类（都是描述维度的属性，df.size = rows × cols），跟 .count() 关系不大。所以：
  • 想要"DataFrame 有几行" → 用 df.shape[0] ✅（最常用、最干净）
  • 想要"每组有几行" → 用 df.groupby('x').count().get('任一列').iloc[i]（见陷阱 6）
  • 不要写 df.size 或 gb.size() —— babypandas 都没有
陷阱 6：.count() 是逐列数非空
df.groupby('Region').count() 的输出每一列都是该 group 的行数（因为没有缺失值时所有列都一样）。要"取 Region=Midwest 的州数"，可以从任何一列读 .get('Capital City').iloc[0]，但语义上要清楚每列的数字含义相同只是巧合。
陷阱 7：.groupby('col').max() ≠ "找最大那一行"
max 是逐列独立求最大值。课件原话："While it's true that the highest rated movie from the 1990s has a rating of 9.2, that movie is not Unforgiven – instead, Unforgiven is the movie that's the latest in the alphabet among all movies from the 1990s." 要"找最大对应的行"，请用 sort_values。
陷阱 8：.shape 没有括号
.shape 是 attribute（属性），.shape() 会报错。同理 .index、.columns 也不带括号。
陷阱 9：states['West'] 写法不能用
babypandas 中取列必须用 .get('col')，不能用方括号。方括号 df[...] 只用于布尔过滤。

▍ Spring 2025 · Problem 1.1 + 1.2 （Lec 5 — 单条件查询）

背景：tariffs 有 50 行，列 Country、Tariffs Charged to USA、Reciprocal Tariff（int 表示百分比）。

1.1 写一行代码，求 Reciprocal Tariff 最高的国家名。

1.2 写一行代码，求 Reciprocal Tariff 高于 30% 的国家数量。

显示答案 
# 1.1 —— 排序后取 index 第一个
tariffs.sort_values(by='Reciprocal Tariff', ascending=False).get('Country').iloc[0]

# 或者：先 set_index('Country') 再取 index[0]
tariffs.sort_values(by='Reciprocal Tariff', ascending=False).get('Country').iloc[0]

# 1.2 —— 布尔过滤后数行数
tariffs[tariffs.get('Reciprocal Tariff') > 30].shape[0]

中文解析：

  • 1.1 是"找最大对应的行"经典模式：sort_values 降序后用 .iloc[0] 取第一行的某列值。
  • 1.2 是"过滤 + 数行"经典模式：df[mask].shape[0]。等价写法包括 (tariffs.get('Reciprocal Tariff') > 30).sum() 和 np.count_nonzero(tariffs.get('Reciprocal Tariff') > 30)。

▍ Winter 2026 · Problem 2 （Lec 5 — 布尔过滤 + max + 等价改写）

填空使表达式得到一个 float，是所有 ingredient 数大于 10 的 pizza 中的最高 rating：

pizza[ ___(a)___ ].___(b)___

2.2 哪些写法可以填到 (b)？多选：

  1. get("rating").apply(max)
  2. get("rating").max()
  3. sort_values("rating", ascending=False).get("rating").iloc[0]
  4. sort_values("rating", ascending=True).get("rating").iloc[-1]
  5. None of the above
显示答案 
# (a)
pizza.get("num_ingredients") > 10

# (b) 正确选项: B、C、D
# B: .get("rating").max()       —— 直接对 Series 取 max
# C: 降序后 iloc[0] —— 第一个就是最大
# D: 升序后 iloc[-1] —— 最后一个就是最大

# A 错误：apply(max) 会把 max 函数应用到每个元素（每个 float 上），
#         单个 float 取 max 是它自己，结果只是返回原 Series，不是单个 float

中文解析：

  • (a) 标准布尔过滤——筛出 ingredient 数大于 10 的 pizza。
  • (b) 这种"找最大值的等价写法"是 Wi26 / Fa24 高频题型。注意 apply(max) 是对每个元素应用 max——而每个元素本身是一个 float，max(float) 不合法，实际会出问题，是错误选项。
  • 背心法："取 Series 最大"四种写法：.max() / 降序 .iloc[0] / 升序 .iloc[-1] / 排序后 index 操作。

▍ Fall 2024 · Problem 3 （Lec 12 / Lec 5 — .str.contains + & + np.count_nonzero）

背景：treats 有 address 和 candy 两列。

street = treats.get("address").str.contains("Street")
sour   = treats.get("candy").str.contains("Sour")

3.1 street 是什么类型？

3.2 np.count_nonzero(street & sour) > sour.sum() 是什么？

显示答案 
# 3.1 —— Series（dtype=bool 的 Series，不是 bool 标量）

# 3.2 —— False
# np.count_nonzero(street & sour)  数同时满足两个条件的行
# sour.sum()                       数仅满足 sour 的行
# 同时满足两个条件 ≤ 仅满足 sour 的行 → 不可能 >
# 所以结果是 False

中文解析：

  • 3.1 关键：.str.contains 作用于 Series，返回的是布尔 Series（不是单个 bool）。这是高频选择题。
  • 3.2 关键洞察：A & B 的 True 数永远 ≤ B 的 True 数（交集大小不大于子集）。所以 np.count_nonzero(street & sour) ≤ sour.sum()，不可能大于。注意 True/False 作为整数写就是 True / False，要按"恰好如何在 Python 中显示"来回答。
  • 顺带：np.count_nonzero(bool_series) = bool_series.sum() = 满足条件的行数，是三种等价写法。

▍ Spring 2024 · Problem 3 （Lec 5 — 字符串后缀过滤）

写一个 Python 表达式，得到一个只包含电话号码以 "6789" 结尾的 contacts 的 DataFrame。
注意：不许用 slicing；只能用本课程教过的方法。

显示答案 
contacts[contacts.get("Phone").str.endswith("6789")]

# 或者用 .str.contains 也可以（但若有 6789 出现在中间会误中），更精确用 endswith

中文解析：

  • .str.endswith(suffix) 是 Series 字符串方法，返回布尔 Series。
  • 把这个布尔 Series 传到 contacts[ ... ] 里就完成过滤。
  • 区分：.str.contains("6789") 会匹配中间带 6789 的（如 "987-6789-123"）；要求严格以 6789 结尾用 endswith。
  • .str.startswith 同理。

▍ Winter 2026 · Problem 5 （Lec 5 — 数行数的多种等价写法）

找出 gluten-free pizza slice 总数的所有正确写法（多选）：

  1. pizza.set_index("store").get("gluten_free").sum()
  2. pizza.get(["gluten_free", "store"]).groupby("store").sum()
  3. pizza[pizza.get("gluten_free")].get("gluten_free").count()
  4. pizza[pizza.get("gluten_free")].shape[0]
  5. pizza.set_index("store").get("gluten_free").count()
  6. pizza.get("gluten_free").shape[0]
显示答案 
# 正确：A、C、D
# A：bool 列 .sum() → True=1 False=0，求和就是 True 数 ✅
# C：先用 bool 列过滤（True 行保留），再 .count() ✅
# D：先用 bool 列过滤，再 .shape[0] ✅

# 错误：
# B：返回 DataFrame（按 store 分组求和），不是单个数字
# E：.count() 数 ALL 行（gluten_free 列没有缺失），返回总行数 ≠ True 数
# F：返回总行数（不管 True/False）

中文解析：

  • 这题集中考"如何数 True 行"。四种正确写法：(1) bool 列直接 .sum()；(2) 用 bool 列过滤后 .shape[0]；(3) np.count_nonzero(bool)；(4) (bool).sum()。
  • 关键陷阱：.count() 数的是所有非空行，不是 True 行——所以 E 错。
  • F：.shape[0] 在没过滤的 Series 上等于全部行数，也错。

▍ Winter 2025 · Problem 2.3 （Lec 5 — groupby + count + reset_index 后查询）

背景：counts 是经过 with_affordability.groupby(["Dining Hall", "Affordability"]).count().reset_index() 后再 .assign(Count=...) 得到的 DataFrame，恰好有列 "Dining Hall"、"Affordability"、"Count"。

已知 "The Bistro" 是 "Cheap" 类菜品最多的 dining hall。哪些表达式必然等于 The Bistro 的 Cheap 菜品数？多选：

  1. counts.sort_values(by="Count", ascending=False).get("Count").iloc[0]
  2. counts.get("Count").max()
  3. (counts[counts.get("Affordability") == "Cheap"].sort_values(by="Count").get("Count").iloc[-1])
  4. counts[counts.get("Dining Hall") == "The Bistro"].get("Count").max()
  5. counts[(counts.get("Affordability") == "Cheap") & (counts.get("Dining Hall") == "The Bistro")].get("Count").iloc[0]
显示答案 
# 正确：C、E
# C：先过滤 Cheap，按 Count 升序后 iloc[-1] = 该子集的最大 Count
#    = 所有 Cheap 类中最多那个 = The Bistro 的 Cheap 数 ✅
# E：精确两条件过滤到唯一一行，iloc[0] 取该 Count ✅

# 错误：
# A：可能取到的是 Moderate 类某 hall 的 Count（不一定是 Cheap）
# B：counts 全表 Count 最大值（同上原因，可能不是 Cheap）
# D：The Bistro 在 Moderate / Expensive 类的 Count 可能更大

中文解析：

  • 这题考"先 groupby reset_index，再写正确的过滤语句"。
  • 关键观察：表里每行是一个 (Dining Hall, Affordability) 组合。所以"取 The Bistro × Cheap 的那一行"必须同时锁定两个条件。
  • A、B、D 都可能跑到错误的 (Hall, Affordability) 组合上去。
  • C 妙在"先按 Affordability 过滤再求最大"——既然题目说 Bistro 是 Cheap 最多的，那 Cheap 子集中的最大 Count 就是 Bistro 的。

▍ Winter 2026 · Problem 4 （Lec 5 — groupby 链式填空求"最佳邻里"）

填空使下面表达式 evaluates 到拥有"最高平均 rating" 的 neighborhood 名字（字符串）：

pizza.get([___(a)___]).groupby(___(b)___)
                       .___(c)___.sort_values(__(d)__, ascending=False)
                       .___(e)___
显示答案 
# (a) ["neighborhood", "rating"]   —— 只保留这两列
# (b) "neighborhood"                —— 按 neighborhood 分组
# (c) mean()                        —— 求平均
# (d) "rating"  或  by="rating"     —— 按 rating 排序
# (e) index[0]                      —— 取 index 第一个 = neighborhood 名字

中文解析：

  • 这是 groupby 三步法（IGA）的标准填空。
  • (a) 先 isolate 用得到的列（grouping 列 + 聚合列），可以避免无意义的字符串列被 mean() 报错。
  • (c) "平均 rating" → .mean()。
  • (e) groupby 后 grouping 列变成 index，所以取 neighborhood 名字用 .index[0]，不能用 .get('neighborhood')（已经不是 column 了）。
  • 另一种等价写法：.reset_index().get('neighborhood').iloc[0]。

▍ Fall 2024 · Problem 8 （Lec 5 / Lec 9 — groupby 何时保留行数）

选择填法使表达式为 True：

treat.groupby(______).mean().shape[0] == treat.shape[0]
  1. "address"
  2. "candy"
  3. "neighborhood"
  4. ["address", "candy"]
  5. ["candy", "neighborhood"]
  6. ["address", "neighborhood"]
显示答案 
# 答案：D ["address", "candy"]
# 原因：treat 每行 = "某地址某种糖"。多个糖来自同一地址，但每行的 (address, candy) 组合是唯一的。
# 用 (address, candy) 这两列 groupby 时，每个组只有 1 行 → 输出行数 = 原表行数。
# A 错：多个候选每地址有多种糖
# B 错：同种糖在多家发
# C 错：同 neighborhood 多家
# E 错：(candy, neighborhood) 不是唯一 —— 同 neighborhood 多家发同一种糖
# F 错：(address, neighborhood) 不唯一 —— 同地址有多种糖（neighborhood 重复）

中文解析：

  • 这题考"groupby 输出行数 = ?"。规律：当被分组的列组合是唯一识别每一行的"key"时，行数不变。
  • 记住：groupby 后行数 = 该列（组合）的不同值的个数。
  • 题目隐含：address 和 candy 一起能唯一标识每行，因为题目数据每行是"某地址 给的 某种糖"。

▍ Fall 2024 · Problem 11 （Lec 5 — groupby + assign 多步推理）

背景：tot 是 trick × treat 的 merged DataFrame，含 address、candy、how_many、price 列。要计算每个地址的 average cost per piece（每块糖平均花费），结果 DataFrame indexed by address，含列 avg_cost。

哪些"Way"是正确的？多选（题目中 Way 1–5 见原 PDF；这里讲思路）。

显示答案 
# 关键：每块糖平均花费 = 总花费 / 总块数 = sum(price * how_many) / sum(how_many)
# 注意：每行的 price 是单价，how_many 是块数。所以总花费需要 price * how_many。

# 正确做法骨架：
ac = tot.assign(x = tot.get("price") * tot.get("how_many"))   # 每行的总花费
ac = ac.groupby("address").sum()                              # 每地址的总花费 + 总块数
ac = ac.assign(avg_cost = ac.get("x") / ac.get("how_many"))   # 平均 = 总花费/总块数
ac = ac.get(["avg_cost"])

# 这正是 Way 5 的写法 → 正确答案：Way 5

# Way 1 错：直接 groupby.sum() 后 price/how_many —— 把单价加起来再除是错的
# Way 2 错：先 price/how_many 得"单块花费"再 groupby.sum() 再 mean()，
#          mean 在已经 sum 过的 DataFrame 上无意义
# Way 3, 4 错：要么忘了 *how_many，要么忘了用 sum 的总块数

中文解析：

  • 核心数学：average cost per piece 是一个加权平均——按糖块数加权。
  • 做法：先在原表上算每行的 price * how_many（行级总花费），groupby sum 后再除以总块数。
  • 陷阱：很多人会错以为"先算每行 price/how_many 再 mean"——但那是把每行视为同等权重，没有按糖块数加权。
  • 这种"求加权平均"题型在 DSC10 反复出现。记住：sum / sum，不要 mean！

▍ Winter 2026 · Problem 3 （Lec 5 — numpy fancy indexing 类比 DataFrame query · NEW CONCEPT）

背景：pizza10 = pizza.take(np.arange(10)) 是前 10 行。考虑：

positions = np.arange(pizza10.shape[0])
result    = positions[pizza10.get("rating") > 4].sum()

3.1 result 代表什么？

3.2 给定数据描述（10 行 rating 列：3.7, 4.0, 3.2, 4.5, 3.7, 4.9, 3.8, 4.6, 4.2, 2.5），result 等于多少？

显示答案 
# 3.1 答案：The sum of the index positions from pizza10 for pizzas with rating > 4
# 解释：positions = [0,1,2,...,9]，bool mask 选出 rating>4 的位置（[1,3,5,7,8]），
#       这些数字相加 = 0+1+...+9 中 rating>4 那些的位置和

# 3.2 答案：23
# rating > 4 的位置是: 1 (4.0)? NO > 4 严格大于
# 实际：4.0 不大于 4。rating > 4 的: 4.5(pos 3), 4.9(pos 5), 3.8 NO, 4.6(pos 7), 4.2(pos 8)
# 1(no, 4.0=4)
# 等等，4.0 严格大于 4 是 False；4.5 → 3 ✅；4.9 → 5 ✅；4.6 → 7 ✅；4.2 → 8 ✅
# 还有？回看：3.7, 4.0, 3.2, 4.5, 3.7, 4.9, 3.8, 4.6, 4.2, 2.5
# 严格 > 4：只有 4.5, 4.9, 4.6, 4.2 → 位置 3, 5, 7, 8
# Sum = 3 + 5 + 7 + 8 = 23 ✅

中文解析：

  • 这是 W26 新考的题型——numpy fancy indexing 类比 DataFrame query。
  • 关键观察：positions[mask] 和 df[mask] 思想完全一样——把布尔 mask 应用到数组上保留 True 位置。
  • positions = np.arange(10) 就是 [0,1,...,9]，再用 mask 选出 True 的元素。
  • 这种"位置数组 + bool mask + sum"的搭配，可以用来求"满足条件的位置之和"——是新颖的考点。
  • 陷阱：4.0 > 4 是 False（严格大于），不要把它算进去。

▍ Fall 2024 · Problem 13 (Extra Credit) （Lec 5 — 双 groupby 语义解读）

定义 double = treat.groupby("candy").count().groupby("address").count()。
已知 double.loc[1].get("how_many") 等于 5。下列哪些解读正确？多选：

  1. 有 5 个 house 各自只发 1 种糖。
  2. 有 5 种糖各自只被 1 个 house 发。
  3. 只有 1 个 house 发 5 种糖。
  4. 只有 1 种糖被 5 个 house 发。
显示答案 
# 答案：B  ("There are five types of candy that are each being given out by only one house.")
#
# 推导：
# 第 1 步: treat.groupby("candy").count()
#   → index = candy；每行 = "这种糖出现在多少行 = 多少 (house, candy) 组合"
#   → 因 row 表 (house × candy)，"这种糖出现的行数" = 发这种糖的 house 数
#   → 列 'address' 的值就是 "发该糖的房子数"
#
# 第 2 步: 再 .groupby("address").count()
#   → 把第 1 步的 'address' 列（=发糖的房子数）当作分组键
#   → index = address（数字，意为"几家发这种糖"）
#   → 各列值 = "有多少种糖落在这一组"
#
# double.loc[1].get("how_many") = 5
#   → "address (= 发糖房子数) 这一类 = 1" 的组里，有 5 种糖
#   → 也即：有 5 种糖恰好只被 1 家发 → 选 B

中文解析：

  • 这是高级题，考察 groupby 后列的含义会变——原本是"address"（地址），groupby count 后"address"列变成"该糖出现的地址数"。
  • 第二次 groupby 把这个"地址数"当分组键，结果 index=1 表示"恰好被 1 家发"的那一类。
  • 这种题最有效的方法：每一步都问自己"现在 index 是什么？列代表什么数？"。
