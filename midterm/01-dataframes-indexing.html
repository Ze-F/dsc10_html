常见出题模式

  1. 「合适的 index 选择」选择题 — 通常是 Q1，要求选出"unique identifier 列"。
  2. 等价表达式判断 — 给若干 .iloc / .loc / .sort_values / .set_index 链式表达式，判断哪些等于某个具体值（"select all that apply"）。
  3. 写一行表达式取极值 — "Write an expression that evaluates to the address with the highest …" 通常是 .sort_values(by=, ascending=False).get('col').iloc[0]。
  4. 填空 .loc[label] / .iloc[pos] — 给函数模板，让你填出按 label 还是按整数位置访问。
  5. 计算行数 — 用 .shape[0]、布尔过滤后 .shape[0]、.count() 等多种 idiom。
  6. babypandas vs pandas 区别 — 题目假设 import babypandas as bpd，没有方括号直接取列。

📚 知识点

1. babypandas vs pandas

UCSD 自己开发的 babypandas 是 pandas 的简化子集，目的是为了减少初学者犯错。

特性babypandas (bpd)pandas
取列df.get('col') ✅df['col'] ✅ df.col ✅
修改原对象❌ 永远返回新 DataFrame✅ 支持 inplace=True
加列只能用 df.assign(...)可用 df['new'] = ...
错误信息更友好原生
 
# DSC10 标准导入
import babypandas as bpd
import numpy as np

states = bpd.read_csv('data/states.csv')

2. bpd.read_csv() — 读入 CSV

  • 路径是相对于当前 notebook的字符串。
  • 返回一个 DataFrame。
  • 默认的行 index 是 0, 1, 2, ...（整数序列）。
states = bpd.read_csv('data/states.csv')
# 50 rows × 6 columns: State, Region, Capital City, Population, Land Area, Party

3. DataFrame 结构：columns / rows / index

  • 列 (column)：每列像一个 array，有相同的数据类型。列名是字符串。
  • 行 (row)：每行有一个 row label（粗体显示）。
  • index：所有 row label 的集合。index 不是一个普通的列！
⚠️ 关键：刚 read_csv 进来时，index 默认是 0,1,2,...,n-1（整数标签），这时整数位置和 label 是相同的（容易混淆 .iloc 和 .loc，但不能依赖这个巧合）。

4. .get('col') vs .get(['col1','col2']) — 极重要！

表达式返回类型用途
df.get('Population')Series（一维，带 index）取单列做运算 / 配 .iloc / .loc / .mean()
df.get(['Population'])DataFrame（仍二维，只有 1 列）保持表格形式，方便接 plot / merge
df.get(['Population', 'State'])DataFrame选取多列
 
states.get('Population')          # Series
states.get(['Population'])        # DataFrame, 1 column
states.get(['Population','State']) # DataFrame, 2 columns

type(states.get('Population'))    # babypandas.bpd.Series

5. Series vs DataFrame

  • Series：像 numpy array，但带 index。支持算术运算（element-wise，需 index 对齐）。
  • DataFrame：二维表格。每个列是一个 Series。
  • 常用 Series 方法：.min()、.max()、.mean()、.median()、.sum()、.count()、.describe()、.iloc[]、.loc[]、.apply(func)、.str.contains()、.str.split()。
# 两个 Series 相除（element-wise）
density = states.get('Population') / states.get('Land Area')
type(density)   # Series

6. df.assign(NewCol=values) — 加列

  • 语法：df.assign(列名=值)
  • 列名不能加引号（它是一个 keyword argument）。
  • 值可以是 Series、numpy array 或 list（长度需匹配）。
  • 返回新 DataFrame，不会修改原 df！必须重新赋值才会"生效"。
# ❌ 错误：原 states 不会被改变
states.assign(Density=states.get('Population') / states.get('Land Area'))
states  # 还是 6 列！

# ✅ 正确：重新赋值
states = states.assign(Density=states.get('Population') / states.get('Land Area'))
# 现在 states 有 7 列

7. 列统计方法

方法返回说明
.min()scalar最小值
.max()scalar最大值
.mean()scalar平均值
.median()scalar中位数
.sum()scalar求和
.count()scalar (int)非空元素个数
.describe()Series一次性给出 count/mean/std/min/25%/50%/75%/max
 
states.get('Density').max()       # 1263.12
states.get('Density').mean()      # 206.55
states.get('Density').describe()  # 一个 8 行的 Series

8. df.sort_values(by='col', ascending=False) — 排序

  • 按某列排序，返回新 DataFrame。
  • 默认 ascending=True（升序，从小到大）。
  • 降序必须用 ascending=False，且必须使用 keyword argument 形式（sort_values(by='Density', False) 会 SyntaxError，因为位置参数不能跟在关键词参数后面）。
  • 排序后 row label 不变（即原先的 index 标签会跟着走，所以 .iloc[0] 给的是排序后的"第一行"，而不是原 row 0）。
ordered = states.sort_values(by='Density', ascending=False)
# 第一行就是密度最高的那个州（New Jersey）
ordered.get('State').iloc[0]   # 'New Jersey'

9. .iloc[i] vs .loc[label] — 核心区别

方法用什么取行例子负数索引
.iloc[i]整数位置（从 0 开始计数，跟数组一样）s.iloc[0] 第 1 个；s.iloc[-1] 最后 1 个✅ 支持
.loc[label]row label（行标签的值）s.loc['California']❌ 不支持（除非 label 本身是负整数）
 
# 假设 states 已经 set_index('State'):
states.get('Density').iloc[4]         # 253.81  （第 5 个州，按当前顺序）
states.get('Density').loc['California'] # 253.81 （按 label "California"）

# 默认 read_csv 后 row label = 0,1,2,...
# 这时 iloc 和 loc 给的结果可能"巧合"相等，但不要依赖！
bpd.read_csv('data/states.csv').get('Capital City').loc[35]   # 'Oklahoma City'
bpd.read_csv('data/states.csv').get('Capital City').iloc[35]  # 'Oklahoma City'

10. df.set_index('col') — 把某列变成 index

  • 该列从普通列变成 row label，列数减 1（消失为 index）。
  • 原先的 index（0,1,2,...）消失。
  • 返回新 DataFrame。
  • 设 index 之后，可以用 .loc['Pennsylvania'] 之类按 label 访问。
  • 选合适的 index 列：必须是唯一标识符（unique identifier，所有值都不重复，类似主键）。
# set_index 前: 6 columns + 默认 index 0..49
# set_index 后: 5 columns + index = 'State'
states = states.set_index('State')
states.get('Density').loc['California']   # 253.81
states.get('Density').loc['Pennsylvania'] # 290.61

11. df.shape — 形状

  • df.shape 返回一个 tuple (行数, 列数)。
  • df.shape[0] = 行数（极常用！判断行数 / 估概率）。
  • df.shape[1] = 列数。
  • 注意：set_index 后，列数会少 1，因为那一列变 index 了。
states.shape       # (50, 6)
states.shape[0]    # 50  → 用于过滤后数行数
states[states.get('Region') == 'West'].shape[0]   # 西部州的数量

12. 不可变性 (immutability)

babypandas 中所有方法都返回新对象，不会修改原 DataFrame：

  • .assign() ✅ 新
  • .sort_values() ✅ 新
  • .set_index() ✅ 新
  • .reset_index() ✅ 新
  • .merge() ✅ 新
  • .groupby() ✅ 新

因此典型工作流：

states = states.assign(Density=...)         # 重新赋值
states = states.sort_values(by='Density')   # 重新赋值
states = states.set_index('State')          # 重新赋值

# 或者链式调用，最后赋值一次
states = (states
          .assign(Density=states.get('Population') / states.get('Land Area'))
          .sort_values(by='Density', ascending=False)
          .set_index('State'))

13. 选 index 的标准

合适的 index 列应同时满足两条：

  1. 唯一性 (uniqueness)：每个值在该列只出现一次（unique identifier）。
  2. 描述性 (descriptiveness)：看到这个值，能直观对应到具体哪一行。
💡 一句话直觉：Index 是用来查找一行的钥匙。钥匙必须 独一无二（唯一性）+ 写着主人的名字（描述性）。

反例 ① 分类列（如 "candy" = "Twix"/"M&M"/...） — 违反唯一性

典型数据：

  candy   store  price
0 Twix    A      1.50
1 M&M     A      1.00
2 Twix    B      1.60   ← Twix 又出现了
3 Twix    C      1.45   ← 又一次

如果硬把 candy 设为 index：

df.loc["Twix"]
# 不是返回一行 Series，而是返回 3 行的 DataFrame
# 因为 "Twix" 这个 label 对应多行

带来的具体麻烦：

  • .loc["Twix"] 想取"那一个 Twix"，结果取出 3 行
  • .loc["Twix"].get("price") 不再是一个数，而是一个 Series
  • 后面所有 .iloc[0] 拿单值的链式写法全部失效
  • midterm 中"哪一列适合做 index"几乎都是用这种"分类列重复"来设坑

反例 ② 地理/分组列（如 "neighborhood"） — 违反唯一性（更隐蔽）

  store_name        neighborhood
0 Joe's Pizza       Mission
1 Tony's Pizza      Mission        ← 同一个区
2 Mama's Pizza      Mission        ← 又一个
3 SF Best           Sunset

为什么比 ① 更容易被坑：

  • 表面看 neighborhood 像个"标签"，听起来像可以做 index
  • 但多家店在同一个区是常态 — 本质上还是重复
  • df.loc["Mission"] 又会返回多行
  • 出题人喜欢出"这一列看起来很有信息量但其实会重复"的迷惑选项
判断诀窍：问自己"这一列的值会不会被多个不同的事物共享？"如果会 → 不能做 index。

反例 ③ 数值统计列（如 "price", "rating"） — 同时违反两条

  item    price   rating
0 A       9.99    4.5
1 B       9.99    4.5    ← 价格、评分撞车太正常了
2 C       12.50   3.8

违反唯一性：两个不同的产品很容易有相同的 price 或 rating。

违反描述性：

  • 看到 index = 4.5，你知道这一行是哪个 item 吗？不知道
  • 看到 index = "Alice Chen" 或 "student_id_4827"，你立刻知道是谁
  • index 的作用是让你一眼认出"这是哪一行"，而不是表达"这一行的某个测量值"

还有个深层原因：

  • 数值列通常是你要 sort / 做算术 / 画图 的目标（sort_values(by="price")、plot(y="rating")）
  • 如果把它设为 index，反而把"被分析的数据"和"行的标签"搞混
  • 数值列的本职工作是"被分析"，不是"做标识"

正例（适合做 index 的列）

  • student_id、order_id、phone_number：天然唯一
  • state（美国 50 州数据集中，每州一行 → 唯一且描述性）
  • unique 名字列（如表里没有重名的 student_name）
📝 midterm 考法：通常给一个 DataFrame schema（几列名 + 一两行示例），让你选哪一列最适合做 index。
解题步骤：
  1. 排除所有"明显会重复"的列（分类、地理、价格、评分、日期等）
  2. 剩下的看是否有 ID 类的列 → 选它
  3. 如果都没有，找一个"看起来不会重复"的描述性列（如 unique name）

14. 取极值的标准模板

# "评分最高的那家店的名字" 类问题：
df.sort_values(by='rating', ascending=False).get('store').iloc[0]

# 等价（升序后取最后）：
df.sort_values(by='rating', ascending=True).get('store').iloc[-1]

# 但如果只要"最高分"这个数值（不要店名），更简洁：
df.get('rating').max()

⚠️ 常见陷阱与考点

陷阱 1：.assign() 不会修改原 DataFrame

states.assign(Density=...)
states.shape   # 仍然 (50, 6)，没有 Density 列！

# 必须：
states = states.assign(Density=...)

陷阱 2：.get('col') 返回 Series；.get(['col']) 返回 DataFrame

这个区别在 plot / merge / 链式取值时影响巨大。

states.get('State').iloc[0]      # 'Alabama'  (一个字符串)
states.get(['State']).iloc[0]    # 一个 1-row DataFrame

陷阱 3：.loc 和 .iloc 在 set_index 之前看起来一样

因为 read_csv 后 row label 默认是 0,1,2,...，整数 label 和整数位置巧合相等。set_index 后就完全不同：

states = states.set_index('State')   # row label = 'Alabama', 'Alaska', ...

states.get('Density').iloc[4]            # ✅ 第 5 个，等于 'California' 的密度
states.get('Density').loc['California']  # ✅ 同上
states.get('Density').loc[4]             # ❌ KeyError! 'Alabama' 不是 4
states.get('Density').iloc['California'] # ❌ TypeError! iloc 需要整数

陷阱 4：sort_values 后用 .iloc[i] 而不是 .loc[i]

排序不会重置 row label！原先的 row label 跟着行一起被排到新位置。所以"排完序的第一行"必须用 .iloc[0]（按位置），不能用 .loc[0]（按 label）— 那拿到的是原始 row label 为 0 的那一行（无关排序）。

ordered = states.sort_values(by='Density', ascending=False)
ordered.get('State').iloc[0]   # ✅ 'New Jersey'（密度最大）
ordered.get('State').loc[0]    # ❌ 拿的是 row label 0 = 'Alabama'

陷阱 5：set_index 之后那一列就消失了

states.shape                    # (50, 6) before
states = states.set_index('State')
states.shape                    # (50, 5)  → 'State' 列没了，变成 index
states.get('State')             # ❌ KeyError，已经不是列了

陷阱 6：reset_index 会把 index 变回普通列

# reset_index 把当前 index 变回一列（叫 'State'），并重置 index 为 0,1,2,...
states_reset = states.reset_index()
states_reset.shape   # 多了一列回来

这个在 .groupby(...).reset_index() 模式中会用到（Section 2 详讲）。

陷阱 7：.iloc[-1] vs .iloc[len-1]

.iloc[-1] 返回最后一个元素（Python 列表风格的负索引）。这跟 .iloc[df.shape[0] - 1] 等价，但前者更常用、更安全。

states.get('State').iloc[-1]              # 最后一个州
states.get('State').iloc[states.shape[0]-1]  # 等价但啰嗦

陷阱 8：sort_values 的 ascending 必须 keyword 形式

states.sort_values(by='Density', ascending=False)  # ✅
states.sort_values(by='Density', False)            # ❌ SyntaxError
                                                    #   "positional argument follows keyword argument"

陷阱 9：column name 大小写敏感、空格敏感

states.get('density')      # ❌ KeyError，列叫 'Density'
states.get('Land area')    # ❌ KeyError，列叫 'Land Area'

陷阱 10：取一个值需要"先取列，再取行"

课程约定：要拿到一个单元格，先 .get(col) 拿到 Series，再 .iloc[i] 或 .loc[label] 拿到具体值。

states.get('Density').loc['California']   # 标准写法
states.get('Density').iloc[4]              # 也可

Fall 2024 Q1 — Lecture 4

题目：Which of the following columns would be an appropriate index for the treat DataFrame?

(treat 的列："address" str, "candy" str, "how_many" int, "neighborhood" str。每行表示某一户给某一种糖。)

  • ○ "address"
  • ○ "candy"
  • ○ "neighborhood"
  • ○ None of these.
显示答案

答案：None of these.

中文解析：合适的 index 必须是唯一标识符。

  • address：同一家可以发多种糖（前 3 行都是 820 Opal Street），重复 → ❌
  • candy：同一种糖在不同家都出现，肯定重复 → ❌
  • neighborhood：一个邻居有多家，重复 → ❌

所以选 None of these。这一题考点是"判断一个列能否做 index = 看是否唯一"。

Fall 2024 Q2 — Lecture 4

题目：Which of the following expressions evaluate to "M&M"? Select all that apply.

(数据如题目截图：treat 的第 0 行是 Twix，第 1 行是 M&M，第 2 行 KitKat ...)

  • ○ treat.get("candy").iloc[1]
  • ○ treat.sort_values(by="candy", ascending=False).get("candy").iloc[1]
  • ○ treat.sort_values(by="candy", ascending=False).get("candy").loc[1]
  • ○ treat.set_index("candy").index[-1]
  • ○ None of these.
显示答案

答案：选项 1 和 选项 3（即 treat.get("candy").iloc[1] 和 treat.sort_values(...).get("candy").loc[1]）。

中文解析：

  • 选项 1 ✅：iloc[1] 取第 2 行（位置 1），就是 "M&M"。
  • 选项 2 ❌：按 candy 降序排，第 2 行（iloc[1]）是按字母倒序的，例如 Twix → Skittles → ...，不是 M&M。
  • 选项 3 ✅：关键 — sort_values 不重置 row label！原先 row label 1 = M&M 那一行的 label 还是 1。.loc[1] 按 label 取，永远拿到原 row label 1 那行的 candy 值，即 M&M。
  • 选项 4 ❌：set_index("candy").index[-1] 取的是 index 的最后一个值，但顺序未排序，最后一行是 Skittles，不是 M&M。

核心考点：iloc 按位置 vs loc 按 label，sort_values 后 label 不变。

Fall 2024 Q5 — Lecture 5（用到 Lec 4 表达式）

题目：Write a Python expression that evaluates to the address of the house with the most pieces of candy available (the most pieces, not the most varieties).

显示答案

答案：

treat.sort_values(by="how_many", ascending=False).get("address").iloc[0]

中文解析："address with the most pieces" → 按 how_many 降序排，第一行的 address。

  • 必须是一行 expression。
  • 注意是 "most pieces"（最多的糖块数）= 按 how_many 排序，不是按糖的种类多少（那需要 groupby，section 2 内容）。
  • 也可以写 treat.sort_values(by="how_many").get("address").iloc[-1]（升序+取最后）。

(dining 的列："Dining Hall" str, "Item" str, "Price" str, "Calories" int。每行表示某 dining hall 提供的一道菜。)

  • ○ "Dining Hall"

Spring 2024 Q2 — Lecture 4 (.apply + iloc / .loc / np.arange)

题目背景：contacts DataFrame 已经被 .set_index("Person")。第一个 contact 是 Calvin，电话号码是 987-654-3210。已有 sum_phone(phone_number) 函数返回电话号码各位数字之和。

下面三个表达式都要等于 Calvin 电话号码各位数字之和（即 9+8+7+6+5+4+3+2+1+0 = 45），填空：

2.1： contacts.get("Phone").apply(sum_phone).iloc[___(a)___]

2.2： sum_phone(contacts.get("Phone").loc[___(b)___])

2.3： np.arange(___(c)___,___(d)___,___(e)___).sum()

显示答案

2.1 答案：(a) = 0（Calvin 是第一个contact，所以 iloc 用整数位置 0）

2.2 答案：(b) = "Calvin"（loc 用 row label，即 Person 列的字符串值）。注意要加引号 — 这是字符串！

2.3 答案：多种写法都可，例如 (c)=0, (d)=10, (e)=1 → np.arange(0, 10, 1).sum() = 0+1+2+...+9 = 45。也可写 np.arange(10).sum()（一个参数）。

中文解析：

  • 2.1 考.apply(func) 在 Series 上 — 把 sum_phone 应用到每个电话号码上，得到一个 Series；然后 iloc[0] 取第一个（Calvin 的）。注意 apply(sum_phone) 不带括号！
  • 2.2 考 .loc[label]。因为 contacts 已经 set_index("Person")，所以 row label 就是人名字符串。
  • 2.3 考 np.arange + .sum()。Calvin 的电话所有位数刚好是 0-9 各一次，所以 arange(10).sum() = 45。

Fall 2025 Q5.2 — Lecture 4 / 9（iloc 取分组结果）

题目背景：grouped 是 words.groupby(["diff_category", "length_category"]).count().reset_index() 的结果，按字母顺序排列。已知 diff_category 有 "easy"/"hard"/"moderate"，length_category 有 "long"/"medium"/"short"。需要"hard 且 short"对应那一行的"ps"列值。

num = grouped.get("ps").iloc[___(a)___]

显示答案

答案：(a) = 4

中文解析：groupby 多列后行按字母顺序排，每个 diff_category 配 3 个 length_category。顺序如下：

  1. ("easy", "long")
  2. ("easy", "medium")
  3. ("easy", "short")
  4. ("hard", "long")
  5. ("hard", "medium") ← 等等，让我们再算一次：按字母 hard 在 easy 之后但在 moderate 之前，所以顺序是：(easy,long)=0, (easy,medium)=1, (easy,short)=2, (hard,long)=3, (hard,medium)=4, (hard,short)=5。所以答案应该是 5。

更准确：(a) = 5。考点：iloc 用整数位置 + groupby 多列后的字典序行排列。这是典型的 W26 / Fa25 新趋势题。

Winter 2026 Q2 — Lecture 4-5（布尔过滤 + 取极值）

题目：Fill in the blanks below so that the expression evaluates to a float that is the highest rating of any pizza slice that has more than 10 ingredients.

pizza[___(a)___].___(b)___

2.2：What can go in blank (b)? Select all that apply.

  • ○ get("rating").apply(max)
  • ○ get("rating").max()
  • ○ sort_values("rating", ascending=False).get("rating").iloc[0]
  • ○ sort_values("rating", ascending=True).get("rating").iloc[-1]
  • ○ None of the above
显示答案

2.1 答案：(a) = pizza.get("num_ingredients") > 10（布尔 Series 用于过滤）

2.2 答案：选项 2、3、4 都对（即 .max()、sort_values 降序 + iloc[0]、sort_values 升序 + iloc[-1]）。

中文解析：

  • 选项 1 .apply(max) ❌：apply 是把 max 函数应用到 Series 的每一个元素，但 max 需要一个 iterable，单个 float 不能 max → 报错。
  • 选项 2 .max() ✅：直接给最大值。
  • 选项 3 ✅：经典模板。
  • 选项 4 ✅：升序后 .iloc[-1]（最后一个）= 最大值。

Winter 2026 Q5 — Lecture 4-5（数行数的多种 idiom）

题目：The DSC 10 staff is planning their quarterly pizza party, and they want to make sure that there are options for everyone. They're trying to find the total number of gluten-free pizza slice options in the San Diego area. Select all lines of code that correctly evaluate to the integer corresponding to the total number of gluten-free pizza slice options.

  • ○ pizza.set_index("store").get("gluten_free").sum()
  • ○ pizza.get(["gluten_free", "store"]).groupby("store").sum()
  • ○ pizza[pizza.get("gluten_free")].get("gluten_free").count()
  • ○ pizza[pizza.get("gluten_free")].shape[0]
  • ○ pizza.set_index("store").get("gluten_free").count()
  • ○ pizza.get("gluten_free").shape[0]
显示答案

答案：选项 1、3、4

中文解析："gluten_free 列是 bool 值（True/False）"是关键。要数 True 的个数：

  • 选项 1 ✅：.sum() 在 bool Series 上等于 True 的个数（True=1, False=0）。
  • 选项 2 ❌：.groupby("store").sum() 返回的是 DataFrame（按 store 分组的总数），不是单个 int。
  • 选项 3 ✅：先用布尔过滤（pizza[pizza.get("gluten_free")] 选出 gluten_free=True 的行），再 .count() 数行数。
  • 选项 4 ✅：跟选项 3 同思路，.shape[0] 给行数（int）。
  • 选项 5 ❌：.count() 数的是非空值的总数（不分 True/False），所有行都会被数，给的是 50（总行数），不是 gluten-free 数。
  • 选项 6 ❌：同 5，.shape[0] 给的是整列长度，所有 pizza 数。

核心考点：bool Series 上 .sum() = True 的个数；.count() = 非空值数（≠ True 数！）；.shape[0] = 行数。
