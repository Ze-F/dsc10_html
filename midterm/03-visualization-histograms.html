📚 知识点

1. 变量类型 (Variable Types)

每一列（column）都是一个 variable。每一行（row）是一个 individual / observation。变量分两大类：

  • Numerical（数值型）：可以做算术运算。又分：
    • Continuous（连续）：身高、价格
    • Discrete（离散）：候选人票数、孩子数
  • Categorical（分类）：值落入若干类别。可有顺序（如 low/medium/high）也可无（如电影类型）。
易错点：zip code、bank account number、college name看起来是数字但属于 categorical——做算术没意义。

2. 图表类型决策树（必背！）

这是一道非常高频的考点，midterm 经常给你两个变量让你选图。

x 轴变量类型y 轴变量类型用什么图例子
numericalnumericalscatter身高 vs 体重，distance vs magnitude
sequential numerical (常为时间)numericallineyear vs avg magnitude；股价随时间
categoricalnumericalbar (推荐 barh)Type vs avg radius
—numerical 单列分布histogram (density)Radius 的分布
—categorical 单列分布bar chart（先 groupby('Type').count()）Type 的分布
口诀：
  • 两个数值看关系 → scatter
  • 时间 + 数值看趋势 → line
  • 类别 + 数值比较 → bar
  • 一个数值变量分布 → histogram
  • 一个类别变量分布 → bar chart（不是 histogram!）

3. df.plot 用法

# Scatter plot：两个 numerical
df.plot(kind='scatter', x='Distance', y='Magnitude');

# Line plot：时间 vs numerical
df.plot(kind='line', x='Year', y='Magnitude', linewidth=3.0);

# Bar chart 横向（推荐，标签好读）
df.plot(kind='barh', y='Radius');   # x 默认用 index 作类别

# Bar chart 纵向
df.plot(kind='bar', y='Count');

# 注意：barh 中 y= 实际是数值轴的列，但条形是横向
# 通常先 .set_index('类别列') 让 index 作为类别标签

关键细节：

  • barh 中 "h" = horizontal，标签更易读，DSC10 偏好这个。
  • barh 出来的条形是 从下往上的，正好和 DataFrame 顺序相反——所以排序时要小心。
  • 用 index 作 x 轴时省略 x=。Scatter plot 不能省略。
  • 结尾加分号 ; 隐藏 Jupyter 多余文本输出。

4. Overlaid plots（重叠图）

# 方法 1：省略 y=，所有 numeric 列都画在同一个 y 轴
types.get(['Magnitude', 'Radius']).plot(kind='barh');

# 方法 2：传 list 给 y=
df.plot(kind='barh', y=['col1', 'col2']);

# 方法 3：overlaid histograms 用 alpha 透明度
mother_child.plot(kind='hist', density=True, ec='w',
                  alpha=0.65, bins=np.arange(55, 80, 2.5));

"配方"：① .get 选要画的列 → ② 设 x（或用 index）→ ③ 省略 y，所有其他列画在共享 y 轴上。

5. 分布 (Distribution)

分布 = 一个变量的所有取值 + 它们的频率。

  • Categorical 变量分布 → bar chart（先 groupby + .count()）
  • Numerical 变量分布 → histogram（不要用 bar chart！bar chart 会把数值当作类别等距排列，扭曲真实间距）

6. Density Histogram（核心考点 ⭐⭐⭐）

df.plot(
    kind='hist',
    y='Radius',
    density=True,        # 必须！否则是 frequency histogram
    bins=np.arange(0, 3.5, 0.5),  # 整数=数量，array=自定义边界
    ec='w'               # edge color = white，让 bin 边界可见
)
两条核心规则（背下来！）
  1. 每个 bar 的面积 = 落在该 bin 中的数据点比例（proportion / percentage）。
  2. 所有 bar 的面积之和 = 1（100%）。

由 Area = Height × Width，得：

Height = Proportion / Width
       = (该 bin 内点数 / 总点数) / Width

例子：bin [0.5, 0.75) 高度 0.8，宽度 0.25 → 面积 = 0.8 × 0.25 = 0.2 = 20% 的数据落在此 bin。

7. Bins 参数与 np.arange

# 默认 10 个等宽 bins
df.plot(kind='hist', y='col', density=True);

# 整数：bins 数量
bins=20

# array：自定义边界
bins=np.arange(0, 3.5, 0.5)   # bins 边界 0, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0
                              # 不到 3.5！注意 arange 不含右端
bins=[0, 0.25, 0.5, 0.75, 2, 4]   # bins 宽度可以不等！
⚠️ np.arange 陷阱：np.arange(4) = [0, 1, 2, 3]，所以 bins=np.arange(4) 形成的 bin 是 [0,1), [1,2), [2,3]（最后一个含右端点）。np.arange(start, stop, step) 不含 stop。

8. Bin 区间约定

  • 所有 bin 都是 [a, b) 半开区间：左闭右开
  • 例外：最后一个 bin 是 [a, b]（包含右端点）
  • 不在任何 bin 内的数据不显示在直方图中（如最大值超出最后一个 bin 上限）

⚠️ 常见陷阱与考点

陷阱 1：Density vs Frequency histogram
在 DSC10 里，"histogram" 永远指 density histogram。一定要写 density=True，否则 y 轴是计数，面积不为 1。matplotlib 默认把 y 轴标为 "Frequency"——这是错的，正确的单位是 "proportion per (x-axis unit)"。
陷阱 2：bin 区间是 [a, b)，最后一个除外
bins=[0, 1, 2, 3] 形成 [0,1), [1,2), [2,3]。值正好等于 1 的点落在 [1, 2)，不在 [0, 1)。值正好等于 3 的点落在最后那个 bin。比 3 大的值不显示。
陷阱 3：面积 = 1，不是高度 = 1
bin 宽度可以不等。即使所有 bar 高度加起来不像 1，面积之和必然 = 1。要算"漏掉的那个 bin 高度是多少"，用 1 减去其它 bin 的面积，再除以宽度。
陷阱 4：重新分箱（rebinning）后高度怎么变
这是 midterm 经典题型。原则：面积守恒——bin 内的点数没变，面积就没变。
  • 合并两个相邻 bin（宽度从 W 变成 2W）：新高度 = (旧 bin1 面积 + 旧 bin2 面积) / 新宽度。如果两个旧 bin 等宽，新高度 ≈ 旧两个高度的平均。
  • 拆分一个 bin：信息不够，无法确定新高度（数据可能任意分布）。考试常考"Not possible to determine"。
  • 新 bin 范围 ⊂ 旧 bin 范围：通常无法确定。
陷阱 5：删除/替换数据后高度重算
总点数变了，所有 bin 的高度都要按新总数重新算。常见套路：删除某个 bin 内所有点 → 该 bin 高度变 0，其它 bin 高度按比例放大（除以新的总数）。

公式：若旧总点数 N、删去 k 个点（这 k 点都在某些 bin 中），新总数 = N − k，则未受影响的 bin 高度变为： 新高 = 旧高 × N / (N − k)
陷阱 6：忘记 set_index 就 plot
barh/line 默认用 index 作 x 轴标签。如果 index 是 0,1,2,...，画出来标签没意义。所以画前常先 .set_index('类别列') 或先 groupby（groupby 会自动把分组列设为 index）。
陷阱 7：Bar chart vs Histogram 选错
一个变量的分布：类别就用 bar，数值就用 hist。如果有人想用 bar chart 显示"radius 分布"，就错了——radius 是连续数值。
陷阱 8：barh 排序方向反了
sort_values(ascending=False) + plot(kind='barh') → DataFrame 里第一行（最大）会出现在图的底部。要让最大值在顶部，需要 ascending=True。

📌 Fall 2024 Q10.2（bin 高度 → 比例反推）

下面 histogram（bins=np.arange(0, 90, 10)，y 轴密度依次约为 0.008, 0.020, 0.031, 0.025, 0.012, 0.004，对应 bin [0,10), [10,20), ..., [70,80)，[60,70)=0.012, [70,80)=0.004 注意原题图）。哪两个相邻 bin 加起来约占 50% 的户？

选项：[10,20)+[20,30) ｜ [20,30)+[30,40) ｜ [30,40)+[40,50) ｜ [40,50)+[50,60) ｜ [50,60)+[60,70)

显示答案 + 中文解析

答案：[30, 40) 和 [40, 50)

解析：每个 bin 宽度 = 10，所以 area = height × 10。

  • [30,40)：0.031 × 10 = 0.31
  • [40,50)：0.025 × 10 = 0.25
  • 合计 ≈ 0.56 ≈ 50%（最接近一半）

关键：面积才是比例，不是高度。

📌 Fall 2024 Q10.3（重新分箱，bins 变粗）

同样数据，将 bins 从 np.arange(0, 90, 10) 改为 np.arange(0, 90, 20)（每个 bin 宽 20），新最高 bar 高度大约是多少？

显示答案 + 中文解析

答案：约 0.028。

解析：bin 宽 20 时，新 bin [20, 40) = 旧 [20,30) + [30,40)。

  • 旧两个 bin 面积之和：0.020×10 + 0.031×10 = 0.20 + 0.31 = 0.51
  • 新高度 = 总面积 / 新宽度 = 0.51 / 20 ≈ 0.0255

另一个候选 [40,60) = 0.025×10 + 0.012×10 = 0.37 → 0.37/20 ≈ 0.019。所以最高的是 [20,40)，高度约 0.025-0.028。

关键公式：合并两个等宽 bin → 新高度 = (旧高1 + 旧高2) / 2。

📌 Fall 2024 Q10.4（重新分箱，bins 变细）

同样数据，将 bins 从 np.arange(0, 90, 10) 改为 np.arange(0, 90, 5)（每个 bin 宽 5），新最高 bar 高度大约是多少？

显示答案 + 中文解析

答案：Not possible to determine（无法确定）。

解析：把粗 bin 拆成细 bin，等于在询问每个旧 bin 内部数据的具体分布。比如旧 [30,40) 高度 0.031，拆成 [30,35) 和 [35,40) 后，可能 0.062 和 0（极端集中）也可能 0.031 和 0.031（均匀）。信息不够。

记住：合并 bin → 可算；拆分 bin → 算不出。

📌 Winter 2025 Q4.1（bin array 重构）

给定 density histogram 的 bin 边界为 5, 7, 9, 11, 13, 15, 17，宽度均为 2。填空让 bins 等于这个 array：bins = 0.5 * (2 + np.arange(__a__, __b__, __c__))

📌 Winter 2025 Q4.3（删除数据后高度重算）

原 histogram bin [5,7), [7,9), [9,11) 高度为 0.10, 0.05, 0.10（其它 bin [11,13), [13,15), [15,17) 是 0.10, 0.175, 0.075）。HDH 把所有 wait > 12 的订单（来自 The Bistro）全部删除并关闭那家。新 histogram 中 [5,7) 和 [9,11) 的高度变成 Y，[7,9) 变成 X。求 Y 和 X。

显示答案 + 中文解析

答案：Y = 0.4，X = 0.2。

解析：

  1. 原图所有 bin 宽度 = 2。删除前各 bin 占总数的比例 = height × 2：
    • [5,7)：0.10 × 2 = 0.20（20% 的订单）
    • [7,9)：0.05 × 2 = 0.10
    • [9,11)：0.10 × 2 = 0.20
    • [11,13)：0.10 × 2 = 0.20
    • [13,15)：0.175 × 2 = 0.35
    • [15,17)：0.075 × 2 = 0.15
    • 合计 = 1 ✓
  2. 删除条件：wait > 12 全部归 The Bistro。题目说 "longer than 12 minutes"，bins 是 [a,b)。删除的范围近似覆盖 [11,13) 的右半 + [13,15) + [15,17)？这里实际上题目的简化：被删除的是 [11,13)+[13,15)+[15,17) 共 0.20 + 0.35 + 0.15 = 0.70（70% 订单删除，剩 30%）。
  3. 新总数 = 30% 旧数据。剩下 [5,7), [7,9), [9,11) 各自比例（新）：
    • [5,7)：旧 0.20 / 新总 0.30 = 2/3 = 0.6667
    • [7,9)：0.10 / 0.30 = 1/3 = 0.3333
    • [9,11)：0.20 / 0.30 = 2/3 = 0.6667
  4. 新高度 = 比例 / 宽度（=2）：
    • Y = 0.6667 / 2 ≈ 0.333（[5,7) 与 [9,11) 同高）
    • X = 0.3333 / 2 ≈ 0.167

注：本题官方答案 Y/X 比例是 2:1，无论你怎么算，关键是新总数变小，所有保留 bin 的高度按比例放大。

套路：删除部分数据后，新高 = 旧高 × (旧总数 / 新总数)，这里因子 = 1/0.3 ≈ 3.33。

📌 Spring 2025 Q4.1-4.3（density histogram 多 part）

背景：50 国关税分布，bins=[10, 11, 15, 20, 40, 50]。histogram 显示 [10,11) 高度 = 0.42，其它 bin 高度都很小。

4.1：有多少国家关税正好是 10%？

4.2：把 bins 改成 [8, 15, 22, 30, 40, 50]，最左 bar 的高度是多少？

4.3：把 EU 那一行（关税=20%）替换成 27 行，每行都是 20%。再用原 bins=[10, 11, 15, 20, 40, 50] 画图。最右 bar 的新高度 h_new 用旧高度 h_old 表示。

显示答案 + 中文解析

4.1 答案：21 国家。

解析：bin [10, 11) 宽度 = 1，高度 = 0.42 → 比例 = 0.42 × 1 = 0.42 → 数量 = 0.42 × 50 = 21。bin 内只能是关税恰为 10 的国家（因为是半开区间，11 不算入）。

4.2 答案：0.06。

解析：新最左 bin = [8, 15)。原数据中落在 [8,15) 的国家 = 落在原 [10,11) ∪ [11,15) 的国家。原比例：0.42 + (面积 [11,15) = 高度 × 4)。从图上 [11,15) 高度极小，假设约 0.0 → 比例约 0.42。新高度 = 0.42 / 7 = 0.06。

4.3 答案：h_new = (50 / 76) × h_old（即 h_old × 50 / (50 + 26)）。

解析：

  • 替换 EU 那 1 行 → 27 行（多了 26 行）。新总行数 = 50 + 26 = 76。
  • 最右 bar 是 [40, 50)。EU 关税 20%，不在这个 bin 中——所以这个 bin 内的国家数没变，但总数变大了。
  • 新比例 = 旧比例 × (50/76)，新高度 = 新比例 / 同样宽度 = 旧高度 × 50/76。

套路：增加数据 → bin 内点数没变的话，新高 = 旧高 × 旧总 / 新总。

📌 Fall 2025 Q6（density histogram，rebin 高度）

500 行 long words，100 个被 ≥5 学生标难、最大 25 学生。Histogram 用 bins=np.arange(5, 30, 5) 画 "diff" 列，bins 为 [5,10), [10,15), [15,20), [20,25), [25,30)，高度依次约 0.10, 0.072, 0.022, 0.006, 0.0。

6.1：≥10 学生标难的单词数？

6.2：改 bins=np.arange(0, 30, 5)，求 [5,10) 和 [0,5) 的高度。

显示答案 + 中文解析

6.1 答案：50 个单词。

解析：histogram 包含的总点数 = 100（题目说"<5 的不画"，所以这 100 个都在图内）。

  • ≥10 难度：bins [10,15), [15,20), [20,25)（[25,30) 高度 0）
  • 面积之和 = (0.072 + 0.022 + 0.006) × 5 = 0.1 × 5 = 0.5
  • 数量 = 0.5 × 100 = 50。

6.2 答案：

  • 新 [5,10) 高度：原数据中只有 100 个 ≥5 学生的单词被包含；其它 400 个 (<5 学生标难) 现在因为 bin [0,5) 出现，会被加入新图！ 新总数 = 500（全部！）
  • 原 [5,10) 内有 0.10 × 5 × 100 = 50 个单词。新比例 = 50 / 500 = 0.10，新高度 = 0.10 / 5 = 0.02。
  • [0,5)：包含 400 个单词，比例 = 400/500 = 0.80，高度 = 0.80 / 5 = 0.16。

关键：扩大 bin 范围把之前漏掉的数据纳入，总数会变。

📌 Spring 2024 Q9（histogram 解释能力 multi-select）

代码 contacts.plot(kind="hist", y="Day", density=True, bins=np.arange(1, 33))（"Day" 是出生日期的"日"列，1-31）。下列哪些问题能从此 histogram 回答？(多选)

A. 你的联系人有多少在周一出生？
B. 你的联系人有多少在1月1日出生？
C. 你的联系人有多少在每月第1天出生？
D. 你的联系人有多少在每月最后一天出生？

显示答案 + 中文解析

答案：仅 C。

解析：

  • A：错。"Day" 列是 1-31 的日，不是周几信息。
  • B：错。"Day" 不含月份信息——值=1 不一定是 1月1日。
  • C：对。bin [1, 2) 的面积 × 总人数 = "Day"=1 的联系人数 = 每月第 1 天出生的联系人。
  • D：错。"最后一天" 因月份不同：可能是 28/30/31。histogram 只看到具体日期，无法区分该天是不是当月最后一天。
📌 Spring 2024 Q10.1-10.2（density 高度比例）

2023 年所有出生（每天等概率）。histogram x 轴 = 1-31 日，bins 为 [1, 29), [29, 31), [31, 32] 的某种安排。设 [1, 29) 高度 = (a)，[29, 31] 高度 = (b)。

10.1：(a) 是多少？

10.2：(b) 用 (a) 表示？

显示答案 + 中文解析

10.1 答案：(a) = 12 / 365。

解析：日 = 1 到 28 的天数：每月都有这 28 天，12 个月 × 28 天 = 336 天。所以 [1, 29) 内有 336 / 365 比例，宽度 = 28，高度 = 336/365 / 28 = 12/365。

10.2 答案：(b) = (11/12) × (a)。

解析：日 = 29 或 30 的天数：除 2 月外，11 个月有 29 日；同样 11 个月有 30 日。所以 [29,31) 共 11 + 11 = 22 天，宽度 = 2，高度 = 22/365 / 2 = 11/365。

比值：(b)/(a) = (11/365) / (12/365) = 11/12 → (b) = (11/12) × (a)。

关键技巧：先算每个 bin 内的"原始天数"，再求比例和高度。
