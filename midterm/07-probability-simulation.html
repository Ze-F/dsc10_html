📚 知识点

📐 Part A：概率（Lec 11）

1. 基本术语

  • Experiment（实验）：会产生某个结果的过程，比如抛硬币、抽卡。
  • Outcome（结果）：experiment 的一个具体可能值，如"正面"、"红桃 K"。
  • Sample space（样本空间）：所有可能 outcome 的集合。
  • Event（事件）：sample space 的子集，比如"抽到红桃"。
  • P(A)：事件 A 发生的概率，0 ≤ P(A) ≤ 1。

2. 古典概率（Equally-likely outcomes）

P(A) = (A 包含的 outcome 数) / (sample space 大小)

前提：所有 outcome 等可能。比如骰子掷一次，每个面 1/6。

3. With replacement（放回） vs Without replacement（不放回）

放回 (with replacement)不放回 (without replacement)
每次抽样后放回去，下次还可能抽到同一个不放回，分母 -1，可选数减少
独立性每次抽样独立不独立（前面抽什么会影响后面）
例（100 颗糖中抽 3 颗 Twix，共 40 颗 Twix）(40/100) × (40/100) × (40/100)(40/100) × (39/99) × (38/98)

4. Multiplication rule（乘法法则）

一般：P(A and B) = P(A) × P(B | A)
独立：P(A and B) = P(A) × P(B)

5. Conditional probability（条件概率） Sp25+ 新趋势

P(B | A) = P(A and B) / P(A) = (#A 且 B) / (#A)

读法："在 A 发生的前提下，B 发生的概率"。注意分母是 P(A) 而不是 1。

形式化考法：题目直接给 P(A)、P(B)、P(A and B)，让你算 P(A|B) 或 P(A or B)。这是 Sp25 起开始考的格式，请熟练。

6. Independence（独立性）

A 和 B 独立 ⇔ P(A and B) = P(A) × P(B) ⇔ P(B | A) = P(B)

判断独立的核心："A 是否发生，是否会改变 B 的概率？" 不变就是独立。

7. Complement（补集）

P(not A) = 1 − P(A)

8. "至少一个"经典套路（最高频考点之一！）

P(at least one A in n trials) = 1 − P(no A in any trial) = 1 − (1−p)n

正面计算"至少一个"非常痛苦（因为可能 1 个、2 个、… n 个都算），必须用补集。

9. Addition rule（加法法则）

互斥（mutually exclusive）：P(A or B) = P(A) + P(B)
一般：P(A or B) = P(A) + P(B) − P(A and B)

💻 Part B：模拟（Lec 12）

1. 为什么模拟？

当数学算式难以推导（比如 Monty Hall、复杂条件下的概率），我们用电脑跑 10000 次实验，统计成功比例作为概率估计。这就叫 simulation。

2. np.random.choice 三种用法

# (1) 单次随机抽一个元素
np.random.choice(options)

# (2) 抽 n 个，默认 with replacement
np.random.choice(options, n)
np.random.choice(options, n, replace=True)   # 等价

# (3) 抽 n 个，不放回（n 不能大于 len(options)）
np.random.choice(options, n, replace=False)
⚠️ 默认 replace=True！ 想要不放回必须显式写 replace=False。这是高频考点。

3. 🌟 模拟模板（必背！）

几乎所有 midterm 模拟题都长这个样子，请把它背成肌肉记忆。

# =========== 通用模拟模板 ===========
# 步骤 1：定义"一次实验"会用的随机源 / 函数
options = np.array([...])         # 抽样池（卡牌、骰子、硬币、字母 …）

# 步骤 2：初始化重复次数和计数器
repetitions = 10000               # 一般 5000 ~ 10000
count = 0                         # 用整数累加器（也可以用 np.array([]) 累加结果）

# 步骤 3：for 循环跑 repetitions 次
for i in np.arange(repetitions):
    # (a) 进行"一次实验"，用 np.random.choice 模拟随机
    outcome = np.random.choice(options, k, replace=True)   # 或 False

    # (b) 检查这次实验是否"成功"，成功就计数 +1
    if <some condition on outcome>:
        count = count + 1

# 步骤 4：估计概率 = 成功次数 / 总次数
estimated_prob = count / repetitions

4. 模板的两种等价写法

写法 A：整数 counter（最常见）

count = 0
for i in np.arange(reps):
    outcome = np.random.choice(...)
    if condition:
        count = count + 1
prob = count / reps

写法 B：用 array + np.append 收集结果

results = np.array([])
for i in np.arange(reps):
    outcome = np.random.choice(...)
    results = np.append(results, condition)   # 存 True/False 或 0/1
    # 注意必须重新赋值！
prob = np.count_nonzero(results) / reps
# 等价：prob = results.mean()
# 等价：prob = np.mean(results)
⚠️ np.append 不是 in-place！ 必须写成 arr = np.append(arr, x)，否则数组不会被修改。

5. 统计成功的几种 idiomatic 写法

表达式含义
count / reps整数 counter 直接除
np.count_nonzero(arr) / repsarr 中非零（即 True、非 0 数字）个数 / reps
arr.mean() 或 np.mean(arr)当 arr 是 0/1 或 True/False 时，平均值就是比例
(arr == "WIN").mean()vectorized：先比较得到 True/False 数组，再求平均
np.mean(arr >= threshold)vectorized 算 "≥ 某阈值的比例"
(arr == "WIN").sum()等价于 np.count_nonzero

6. Worked example：60+ heads in 100 flips

reps = 10000
count = 0
for i in np.arange(reps):
    flips = np.random.choice(["H", "T"], 100)        # 100 次抛硬币
    num_heads = np.count_nonzero(flips == "H")
    if num_heads >= 60:
        count = count + 1
prob_60_plus = count / reps                          # 大约 0.028

7. Worked example：Monty Hall（换门 vs 不换门）

# 不换门 → 命中概率 ≈ 1/3
# 换门 → 命中概率 ≈ 2/3
reps = 10000
wins = 0
for i in np.arange(reps):
    car = np.random.choice([1, 2, 3])      # 车在哪扇门
    pick = np.random.choice([1, 2, 3])     # 玩家初选
    # ... 主持人开一扇没车的门，玩家"换门"...
    if switched_pick == car:
        wins = wins + 1
prob_switch_wins = wins / reps             # ≈ 0.667

⚠️ 常见陷阱与考点

陷阱 1：replace=True 是默认值
np.random.choice(arr, 3) 是放回抽样。考题如果说"不重复"、"不放回"、"each digit only used once"，必须显式写 replace=False。
陷阱 2：np.append 必须重新赋值
错误：np.append(arr, x)
正确：arr = np.append(arr, x)
这是因为 numpy array 大小不可变，append 返回新数组而不是改原数组。
陷阱 3："至少一个"必须用补集
"至少一个 A" 包括 1 个、2 个、…、n 个全发生，正面计算太繁琐。永远写 P(at least one) = 1 − P(none) = 1 − (1−p)n。
陷阱 4：条件概率的分母是 P(A)，不是 1
P(B | A) = P(A and B) / P(A)。常见错误：直接拿 P(A and B) 当作 P(B|A)。
陷阱 5：独立 ≠ 互斥（恰恰相反！）
互斥（mutually exclusive）：P(A and B) = 0，发生 A 就不能发生 B。
独立（independent）：P(A and B) = P(A) × P(B)，发生 A 不影响 B。
事实：如果 A、B 都有正概率，那么互斥就不可能独立（因为 P(A and B)=0 ≠ P(A)×P(B) > 0）。这是 T/F 题的经典坑。
陷阱 6：count_nonzero 与 np.mean 的差别
np.count_nonzero(arr) = 个数（要 / reps 才得概率）
np.mean(arr)（当 arr 是 0/1）= 比例（已经是概率）
Sp25 Q7.3 就考过：用 "WIN!" 字符串填 array 时，(results=="WIN!").mean() 直接得概率，np.count_nonzero(results) 得个数（仍需 /reps）。
陷阱 7：with vs without replacement 改变分母
放回：每次分母不变（如 100/100/100）
不放回：每抽一次分母 -1（如 100/99/98）
Fall 2024 Q6 就靠分母选项区分对错。
陷阱 8：counter 初始化必须在 loop 之外
错误：把 count = 0 写到 for 循环里 → 每次都重置，最终只剩最后一次的值。
正确：count = 0 在 for 之前。
陷阱 9：模拟得到的是估计概率
模拟跑 10000 次得到的概率会有随机波动（一般小数点后 1-2 位是稳定的）。题目通常说 "estimate" 或 "approximate"。

—— 概率题 ——

📍 Spring 2024 Q4.1 + Q4.2（概率：放回与不放回）

(a) 一只猴子在键盘上随机按 10 次按键（每次等可能选 0-9）。问：猴子拨出的这个十位数恰好等于你 contacts DataFrame 中某一行的概率？用 Python 表达式回答。

(b) 一只猫在键盘上不放回地按 10 个键（每个 0-9 数字只用一次）。已知它已经按了 987-654，问：猫接下来恰好拨出朋友 Calvin 的号码 987-654-3210 的概率？用未化简数学式回答。

显示答案 + 中文解析

(a) 共 1010 种可能号码，其中 contacts 中的号码命中数为 contacts.shape[0]。所以答案：contacts.shape[0] / (10 ** 10)

(b) 已经按了 6 位，剩余 4 位要从 {0,1,2,3} 中按"3,2,1,0"的顺序无放回抽出。剩 4 个数字，按特定顺序 = 1 / (4 × 3 × 2 × 1) = 1/24。

📌 解析：放回 vs 不放回的核心差别就在分母。猴子是放回（每位 10 选 1，共 1010）；猫是不放回，且已经知道前 6 位，所以只需算最后 4 位的排列。

📍 Fall 2024 Q6.1（不放回基本概率）

糖果碗有 40 Twix、50 M&Ms、10 KitKats（共 100）。你不放回地拿 3 颗。求"全是 Twix"的概率？

选项：(A) 40/100 · 39/100 · 38/100　(B) 40/100 · 40/99 · 40/98　(C) 40/100 · 40/100 · 40/100　(D) 40/100 · 39/99 · 38/98

显示答案 + 中文解析

答案：D　40/100 · 39/99 · 38/98

📌 解析：不放回 → 每抽一次分子 -1（剩余 Twix 数减少）、分母 -1（总剩余数减少）。第一次抽到 Twix 概率 40/100；条件下第二次 39/99；第三次 38/98。这道题就是 with/without replacement 的"分母测试"。

📍 Fall 2024 Q6.2 + Q6.3（complement & "some of each"）

(b) 同上场景，求"3 颗中没有一颗 Twix"的概率（保留未化简形式）。

(c) 设 a 是 (a) 答案、b 是 (b) 答案。写一个 a、b 表达式表示"既有 Twix 又有非 Twix 糖"的概率。

显示答案 + 中文解析

(b) (60/100) · (59/99) · (58/98)　（不放回拿 3 颗都不是 Twix）

(c) 1 − a − b

📌 解析：3 颗的可能性只有三类：全 Twix（概率 a）、全非 Twix（概率 b）、混合（即"some of each"）。三类互斥且穷尽 → 混合概率 = 1 − a − b。这就是补集思维。

📍 Spring 2025 Q6.1 + Q6.2（条件概率 + 独立性 T/F）

表格给出每个国家/大陆的 shipping container 概率。

(a) 已知一个 container 来自 Asia 但不是 Japan，求它来自 China 的概率？

(b) True/False：从 Vietnam 选 container 与从 Ireland 选 container 是独立事件。

显示答案 + 中文解析

(a) P(China | Asia and not Japan) = P(China) / (P(Asia) − P(Japan))。China 在 Asia 内，所以 P(China and Asia and not Japan) = P(China)。

(b) False。一次只选 一个 container，事件 "来自 Vietnam" 和 "来自 Ireland" 是互斥的（不可能同时成立）。如前所述，互斥且都正概率 → 不可能独立。

📌 解析：条件概率公式 P(B|A) = P(A and B) / P(A)，分母是缩小后的样本空间。独立性陷阱题的关键："只能选一个"必然互斥 → 不独立。

📍 Spring 2025 Q6.3 + Q6.4（with replacement multi-event + complement）

(c) 放回选 2 个 container：恰好一个来自 Germany、另一个来自北美的概率？

(d) 放回选 3 个 container：没有一个来自 Asia 的概率？

显示答案 + 中文解析

(c) 两个顺序（先 Germany 后 NA / 先 NA 后 Germany）：2 × P(Germany) × P(North America)

(d) 放回，独立：(1 − P(Asia))3

📌 解析：(c) 用 multiplication rule 加乘 2（两个顺序对称）。(d) 是 "no A in n trials" 的标准 complement 模式：(1−p)n。

📍 Fall 2025 Q8（独立、补集、"至少一个"、条件概率合集）

每个英文单词，Matt 觉得难的概率为 0.7，每个单词独立。

(1) 10 个单词组成的句子，Matt 觉得每个都难的概率？

(2) 10 个单词中至少有一个让 Matt 觉得难的概率？

(3) 已知 Matt 觉得难的话 Ryan 觉得难概率 0.9；Matt 觉得不难的话 Ryan 觉得难概率 0.5。求一个单词中至少一个人觉得难的概率？

(4) 10 个单词，每个单词恰好一个人觉得难的概率？

显示答案 + 中文解析

(1) 0.710（独立连乘）

(2) 1 − 0.310（补集！"至少一个"= 1 − "一个都不"）

(3) 用 complement 一步到位：P(at least one) = 1 − P(都不难)。
P(都不难) = P(Matt 不难) × P(Ryan 不难 | Matt 不难) = 0.3 × 0.5 = 0.15
所以 P(at least one) = 1 − 0.15 = 0.85

(4) 一个单词恰好一人觉得难 = "Matt 难且 Ryan 不难" 或 "Matt 不难且 Ryan 难"。
= 0.7 × 0.1 + 0.3 × 0.5 = 0.07 + 0.15 = 0.22
10 个单词独立 → 0.2210

📌 解析：本题把"独立"、"补集"、"条件概率（树状概率）"、"互斥相加"四个套路全考了。条件概率的关键：P(Matt 难且 Ryan 难) = P(Matt 难) × P(Ryan 难 | Matt 难)，不能直接 0.7×0.9（除非独立）。

📍 Winter 2026 Q6（条件 + 给定 P(A)、P(B) 求 P(A or B)）

Pizza by Peter 的概率：Bianca 单独做不能上桌 1/4；Ella 单独做不能上桌 1/3；Bianca 揉面 Ella 烤不能上桌 1/6；Ella 揉面 Bianca 烤不能上桌 1/5。

(a) Ella 单独做一个、Bianca 单独做一个，独立。求"至少一个不能上桌"的概率（化为分数/小数）？

(b) 设 A = "Ella 参与了制作"，B = "pizza 可以上桌"。给定 P(A) = 1/4、P(B) = 3/5。求 P(A or B)？

显示答案 + 中文解析

(a) P(at least one fail) = 1 − P(both pass) = 1 − (3/4)(2/3) = 1 − 1/2 = 1/2

(b) 用一般 addition rule：P(A or B) = P(A) + P(B) − P(A and B)。
要求 P(A and B)，即"Ella 参与且 pizza 可上桌"。这道题答案需要 P(A and B)，根据条件信息（Ella 参与 → 1/4 alone，或与 Bianca 合作）我们不能精确算出，所以答案是 "Not enough information"（题目把它列为可选答案）。

📌 解析：(a) 用补集巧妙避开 union 公式。(b) 是新趋势题：给 P(A)、P(B) 但 没有给 P(A and B)，又不能假设独立 → 不够算。仔细判断"是否独立 / 是否给了 P(A and B)"是关键。
