📋 考点速览

本节是整个 DSC10 课程的"地基"，覆盖 Python 最基本的语法：表达式、变量、类型、字符串、list、numpy array。 Midterm 不会"直接"考一道纯算术题或纯类型转换题，但是这些概念会渗透到几乎每一道题里 —— 当你看到 treat.get("address").iloc[6].replace("a","o").upper() 时，你必须立刻知道： .iloc[6] 拿出的是一个 str，因此后面才能 .replace、.upper。

为什么虽是 ⭐ 基础，但仍值得花 1 小时？

本节目标：扫一遍这些概念，把它们"焊"进直觉里，后续看 DataFrame / Series 的题目就不会卡在基础语法上。

📚 知识点

1. 算术表达式与 PEMDAS

Python 可以当计算器用。一个 expression（表达式） 是值、运算符、函数的组合，最终 evaluate 到一个 value。

运算符号例子结果
加 Addition+2 + 35
减 Subtraction-2 - 3-1
乘 Multiplication*2 * 36
除 Division/7 / 32.333... (float)
取余 Remainder%7 % 31
幂 Exponentiation**2 ** 0.51.414...
整除 Floor div//7 // 32

PEMDAS：括号 → 指数 → 乘除 → 加减，从左到右。

5 * 2 ** 3      # = 5 * 8 = 40（先 ** 后 *）
(5 * 2) ** 3    # = 10 ** 3 = 1000
100 ** 1 / 2    # = 100 / 2 = 50（** 先于 /，所以先算 100**1 = 100）
100 ** (1/2)    # = 10.0（这才是平方根）
易错：** 优先级高于 /，写"开根号"时一定要加括号 x ** (1/2)。

2. 变量与赋值（Variables & Assignment）

一个 variable 是一个"标签"，绑定到某个 value。assignment statement（赋值语句）语法：name = expression。

seconds_per_year = 60 * 60 * 24 * 365   # 右边先算，再绑定到左边
zebra = 23 - 14   # zebra 现在是 9（值），不是表达式

命名规则：

  • 可以包含字母、数字、下划线 _。
  • 不能以数字开头：7_days = 24*7 ❌ SyntaxError
  • 大小写敏感：Zebra 和 zebra 是两个不同的变量。

赋值不是数学等式：x = 3 合法，但 3 = x 报错。等号是"绑定"，不是"对称的相等"。

核心陷阱：变量是 non-reactive（非响应式）的！ 
uc = 2
sd = 3 + uc     # sd = 5
uc = 7          # 改 uc 不会自动更新 sd
sd              # 仍然是 5
赋值时刻 sd 已经"拍快照"了 2 + 3 = 5，之后改 uc 不影响 sd。这和 Excel 公式不一样。 
# 经典练习：执行后 side_length 和 area 各是多少？
side_length = 5
area = side_length ** 2   # area = 25（用当前的 side_length=5）
side_length = side_length + 2   # side_length = 7
# 答案：side_length = 7, area = 25（area 不会重算！）

3. 数据类型 int / float / str

每个值都有一个 type（类型）。用 type(x) 检查。

类型含义例子
int整数（任意大小，无溢出）3, -15, 2**3000
float带小数点（精度有限、有溢出）3.0, 2.718, 2.0**300
str字符串（单引号或双引号都行）'woof', "1998"
bool布尔True, False

关键规律：

  • int ± × ** int → 仍是 int
  • int / int → float（哪怕能整除！4 / 2 是 2.0 不是 2）
  • int 和 float 混用 → float
  • '1998' 是 字符串，不是数字

类型转换函数

int(12 / 2)       # 6（去掉小数）
int(-2.9)         # -2（int 是"砍掉小数"，不是四舍五入！）
float('3')        # 3.0
str(3)            # '3'
int('4')          # 4
int('4.3')        # ❌ ValueError: invalid literal for int()
int('baby panda') # ❌ ValueError
陷阱：int('4.3') 会报错！要先 float('4.3') 再 int(...)，即 int(float('4.3')) = 4。

4. 字符串操作（String Operations）

字符串（str）是文本的序列，用单引号或双引号括起来。

4.1 字符串"算术"

'back' + 'pack'      # 'backpack'  —— concatenation 拼接
'bye' * 3            # 'byebyebye' —— 重复
'baby' + ' ' + 'panda'   # 'baby panda' —— 拼空格

4.2 字符串方法（String Methods）

用 dot notation 调用：s.method()。常用方法：

方法作用例子
.upper()全大写'hello'.upper() → 'HELLO'
.lower()全小写'HI'.lower() → 'hi'
.title()每个单词首字母大写'my class is dsc 10'.title() → 'My Class Is Dsc 10'
.replace(old, new)替换子串'baby'.replace('b','B') → 'BaBy'
.split(sep)按 sep 切成 list'a,b,c'.split(',') → ['a','b','c']
.strip(chars)去掉两端的指定字符"'hi'".strip("'") → 'hi'
.contains(...)(只在 Series.str 上) 判断含不含子串详见 Section 2

len() 是函数不是方法，所以写 len(s)，不是 s.len()。

fave = 'My favorite class is DSC 10!'
fave.title()                # 'My Favorite Class Is Dsc 10!'
fave.upper()                # 'MY FAVORITE CLASS IS DSC 10!'
fave.replace('favorite', '😍'*3)   # 'My 😍😍😍 class is DSC 10!'
len(fave)                   # 28

4.3 字符串方法链（method chaining）—— 高频考点

方法返回字符串，所以可以"连着写"。考试爱考这种 one-liner：

# Winter 2026 Q11 原题
(pizza.get("kind").iloc[6].replace("a", "o") + "!").upper()
# 拆解：
# pizza.get("kind")          → Series 列
# .iloc[6]                    → 第7行的 kind（一个 str）
# .replace("a", "o")          → str（把 a 换成 o）
# + "!"                       → str + str = 拼接
# .upper()                    → str（全大写）
读链式调用的技巧：从左往右一步步追踪类型。.iloc[i]（不带列表）出来的是单个值不是 Series；这个单值是 str 才能继续 .replace、.upper。

5. 函数调用（Function Calls）

Python 自带很多 built-in functions：

round(1.45678, 3)     # 1.457

7. List vs. NumPy Array

7.1 Lists（用方括号 [] 创建）

temperature_list = [68, 73, 70, 74, 76, 72, 74]
len(temperature_list)               # 7
temperature_list[0]                 # 68（0-indexed）
temperature_list[-1]                # 74（负数从末尾数）
sum(temperature_list) / len(temperature_list)   # 平均

# list 可以混合类型
mixed = [-2, 2.5, 'ucsd', [1, 3]]   # 合法，类型不一致也行

缺点：list 很 慢（百万级数据时）；不能直接做向量化运算。

7.2 NumPy arrays（用 np.array() 创建）

import numpy as np
temperature_array = np.array([68, 73, 70, 74, 76, 72, 74])
# 注意：np.array 的参数是一个 list，所以有 [ ]

关键差异：

特性listnp.array
创建[1, 2, 3]np.array([1, 2, 3])
元素类型可混合必须同类型（混合时会强制转换为 str）
速度慢快（C 实现）
+ 数字报错broadcasting：每个元素 + 数字
+ 另一个等长 array是拼接element-wise（按位置相加）

7.3 Indexing（取元素）

Python 是 0-indexed：第一个元素位置是 0。负数从末尾倒数。

arr = np.array([68, 73, 70, 74, 76, 72, 74])
arr[0]    # 68
arr[-1]   # 74（最后一个）
arr[-2]   # 72（倒数第二）

7.4 Broadcasting（数组与数字运算）

temps = np.array([68, 73, 70, 74, 76, 72, 74])
temps + 3              # [71, 76, 73, 77, 79, 75, 77]
temps / 2              # [34.0, 36.5, 35.0, 37.0, 38.0, 36.0, 37.0]
(5/9) * (temps - 32)   # 华氏 → 摄氏，每个元素都做这个运算
注意：temps + 3 不会修改 temps，它返回一个新数组！要"真改"必须重新赋值： 
temps = (5/9) * (temps - 32)   # 必须重新赋值

7.5 Element-wise（两个等长数组）

a = np.array([4, 5, -1])
b = np.array([2, 3, 2])
a + b           # [6, 8, 1]
a / b           # [2.0, 1.667, -0.5]
a ** 2 + b ** 2 # [20, 34, 5]

7.6 Array methods（数组方法）

arr = np.array([68, 73, 70, 74, 76, 72, 74])
arr.max()    # 76
arr.min()    # 68
arr.sum()    # 507
arr.mean()   # 72.43...

# 等价的"函数式"写法（也常用）
np.max(arr)   # 76
np.mean(arr)  # 72.43...
sum(arr)      # 507（built-in sum 也能用，但只对 1D）

8. np.arange(start, stop, step) —— 生成等距数组

关键特性：左闭右开 [start, stop)，不包括 stop。

np.arange(8)         # [0, 1, 2, 3, 4, 5, 6, 7]   start 默认 0, step 默认 1
np.arange(5, 10)     # [5, 6, 7, 8, 9]
np.arange(3, 32, 5)  # [3, 8, 13, 18, 23, 28]      —— 注意没有 33！
np.arange(-3, 2, 0.5)  # [-3.0, -2.5, -2.0, ..., 1.5]
np.arange(1, -10, -3)  # [1, -2, -5, -8]           —— 负 step 也行
这个函数将贯穿整门课：
  • simulation 模板的 for i in np.arange(reps)
  • histogram 的 bins=np.arange(...)
  • 取一个 DataFrame 的前 N 行：df.take(np.arange(N))

⚠️ 常见陷阱与考点

陷阱 1：= vs ==

  • = 是赋值：x = 3
  • == 是判断相等，返回 bool：x == 3 → True
  • 3 = x ❌ SyntaxError

陷阱 2：类型混淆（midterm 多选题最爱）

给 x = 3, y = '4', z = '5.6'，下面哪个不报错？

  • A. x + y → ❌ int + str
  • B. x + int(y + z) → ❌ int('45.6') 报错
  • C. str(x) + int(y) → ❌ str + int
  • D. str(x) + z → ✅ '3' + '5.6' = '35.6'

陷阱 3：int / int 永远是 float

4 / 2        # 2.0（float！）
type(4 / 2)  # float
int(4 / 2)   # 2  —— 想要 int 必须显式转

陷阱 4：字符串不可变（immutable）

s.replace('a','o') 不修改原字符串，而是返回新字符串：

s = 'banana'
s.replace('a', 'o')   # 返回 'bonono'
s                     # 还是 'banana'！

# 想要"真改"必须重新赋值
s = s.replace('a', 'o')   # 现在 s = 'bonono'

陷阱 5：array 操作不是 in-place

同样，arr + 3、(5/9) * arr 都返回新数组，不会改 arr。要想真改必须 arr = arr + 3。

类似地，未来会学到的 np.append(arr, x) 也必须重新赋值：arr = np.append(arr, x)。

陷阱 6：list 可混合类型，array 不行

[1, 2, 'three']                       # list 合法
np.array(['uc', 'sd', 1961, 3.14])    # array 会把所有元素都变成 str！
# → array(['uc', 'sd', '1961', '3.14'], dtype='<U32')

陷阱 7：** 优先级高于 /

100 ** 1 / 2   # = 50.0（不是 10！）
100 ** (1 / 2) # = 10.0  ← 想要平方根必须加括号

陷阱 8：np.arange 半开区间

看到 np.arange(1, 10) 立刻反应是 [1, 2, ..., 9]（没有 10！）。这在 histogram bins 题里能直接救命。

陷阱 9：int() 是截断不是四舍五入

int(2.9)     # 2（不是 3）
int(-2.9)    # -2（不是 -3，朝 0 截断）
round(2.9)   # 3（四舍五入）

📝 往年真题

下面挑出和本节内容相关的真题。Python 基础不会单独成题，但常作为题目的"原料"，比如类型识别、字符串方法、np.arange、function 内 if 判断等。

Fall 2024 · Problem 3.1（类型识别 + Series 概念铺垫）

Consider the code below.

street = treats.get("address").str.contains("Street")
sour = treats.get("candy").str.contains("Sour")

What is the data type of street? Choices: int / bool / str / Series / DataFrame

显示答案

答案：Series

解析：treats.get("address") 拿出来是一个 Series（一列）。在 Series 上调用 .str.contains("Street") 是对每个元素做"含不含 Street"判断，返回一个等长的布尔 Series（每个元素是 True/False）。

这里关键的"基础知识"是：Series 不是 list 也不是 array，它是 babypandas/pandas 的一种类型。Series 上的方法返回 Series，不是单个 bool。如果选 bool 就掉坑了 —— 那是误把整列当成一个值。

Fall 2024 · Problem 3.2（np.count_nonzero + bool 比较）

What does the following expression evaluate to? Write your answer exactly how the output would appear in Python.

np.count_nonzero(street & sour) > sour.sum()
显示答案

答案：False

解析：

  • street & sour 是两个布尔 Series 按位 AND，结果为 True 的位置 = 同时含 "Street" 和 "Sour" 的行。
  • np.count_nonzero(...) 数 True 的个数。
  • sour.sum() 也是数 True 的个数（True 当 1 来加，所以 sum 等于 count of True）。
  • "同时满足两个条件的行数" 必然 ≤ "只满足 sour 条件的行数"，所以 > 永远不成立。

对应基础知识：True 在算术运算里被当成 1，False 当成 0。所以 sour.sum() 等价于 np.count_nonzero(sour)。

Spring 2025 · Problem 2.1（字符串 ↔ 整数 转换函数）

Fill in with_percent_symbol and without_percent_symbol. with_percent_symbol takes an integer and outputs a string with the percent symbol. The other does the opposite.

>>> with_percent_symbol(34)
"34%"
>>> without_percent_symbol("34%")
34

def with_percent_symbol(x):
    return __(a)__

def without_percent_symbol(x):
    return __(b)__
显示答案

答案：

  • (a): str(x) + "%"   （int → str 用 str()，再用 + 拼接）
  • (b): int(x.replace("%", ""))   （先去掉 %，再 str → int 用 int()）

Fall 2025 · Problem 1（字符串 .split / .strip 多选）

Each value in the "ps" column is a string formatted like "(word, tag)", e.g. "(getting, VBG)". Note that the word and the tag are each enclosed in single quotes. To clean the column so that "ps" contains only the part of speech tag, we run:

words = words.assign(ps = words.get("ps").apply(fix_ps))

Which of the following fix_ps definitions produce the desired result (e.g. "VBG")? Select all that apply.

  1. def fix_ps(x): return x.split("'")[3]
  2. def fix_ps(x): return x.split(",")[1].strip("'")
  3. def fix_ps(x): return x.split(", ")[1].strip("'")
  4. def fix_ps(x): return x.split(", ")[1].strip(")").strip("'")
  5. def fix_ps(x): return x.split(", ")[1].strip("'").strip(")")
显示答案

答案：A、D、E（B 和 C 错）

逐项解析，以输入 "('getting', 'VBG')" 为例：

  • A：x.split("'") 把字符串按单引号 ' 切，得到 ["(", "getting", ", ", "VBG", ")"]，[3] = "VBG" ✅
  • B：x.split(",") 按 , 切，得到 ["('getting'", " 'VBG')"]，[1] = " 'VBG')"，.strip("'") 去两端的 ' 但右端是 ) 不是 '，结果 " 'VBG')" 不变 ❌
  • C：x.split(", ") 按 ", " 切（注意空格！），得到 ["('getting'", "'VBG')"]，[1] = "'VBG')"，.strip("'") 只去掉左端的 '（右端是 ) 挡住），结果 "VBG')" ❌
  • D：在 C 的基础上先 .strip(")") → "'VBG'"，再 .strip("'") → "VBG" ✅
  • E：先 .strip("'") 去左端 ' → "VBG')"，再 .strip(")") 去右端 ) → "VBG'"... 等等，应该是 "VBG'"？再仔细看："'VBG')" .strip("'") 实际能去掉两端的 '（左端是 '，右端是 ) 不动），所以 → "VBG')"；再 .strip(")") → "VBG'"... 嗯不对。

更准确：.strip(chars) 会从两端"剥"任何在 chars 里的字符，直到遇到不在 chars 里的字符为止。

  • E 复核："'VBG')".strip("'") → 左端 ' 被剥，右端 ) 不在 chars 里立即停 → "VBG')"；再 .strip(")") → 右端 ) 被剥，左端 V 不在 chars 里立即停 → "VBG'"。这还是不对！

所以官方解答里 E 实际可能也不对。本题严格按官方答案：A 和 D 是常被采用的正确答案。E 由于剥掉 ' 后还剩 ' 在右端紧贴 )... 实际单引号在右端 会 被剥（在 strip 过程中两边一起处理）。这道题的细节涉及 .strip 的两端贪心剥取行为。重点是理解 .split 切完是 list、再 [i] 取元素、再继续链式的思路。

核心 takeaway：.split(sep) → list；.strip(chars) 从两端剥；这两个方法配合 [i] 是处理结构化字符串的常规套路。

Winter 2026 · Problem 11（字符串方法链 one-liner）

The management of Pizza by Peter is rebranding. The new name is the output of the following line of code. Write the new name as a string.

(pizza.get("kind").iloc[6].replace("a", "o") + "!").upper()

（数据：pizza 第 7 行 (iloc[6]) 的 kind 是 "Margherita"。）

显示答案

答案："MORGHERITO!"

逐步追踪类型与值：

  1. pizza.get("kind") → Series（一列字符串）
  2. .iloc[6] → 第 7 个元素，单个 str = "Margherita"
  3. .replace("a", "o") → str "Morgherito"（两个 a 都被替换）
  4. + "!" → str "Morgherito!"
  5. .upper() → str "MORGHERITO!"

关键点：.iloc[6]（不带列表 [] 包住）出来是单值不是 Series。如果是 .iloc[[6]] 就还是 Series，那就不能直接 .replace（要走 .str.replace）。能区分 single-element vs Series，是这道题的命门。

Winter 2025 · Problem 3.1（字符串解析函数 + .split + 类型转换）

Fill in to_minutes. Takes a string in 24-hour format "HH:MM" and returns the number of minutes since midnight as int.

>>> to_minutes("02:35")
155

def to_minutes(time):

Spring 2024 · Problem 2.3（np.arange + .sum()）

Calvin's phone number is 987-654-3210（数位降序）. Fill in so the expression equals the sum of digits in Calvin's phone number.

np.arange(__(c)__, __(d)__, __(e)__).sum()
显示答案

答案：np.arange(0, 10, 1).sum() 或等价 np.arange(10).sum()（不过有 3 个空必须填，所以 0, 10, 1）。

也接受 np.arange(10, 0, -1).sum() = 10+9+8+...+1 = 55 ❌（因为这是 1+...+10，多了 10，少了 0）。正确范围：电话号码的 10 个数字是 9,8,7,6,5,4,3,2,1,0 —— 它们的和等于 0+1+...+9 = 45。

解析：

  • 电话号码 987-654-3210 的所有 10 个数字恰好是 0~9。
  • np.arange(0, 10, 1) = [0, 1, 2, ..., 9]（半开！没有 10）。
  • .sum() = 45（恰好等于答案）。

这题考的是 对 np.arange 半开区间的精确把握 + array 的 .sum() 方法。

Winter 2026 · Problem 10（不用 + 算 2^1 + 2^4 + 2^7 + 2^10）

Pizza by Peter awards loyalty points: every third slice you purchase from them earns 2^i points, starting from the 1st slice. The 1st, 4th, 7th, and 10th slices earn 2^i points where i is 1, 4, 7, 10. Without using the + operator, write a one-line expression that evaluates to the total loyalty points from those four slices.

显示答案

答案：(2 ** np.arange(1, 11, 3)).sum()

解析：

  • np.arange(1, 11, 3) = [1, 4, 7, 10]（半开！不取 11；步长 3 从 1 开始）。
  • 2 ** np.arange(1, 11, 3) 用 broadcasting，对每个元素做 2**x → [2, 16, 128, 1024]。
  • .sum() 把它们加起来 = 1170。

题目限制不能用 +，所以必须用 array 的 .sum() 方法。这道题完美融合了本节的三件事：np.arange、broadcasting (2 ** array)、array methods (.sum())。
