📋 考点速览

这一节是第二阶段（统计/概率/模拟）的基础工具。Midterm 几乎每年都会考一道条件函数填空或for-loop accumulator的题。Lec 10 本身代码不难，但和 .apply、模拟、字符串方法结合后变化很多。

  • 布尔与运算符：and/or/not 用于标量 bool；&/|/~ 用于 Series（一定要加括号！）
  • in 运算符：判断元素在 list/array/string 中是否出现，结果是 bool
  • if/elif/else：缩进敏感，函数里搭配 return；考点是等价改写（去 else、扁平化嵌套）
  • for 循环：遍历 list / array / string / np.arange(n)
  • Accumulator pattern（累加器模式）：
    • 整数计数：counter = 0 → 循环里 counter += 1
    • 数组累加：arr = np.array([]) → 循环里 arr = np.append(arr, x)（必须重新赋值！）
  • 真题主考形式：① 条件函数填空 ② if/else 等价改写 ③ for-loop 求和（电话号码digits）④ for-loop 模拟概率（与 Sec 7 重叠）
💡 一句话记忆：and/or 看到 Series 就报错（"truth value ambiguous"）；&/| 看到标量也能用，但必须每个比较都加括号。

📚 知识点

1. Booleans 与比较运算符

bool 是 Python 的一种数据类型，只有两个值：True 和 False。所有比较运算（==、!=、<、>、<=、>=）都返回 bool。

course = 10
course < 20            # True
type(course < 20)      # bool
course == 10           # True
1 == 1.0               # True   ← 数值上相等就 True，类型不同也行
1 == '1'               # False  ← 数字 vs 字符串永远不等

2. in 运算符（成员/子串）

10 in [10, 20, 30]      # True   ← 元素是否在 list/array
'DS' in 'Data Science'  # False  ← 子串是否在字符串
'DS' in 'DSC'           # True

考点：在条件函数里经常用来判断"某个值是否属于一组合法选项"。

3. 标量 boolean 运算符：and / or / not

用于把单个 bool 值组合起来。注意优先级：not > and > or。考试时建议用括号，不要靠优先级心算。

dept = 'DSC'; course = 10

dept == 'DSC' and course == 10   # True
not dept == 'DSC'                # False
course == 80 or type(course) == str  # False

# 优先级陷阱：and 先于 or
course == 10 or dept == 'DSC' and dept == 'CSE'
# 等价于：course == 10 or (dept == 'DSC' and dept == 'CSE')  → True

4. Series 位运算：& / | / ~

对布尔 Series做逐元素运算（element-wise），用于 DataFrame 查询过滤。每个比较都要加括号，因为 &/| 优先级高于 ==/</>。

# ✅ 正确：每个比较都加括号
df[(df.get('dept') == 'DSC') & (df.get('course') == 10)]

# ❌ 错误：会被解析成 'DSC' & df.get('course') 然后 == 10
df[df.get('dept') == 'DSC' & df.get('course') == 10]

# ❌ 错误：对 Series 用 and/or → "truth value of Series is ambiguous"
df[(df.get('dept') == 'DSC') and (df.get('course') == 10)]

Series 上做 .apply 里的判断and / or（因为函数处理的是单个标量）函数体里 if x > 5 and x < 10:

5. if / elif / else

if <condition_1>:
    <body_1>
elif <condition_2>:
    <body_2>
else:
    <body_3>
  • 缩进敏感：每一层 4 个空格。
  • 冒号别忘：if/elif/else 后面都要 :。
  • elif 链一旦有一个为 True，后面的全部跳过；如果用多个独立 if（不是 elif），每个条件都会被检查。
  • 函数里用 return 时，触发 return 就立即退出，elif 和 if 在这种情况下行为相同——这是 Sp25 Q3.2 / Fa25 Q4 的等价改写考点。

例：百分制转字母分

def grade_converter(grade):
    if grade >= 90:
        return 'A'
    elif grade >= 80:
        return 'B'
    elif grade >= 70:
        return 'C'
    elif grade >= 60:
        return 'D'
    else:
        return 'F'

等价改写技巧（考点）

下面四种写法在所有正整数输入下行为相同（只要每个分支都 return）：

# 写法 A：标准 if/elif/else
def f(x):
    if x <= 6: return "short"
    elif x <= 10: return "medium"
    else: return "long"

# 写法 B：扁平化（多个 if + return，无 elif/else）
def f(x):
    if x <= 6: return "short"
    if x <= 10: return "medium"     # x <= 6 已 return，到这里说明 x > 6
    return "long"                    # 到这里说明 x > 10

# 写法 C：先把"中间情况"挑出来（Fa25 Q4 思路）
def f(x):
    if 7 <= x <= 10: return "medium"
    if x >= 7: return "long"          # 已排除 medium，所以 >=7 就是 long
    return "short"

# 写法 D：无分支（Sp25 Q3.2 思路）—— 用算术表达
# 例：half = max(int((charged+1)/2), 10)

6. for 循环

for <element> in <sequence>:
    <body>

"sequence" 可以是 list、numpy array、字符串、np.arange(...) 等。

# 遍历元素本身
for college in colleges:
    print(college, 'College')

# 遍历下标（用 np.arange）
for i in np.arange(len(actions)):
    print('I', actions[i], 'and I felt', feelings[i])

# 遍历字符串：每次拿一个字符
for ch in 'hello':
    print(ch)

# 仅用循环重复 n 次（不关心循环变量）
for i in np.arange(3):
    print('🐻')

7. Accumulator Pattern（累加器模式）

DSC10 里几乎每个 for-loop 都属于这种模式：循环外初始化一个变量，循环里反复"加东西"进去。

(a) 整数计数器

# 数 array 里有多少个 > 5 的元素
counter = 0
for x in arr:
    if x > 5:
        counter = counter + 1     # 也可写 counter += 1
counter

(b) 数组累加器

results = np.array([])               # 循环外：空数组
for i in np.arange(10000):
    num_heads = flip(10)
    results = np.append(results, num_heads)   # 必须重新赋值！
⚠️ np.append 不修改原数组！它返回一个新数组。 
some_array = np.array([])
np.append(some_array, 'hello')   # 这行返回了新数组，但没存
some_array                       # 仍然是空！

# 正确做法：
some_array = np.append(some_array, 'hello')

(c) 数字累加（求和）

# Spring 2024 Q1 模板：算电话号码各位数字之和
def sum_phone(phone_number):
    total = 0
    for digit in phone_number:        # digit 是字符 '5','0','1','-','8',...
        if digit != '-':              # 跳过短横线
            total = total + int(digit)
    return total

8. 不要用 for-loop 做 Series 元素级运算！

这是 DSC10 反复强调的一条准则：能用 Series/numpy 内置方法的就别写 for-loop。

# ❌ 慢且丑
total = 0
for x in df.get('price'):
    total = total + x * 1.05

# ✅ 简洁
total = (df.get('price') * 1.05).sum()

那 for-loop 什么时候必须用？—— 模拟（simulation）：每次循环要调用一次 np.random.choice，结果不能直接向量化。

⚠️ 常见陷阱与考点

陷阱 1：and 用在 Series 上
df[(s1 == 'a') and (s2 > 5)] → 报错 ValueError: The truth value of a Series is ambiguous。
Series 之间用 &，并且每个比较都加括号。
陷阱 2：& / | 不加括号
s == 'a' & s2 > 5 会被解析为 s == ('a' & s2) > 5，类型错误或结果离谱。永远写 (s == 'a') & (s2 > 5)。
陷阱 3：忘记给 np.append 赋值
np.append(arr, x) 是返回新数组，不改原数组。永远写 arr = np.append(arr, x)。
类似地，babypandas 所有方法都不 in-place（如 df.assign(...) 也要赋值）。
陷阱 4：修改循环变量不影响原序列 
arr = np.array([1, 2, 3])
for x in arr:
    x = x * 10
arr   # 还是 [1, 2, 3]！
要修改原数组就用累加器或直接向量化运算 arr = arr * 10。
陷阱 5：缩进不一致
Python 严格区分缩进。同一个 block 必须用相同空格数（一般 4 空格）。混用 tab 和 space 会报 IndentationError。
陷阱 6：函数里漏写 else → 隐式返回 None 
def f(x):
    if x > 0:
        return 'positive'
    # 没写 else，x <= 0 时函数返回 None（不是 'negative'，也不报错！）
考试时要确认每条路径都有 return。
陷阱 7：类型比较的坑
  • 1 == 1.0 → True（int 和 float 数值上相等）
  • 1 == '1' → False（数字 vs 字符串永远不等）
  • True == 1 → True（bool 是 int 的子类）
  • 从 '1-2-3' 这种字符串里 for 出来的每个字符都是 str，要 int(digit) 才能加。
陷阱 8：用多个独立 if vs elif
多个独立 if（无 elif）会每个都判断一遍。如果有多个分支同时为 True，会 print / 执行多次。
但如果每个分支都 return，则等价于 elif（一旦 return 就退出函数）。这是 Fa25 Q4 等价改写题的核心。
陷阱 9：and/or 优先级
a or b and c = a or (b and c)。考试时一律加括号，不要靠记忆。
陷阱 10：在 .apply(func) 的 func 内部用 and 还是 &?
.apply 是把 Series 的每个元素分别送进 func，所以 func 内部是标量，用 and/or！这和直接对 Series 过滤完全相反。

题 3 · Spring 2025 Q3.2（Lec 10：等价改写，无分支版本）

填写 reciprocate_2 的 return 语句，使其与 reciprocate 在所有输入下行为一致。不能调用 reciprocate，且只用一行表达式。

显示答案与解析

答案：max(int((charged + 1) / 2), 10)

  • 原函数等价于："计算 half，但保证不低于 10"。这正是 max(half, 10) 的含义。

题 4 · Spring 2025 Q3.3（Lec 10：等价性测试）

已知 ch = tariffs.get("Tariffs Charged to USA")。写一个表达式，当且仅当 reciprocate 和 reciprocate_2 在 ch 的所有元素上输出一致时为 True。

显示答案与解析

答案：(ch.apply(reciprocate) == ch.apply(reciprocate_2)).all()

中文解析：

  • ch.apply(reciprocate) 返回一个 Series（每个元素调用一次函数）。
  • 两个 Series 用 == 做逐元素比较，得到一个 bool Series。
  • .all() 当所有元素都为 True 时返回 True，否则 False。

题 5 · Fall 2025 Q4（Lec 10：if/elif/else 等价改写）

原函数：

def classify_length(x):
    if x <= 6:
        return "short"
    elif x <= 10:
        return "medium"
    else:
        return "long"

填空使 classify_length_2 在所有正整数输入下输出相同。每个空必须是一行代码。

def classify_length_2(x):
    if ___(a)___:
        return "medium"
    if x >= 7:
        ___(b)___
    ___(c)___
显示答案与解析

答案：

  • (a)：7 <= x <= 10 （或 x >= 7 and x <= 10）
  • (b)：return "long"
  • (c)：return "short"

中文解析（按代码执行流程逆推）：

  1. 新函数第一个 if 处理 medium，所以 (a) 必须刚好是 medium 的范围：7 ≤ x ≤ 10。
  2. 到第二个 if x >= 7 时，medium 已被 return 排除（x 不是 7-10）。剩下 x >= 7 的只能是 x > 10，即 long → (b) 返回 "long"。

题 8 · Winter 2025 Q6.2（Lec 10 + Lec 12：模拟 + for-loop + counter）

6 种 topping，每种以概率 0.5 独立加入披萨。估计"恰好 3 种 topping"的概率。

tiptop = 0
num_trials = 10000
for i in np.arange(num_trials):
    pizza = np.random.choice([0, 1], __(a)__)
    if np.__(b)__ == 3:
        tiptop = __(c)__

toppa = __(d)__
显示答案与解析

答案：

  • (a)：6 （第二个参数：每次抽 6 个 0/1）
  • (b)：sum(pizza) （或 count_nonzero(pizza)）
  • (c)：tiptop + 1
  • (d)：tiptop / num_trials

中文解析：

  • 每次试验抽 6 个 0/1（np.random.choice([0,1], 6)，默认 with replacement），相当于 6 种 topping 各独立掷一次硬币。

题 9 · Spring 2024 Q5（Lec 10 + Lec 12：模拟 + len/np.unique）

Arya 电话区号后 7 位只用 2 个不同的数字。估计"猫/猴随机拨号后区号后 7 位恰好 2 种不同数字"的概率：

digits = np.arange(10)
property_count = 0
num_trials = 10000
for i in np.arange(num_trials):
    after_area_code = __(x)__   # 猴：with replacement；猫：without replacement
    num_distinct = len(np.unique(after_area_code))
    if __(y)__:
        property_count = property_count + 1

probability_estimate = property_count / num_trials
显示答案与解析

答案：

  • 猴（5.1）：np.random.choice(digits, 7)（默认 replace=True）
  • 猫（5.2）：np.random.choice(digits, 7, replace=False)
  • (y)：num_distinct == 2
  • 5.4：0（猫一定 7 个全不同，num_distinct 永远 = 7，永远不会进 if）

中文解析：

  • 模板还是整数累加器：每次试验生成数据 → 检查条件 → 满足就 +1。
  • 关键区别在 replace 参数：猴可重复（猴随机按键），猫不可（猫不会重按）。
  • 5.4 的"陷阱"：猫场景下 num_distinct 永远是 7，所以 if 永远不触发，最终概率 = 0/10000 = 0。这考的是对模拟逻辑的理解，不是代码。
