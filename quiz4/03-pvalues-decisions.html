Section 3 · p-values & Decisions

覆盖讲义：Lec 20（Hypothesis Testing & Total Variation Distance 前半）｜ Quiz 4 出现率：⭐⭐⭐ Tier 1（几乎每年都有 p-value 决策题）｜ 预计学习时间：1-1.5 小时

🎯 本节定位：Lec 19 教了 hypothesis test 的"流程"，本节给它定量决策标准 —— p-value、cutoff 0.05/0.01、reject vs fail to reject。每个 Quiz 4 都会出至少一道 "p = 0.03 → 该怎么办" 的判断题，速答率必须 100%。

📚 知识点

1. p-value：让"罕见"量化

p-value = P( test stat 等于 observed 或更极端 | null 为真 )

"更极端"的方向取决于 alt：

  • 如果 alt 让大值偏 alt（右尾）→ 更极端 = 大于等于 observed
  • 如果 alt 让小值偏 alt（左尾）→ 更极端 = 小于等于 observed

正式名字：observed significance level。

2. 从 empirical distribution 算 p-value

Simulate under null 得到 array results（长度 N），observed statistic 是 obs：

# 大值偏 alt（右尾）—— 比如 "A 比 B 大"
p_value = np.count_nonzero(results >= obs) / N

# 小值偏 alt（左尾）—— 比如 "硬币偏 tails，heads 数量少"
p_value = np.count_nonzero(results <= obs) / N

# 双边（带 abs 的 test stat）—— observed 也已经是 abs 后的，方向只有 "大"
p_value = np.count_nonzero(results >= obs) / N
⚠️ 用 >= 或 <=，不要用 > 或 <。p-value 定义里包含 "等于 observed" 的情况 —— observed 本身也算"或更极端"。

3. 决策：p-value 和 cutoff 比

条件结论常用措辞
p < 0.01 Reject the null（at both 0.01 and 0.05） "highly statistically significant"
0.01 ≤ p < 0.05 At 0.05: Reject ｜ At 0.01: Fail to reject "statistically significant" at 0.05 only
p ≥ 0.05 Fail to reject the null（at both） "not significant"
🔥 永远不要说 "accept the null"。Null hypothesis 可能看起来 plausible，但也可能有其他没考虑的解释。所以我们只能说 "fail to reject"。

考试出现 "accept the null" 选项时，一定是错的，可以直接排除。

4. p-value 的精确解读

✅ 正确说法：
  • "在 null 假设为真的前提下，看到 observed 或更极端的概率是 p"
  • "If null is true, the probability of seeing a difference as or more extreme as observed is about p"
❌ 错误说法（考试常以选项形式出现）：
  • ❌ "p-value 是 null 为真的概率"（hypothesis 不是 random variable，没有"为真的概率"）
  • ❌ "p-value 是 alt 为真的概率"
  • ❌ "p-value 是数据出错的概率"
  • ❌ "p-value 是 1 减去 null 为真的概率"
  • ❌ "If alt is true, probability of seeing observed is p"（条件颠倒了，应该是 if null is true）
  • ❌ "p-value 为 0 说明代码错了"（不一定 —— 极罕见 observed 完全可能导致 simulation 中从未达到，p ≈ 0）
  • ❌ "p-value 为 0.5 说明 simulation 错了"（也不一定 —— 0.5 就是中间值，说明 null 完全 plausible）

5. Left or right tail？

方向决定用 >= 还是 <=。回顾 Section 2 的决策树：

Alt 形式Test stat 通常是"更极端" 方向代码
A > B A − B（不带 abs） 右尾（大值偏 alt） sim >= obs
A < B A − B（不带 abs） 左尾（小值偏 alt） sim <= obs
different / not equal |A − B|（带 abs） 右尾（abs 后小值偏 null，大值偏 alt） sim >= obs
"coin 偏 tails"（heads 少） number of heads 左尾 sim <= obs

6. 完整例子：硬币 188 heads 公平吗？（Lec 20）

Found a coin, flipped 400 times, got 188 heads。问硬币公不公平。

  • Null: coin is fair (P(heads) = 0.5)
  • Alt: coin is not fair（双边）
  • Test stat: |number of heads − 200|（双边用 abs）
  • Observed: |188 − 200| = 12
results = np.array([])
for i in np.arange(10000):
    one = abs(np.random.multinomial(400, [0.5, 0.5])[0] - 200)
    results = np.append(results, one)

# p-value：abs 后大值偏 alt → 右尾
p_value = np.count_nonzero(results >= 12) / 10000
# ≈ 0.25 -ish

得到 p ≈ 0.25 → fail to reject the null（> 0.05）→ 硬币看起来是公平的。

⚠️ Lec 20 的 另一种 写法：用 number of heads（无 abs）作 test stat 是错的（因为双边，但 number of heads 单方向）—— Lec 19 已经讨论过。考试要识别这种"用错 test stat 类型"的写法。

7. 5% Cutoff 的"双重解读"

用 5% cutoff 的含义：

如果 null 是真的，我们用这个流程会错误地 reject 它约 5% 的次数。

也叫 significance level（α）。常见值：

  • α = 0.05：standard（5%）
  • α = 0.01：highly significant（1%）
  • α = 0.10 也偶有用到

💡 Cutoff 是历史习惯，并非数学必然。但 DSC 10 考试默认 0.05 (standard)；如果题目说 "highly significant"，就是 0.01。

8. Determining a cutoff：lowest 5%（Lec 20）

"Sufficiently rare" 的精确定义之一：observed statistic 落在 lowest 5%（左尾题）或 highest 5%（右尾题）。

用 percentile 求 cutoff 值：

# 左尾题（如硬币偏 tails，看 heads 少）
np.percentile(results, 5)
# 比如 184 → 如果 observed < 184，reject

# 右尾题（如双边 |heads-200|）
np.percentile(results, 95)
# 比如 24 → 如果 observed > 24，reject

9. p-value 决策表（背在桌前）

万能决策表：
  • p < 0.01 → reject at 0.01 ✓，reject at 0.05 ✓
  • 0.01 ≤ p < 0.05 → fail to reject at 0.01，reject at 0.05
  • p ≥ 0.05 → fail to reject at both

⚠️ 常见陷阱与考点

陷阱 1：⭐ "Accept the null"

永远不对！只能 "fail to reject"。考题里看到 "accept" 选项，直接排除。

陷阱 2：⭐ p-value 解读错误

p-value 不是："null 为真的概率"、"alt 为真的概率"、"data 出错的概率"。它是："给定 null 为真，看到 observed 或更极端的概率"。
Wi25 Q5 P1.2 整道题就是这个考点的多选陷阱（5 个选项里几乎全都是错的解读）。

陷阱 3：⭐ 左/右尾用错符号

Test stat 是 mean_A − mean_B，alt 是 "A 比 B 小"（左尾）→ p-value 必须用 sim <= obs。
Test stat 是 |...| 或 diff^2（恒非负），观测值偏 alt 总是"大" → 用 sim >= obs。

陷阱 4：用 > 而不是 >=

包括 observed 本身（"as or more extreme as"）。考试代码题里 > 经常是干扰选项。

陷阱 5：p-value 0 不一定代表 bug

如果 observed 极度极端（在 10000 次 simulation 中都没出现），就会得到 0.0。这恰好是 reject 最强的证据，不是代码错。
Wi25 Q5 P1.2 选项 "p-value = 0 说明 permutation test 实现错了" —— 错。

陷阱 6：p-value 0.05 算 reject 吗？

严格意义上 cutoff 是 strictly less than 0.05 才 reject。所以 p = 0.05 通常算 "fail to reject"。但课堂常用 "≤ 0.05 reject" 的说法 —— 不会考边界情形，所以这点不用太紧张。考试给的 p-value 通常远离 0.05（如 0.001、0.03、0.14、0.5）。

陷阱 7：连续 cutoff 测试

"是否在 0.01 和 0.05 显著" 是常见双选题。要会同时判断：p = 0.03 → at 0.05 reject，at 0.01 fail to reject。不要选 "both reject" 或 "both fail to reject"。

陷阱 8：reject ≠ "证明 alt 为真"

Reject 只是说 null 看起来不合理；不等于 alt 一定对。Deflategate 例子尤其强调：reject 不能推 causation。

📝 真题 & 练习

真题 1：Sp25 Q4 P1.2 ⭐⭐⭐（p=0.03 决策）

Spring 2025 Quiz 4, Problem 1.2 (Lec 20)

Null: 录取率 = 0.20；Alt: 录取率 > 0.20。Test stat 是 number of offers (out of 50)。p-value = 0.03。结论？

  1. Reject at both 0.01 and 0.05
  2. Fail to reject at 0.01, reject at 0.05
  3. Reject at 0.01, fail to reject at 0.05
  4. Fail to reject at both
显示答案

答案：B。0.01 < 0.03 < 0.05，所以 0.05 reject、0.01 fail to reject。

陷阱：选项 C 是 "0.01 reject, 0.05 fail to reject" —— 方向反了，永远不会出现（cutoff 越小越严格）。

真题 2：Sp24 Q5 P1.3 ⭐⭐⭐（p=0.04 多选）

Spring 2024 Quiz 5, Problem 1.3 (Lec 20)

Permutation test for road vs hybrid bikes，p = 0.04。多选下面正确的：

  1. Reject at 0.01
  2. Fail to reject at 0.01
  3. Reject at 0.05
  4. Fail to reject at 0.05
显示答案

答案：B 和 C。p = 0.04 在 0.01 和 0.05 之间。

真题 3：Fa24 Q4 P4.2 ⭐⭐（p=0.14）

Fall 2024 Quiz 4, Problem 4.2 (Lec 20)

p = 0.14，用 standard cutoff 0.05。哪个 hypothesis 更被 data 支持？

  1. Null
  2. Alternative
显示答案

答案：A. Null。p = 0.14 > 0.05 → fail to reject → data 与 null consistent → null 更被支持。
⚠️ 注意："更被支持" ≠ "证明 null 为真"。考试措辞如果是 "we accept the null" 就要选错。

真题 4：Wi25 Q5 P1.2 ⭐⭐⭐（p-value 解读多选 陷阱密集）

Winter 2025 Quiz 5, Problem 1.2 (Lec 20)

Permutation test，关于 p-value 解读，多选正确的：

  1. If p = 0.18, this means that if the alternative hypothesis is true, the probability of seeing a difference as or more extreme as observed is about 18%.
  2. If p = 0.03, this means that if the null hypothesis is true, the probability of seeing a difference as or more extreme as observed is about 3%.
  3. If p = 0, this means that we implemented our permutation test incorrectly.
  4. If p = 0.5, this means that exactly half of the simulated test statistics were less extreme than our observed difference.
  5. None of the above.
显示答案 & 解析

答案：B 和 D 都对（官方）。

  • A 错：条件应该是"if null"，不是"if alt"。这是最常见的错误措辞。
  • B 对：正确解读。"if null is true" + "or more extreme" 都满足。
  • C 错：p = 0 是合法的，意味着 simulation 中从未达到 observed 的极端程度（说明 observed 非常极端）。不是代码错。
  • D 对（⚠️ 易错）：p-value 定义为 count(simulated >= observed) / N。p = 0.5 意味着恰好一半 simulated stats >= observed（即"as or more extreme"），因此恰好另一半 simulated stats < observed（即 less extreme than observed）。D 的措辞字面成立。

陷阱：D 的字眼"exactly half"和"less extreme"乍看像是想坑你，但其实 p = 0.5 是数学上的精确分界点——刚好一半 ≥ observed、一半 < observed，所以"恰好一半 less extreme" 完全准确。

这是 Quiz 4/5 最高频陷阱题之一。考前一定要在脑海里默念正确的 p-value 解读。

真题 5：Wi25 Q4 P3.4 ⭐⭐（observed 0.04 信息不足）

Winter 2025 Quiz 4, Problem 3.4 (Lec 20)

已知 observed test statistic = 0.04（test stat 是 |0.8 − sample proportion|）。结论是什么？

  1. Reject the null
  2. Accept the null
  3. Fail to reject the null
  4. Not enough information
显示答案

答案：D. Not enough information。

  • 给的是 observed test statistic（0.04），不是 p-value。
  • 要决策，必须把 0.04 和 simulated distribution 比较，算出 p-value。光看 0.04 本身无法决定 reject 与否（它可能落在 dist 的中间，也可能落在尾部）。
  • B "Accept" 一定错，可以排除。

这道题专门考"observed statistic ≠ p-value"。别看到一个小数就当 p-value 用。

练习 6：写 p-value 代码

综合代码练习

对下面每个场景，写出正确的 p-value 代码（假设 sim 是长度 10000 的 simulated array，obs 是 observed scalar）：

  1. Alt: 硬币偏 tails，test stat = number of heads
  2. Alt: 红 candy 比绿多，test stat = n_red − n_green
  3. Alt: 硬币不公平，test stat = |heads − 200|
  4. Alt: Patriots 比 Colts pressure drop 大，test stat = mean_Patriots − mean_Colts
  5. Alt: Jeffrey 比 Kate 严，test stat = mean_Kate − mean_Jeffrey
显示答案
  1. np.count_nonzero(sim <= obs) / 10000（左尾：heads 少偏 alt）
  2. np.count_nonzero(sim >= obs) / 10000（右尾）
  3. np.count_nonzero(sim >= obs) / 10000（abs 后看大值，右尾）
  4. np.count_nonzero(sim >= obs) / 10000（右尾）
  5. np.count_nonzero(sim >= obs) / 10000（Kate − Jeffrey 大 → Jeffrey 严，右尾）

练习 7：解读 p-value 是非题

综合判断

下面每个 statement 判断 T/F：

  1. p-value 等于 null 为真的概率
  2. p-value 越小，越倾向 reject null
  3. 若 p = 0.001，可以说 null 一定是错的
  4. p-value = 0.05 时，结论是 "fail to reject"（按 standard cutoff）
  5. "Fail to reject" 等价于 "accept the null"
  6. p-value 计算用 > 而不是 >= 也对
显示答案
  1. F。p-value 是"在 null 为真的前提下，看到 observed 或更极端的概率"。Null 不是 random variable。
  2. T
  3. F。Reject 不等于"证明 null 错"，只是"data 与 null 不一致，需要解释"
  4. T（严格 cutoff：p < 0.05 才 reject）。但 DSC 10 实际不会考这个边界 case
  5. F。永远不要 "accept"
  6. F。必须用 >=（包括 observed 本身）
