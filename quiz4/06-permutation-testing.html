📚 知识点

1. 为什么需要 permutation test

Standard hypothesis test（Lec 19-20）的设定：知道 population 分布，问 "这个 sample 是不是从这个 population 抽的"。

但很多实际问题不是这样。例子：

  • Smoking 母亲的婴儿 vs non-smoking 母亲的婴儿，出生重是否相同？
    → 我们有两个 sample，但不知道任何一个的 population 分布
  • Patriots 和 Colts 的 ball pressure drop 是否相同？
    → 同样：两个 sample，无 population
  • UCSD 旧版 vs 新版 TritonLink 网站，停留时间是否相同？

这种"两个 sample，无 population"的场景叫 two-sample problem 或 A/B testing。Permutation test 就是处理这类问题的标准工具。

2. Permutation test 的核心 idea

Null hypothesis: 两个 sample 来自同一个 population distribution
如果 null 真，那么 group label（A 还是 B）不带任何信息，
所以我们可以把 label 打乱重新分配，结果应该差不多。

具体做法：把两组合并成一个 pool，然后随机分回两组（保持原来的两组大小），重复成千上万次，得到 test statistic 在 null 下的分布。

3. 标准 hypothesis pair（permutation test 模板）

  • Null: The two samples come from the same population distribution; the observed difference is due to random chance.
  • Alt (双边): The two samples come from different distributions.
  • Alt (单边): One group has higher / lower mean (or median, etc.).

4. 完整代码模板（背！）

# 数据：DataFrame df，一列 'group' (两个 label，如 True/False)，一列 'value' (数值)
# 目标：测 group 是否影响 value（permutation test）

# Step 1: observed statistic
group_means = df.groupby('group').mean().get('value')
observed_diff = group_means.loc['A'] - group_means.loc['B']

# Step 2: 写计算 simulated stat 的函数
def difference_in_group_means(shuffled_df):
    means = shuffled_df.groupby('shuffled').mean().get('value')
    return means.loc['A'] - means.loc['B']

# Step 3: shuffle 循环
n_repetitions = 5000  # 或 10000
differences = np.array([])

for i in np.arange(n_repetitions):
    shuffled_labels = np.random.permutation(df.get('group'))  # ← 关键！
    shuffled = df.assign(shuffled=shuffled_labels)
    diff = difference_in_group_means(shuffled)
    differences = np.append(differences, diff)

# Step 4: p-value（根据 alt 方向）
# 假设 alt: A's mean > B's mean → 右尾
p_value = np.count_nonzero(differences >= observed_diff) / n_repetitions

5. 三个 关键代码陷阱

5.1 用 np.random.permutation，不是 df.sample

# ✗ 错误：sample 会改变行顺序，把 value 和 group 的对应破坏了
df.sample(df.shape[0])

# ✓ 正确：shuffle 一列，保持原 index 不动
shuffled = np.random.permutation(df.get('group'))
df.assign(shuffled=shuffled)

Lec 22 用 5 行 DataFrame 演示这个区别：

data:
  x  y
0 a  1
1 b  2
2 c  3
3 d  4
4 e  5

# data.sample(5) 打乱整行 → (x, y) 配对不变（错）
# np.random.permutation(data.get('x')) → 只打乱 x，y 不变（对）

5.2 没 resample with replacement

# ✗ 这是 bootstrap，不是 permutation
df.sample(df.shape[0], replace=True)

# ✓ Permutation 是不放回的重排
np.random.permutation(df.get('group'))

5.3 shuffle 完后用 assign 重新挂列

# 直接对 shuffled 用 groupby 是错的；必须先 assign 回 DataFrame
shuffled_labels = np.random.permutation(df.get('group'))
shuffled_df = df.assign(shuffled=shuffled_labels)  # ← 重新挂列
# 然后才能 groupby('shuffled')

这是 Quiz 4/5 的高频概念题。题目：

def one_stat():
    shuffled = np.random.permutation(___(a)___)
    shuffled_df = tritonlink.assign(shuffled = shuffled)
    grouped = shuffled_df.groupby(___(b)___).mean().get("minutes")
    return ___(c)___

问 "(a) 该填 'version' 还是 'minutes'"？

正确答案："We must shuffle 'version' because of the structure of the provided code"。
解释：(a) 处填什么取决于其余代码。这里：
  • (b) 必须按"组别"groupby，所以新挂的 'shuffled' 列是组别
  • 所以 (a) shuffle 的必须是组别 'version'，让新挂的 shuffled 是版本标签
概念上：理论上 shuffle 哪一列都行（因为 permutation test 只关心"label 和 value 的配对是否随机"，打乱 label 或 value 等价）。但给定的代码结构限制了答案。
⚠️ 干扰选项："只有 numerical column 能 shuffle" / "只有 group label 能 shuffle" —— 错。两个都能，但具体代码结构决定。

7. 经典例 1：Smoking & Birth Weight（Lec 21-22）

  • 1174 个 babies，2 列：'Maternal Smoker'（True/False）、'Birth Weight'
  • Null: smokers' babies 和 non-smokers' babies 出生重相同（同一分布）
  • Alt: smokers' babies 平均出生重更轻
  • Test stat: non-smokers' mean − smokers' mean（差值；不带 abs）
  • Observed: 9.27 ounces
def difference_in_group_means(weights_df):
    group_means = weights_df.groupby('Shuffled_Labels').mean().get('Birth Weight')
    return group_means.loc[False] - group_means.loc[True]

# observed
observed_diff = babies.groupby('Maternal Smoker').mean().get('Birth Weight')
observed_diff = observed_diff.loc[False] - observed_diff.loc[True]  # 9.27

# permutation 循环
differences = np.array([])
for i in np.arange(500):
    shuffled = np.random.permutation(babies.get('Maternal Smoker'))
    shuffled_df = babies.assign(Shuffled_Labels=shuffled)
    differences = np.append(differences,
                             difference_in_group_means(shuffled_df))

# alt: smokers' 更轻 → non-smokers − smokers 应该大 → 右尾
p_value = np.count_nonzero(differences >= observed_diff) / 500
# ≈ 0 → reject

结论：reject null。但不能说 "smoking causes lower birth weight" —— 可能有其他因素（如 smoker 更可能 drink caffeine）。

8. 经典例 2：Deflategate（Lec 22）

  • Patriots 10 个球，Colts 4 个球，每个球记 pressure drop
  • Null: 两队的 pressure drop 来自同一分布
  • Alt: Patriots 的 pressure drop 更大
  • Test stat: mean_Patriots − mean_Colts
  • Observed: 0.74 psi
def difference_in_mean_pressure_drops(pressures_df):
    team_means = pressures_df.groupby('Team').mean().get('Shuffled_Drops')
    return team_means.loc['Patriots'] - team_means.loc['Colts']

# 注意这里 shuffle 的是 PressureDrop（数值列）而不是 Team —— 同样合法
n_repetitions = 5000
differences = np.array([])
for i in np.arange(n_repetitions):
    shuffled_drops = np.random.permutation(footballs.get('PressureDrop'))
    shuffled = footballs.assign(Shuffled_Drops=shuffled_drops)
    differences = np.append(differences,
                             difference_in_mean_pressure_drops(shuffled))

# alt 单边右尾
p_value = np.count_nonzero(differences >= 0.74) / 5000
# ≈ 0.0048 → highly significant，reject null

注意点：

  • Lec 22 此例 shuffle 的是数值列 PressureDrop（不是 Team label）—— 同样合法。这印证 "shuffle 哪个都行" 的概念。
  • Reject 之后不能推 Patriots 故意 deflate（causation）。NFL 后续的 Wells Report 用物理学 + 其他证据综合判断才有 causation 结论。

9. Test statistic 选择（permutation 专属强化）

AltTest statisticp-value 方向
"A and B come from different distributions" |mean_A − mean_B| 或 (mean_A − mean_B)² 右尾 >=
"A's mean > B's mean" mean_A − mean_B（不带 abs） 右尾 >=
"A's mean < B's mean" mean_A − mean_B（不带 abs） 左尾 <=
"A and B 中位数不同" (Sp25 Q5 P1) |median_A − median_B| 右尾
🔥 Permutation test 不能用 TVD。TVD 是 categorical 分布距离；permutation test 通常处理 numerical 数据。Lec 21 summary 明确："TVD is not used for permutation tests"。

10. A/B testing（Lec 21-22 注释）

A/B test ≈ Hypothesis test 用 permutation 方法测两个 sample

Lec 21 原话："Permutation tests are one way to perform A/B tests. For us, they mean the same thing."

📌 考试看到 "A/B test" 就当 permutation test 来做即可。

11. Permutation test 结论 ≠ Causation（必背）

  • Reject null → two samples 看起来不来自同一分布
  • 但没证明谁导致了谁（confounders 可能存在）
  • Smoking & birth weight：reject 后只能说"相关"，不能说"smoking causes"
  • Deflategate：reject 后只能说"压力降幅不同"，不能直接说"Patriots intentionally deflated"

这是 Quiz 4 的哲学性陷阱，选项里如果含 "causes" 通常错。

⚠️ 常见陷阱与考点

陷阱 1：⭐ Permutation vs Bootstrap 混淆

PermutationBootstrap

陷阱 2：⭐ 用 TVD 做 permutation test

题目里 smoking & birth weight、bike prices、pressure drops 都是数值变量，不能用 TVD。
Wi26 Q4 P2.2 选项 "The total variation distance between two graders' score distributions" —— 错（score 是数值，不是 categorical）。

陷阱 3：⭐ Test stat 方向反

Wi26 Q4 P2.2 重磅陷阱：alt 是 "Jeffrey 比 Kate 严"（Jeffrey 的 mean 小）。
✓ Valid：Kate's mean − Jeffrey's mean（这样大 → Jeffrey 严 → 偏 alt，看右尾）
✓ Valid：Jeffrey's mean − Kate's mean（小/负 → Jeffrey 严 → 偏 alt，看左尾）
✗ 不 valid：|...|（abs 后无法区分谁严）
✗ 不 valid：TVD（数值）

陷阱 4：⭐ Permutation 不能从中推 causation

"Reject null → smoking causes lower birth weight" —— 错。Reject 只说明相关，不说 causation。

陷阱 5：⭐ shuffle 完忘记 assign

np.random.permutation(...) 返回 array，要 df.assign(...) 才能用 groupby。

陷阱 6：⭐ 用 df.sample 当 permutation

df.sample(df.shape[0]) 整体打乱所有列的行，但同一行内各列仍配对 —— 不破坏 group-value 配对，达不到 permutation 效果。必须 np.random.permutation 单独 shuffle 一列。

陷阱 7：⭐ 把 "两个 sample 比较" 用 simulate under null（Lec 19 风格）

Wi26 Q4 P1 干扰选项 "Under the null, simulate by resampling with replacement from the combined 50 scores" —— 错。这是 bootstrap 风格，不是 permutation。Permutation 必须 shuffle，without replacement。

陷阱 8：⭐ 把已知 "categorical proportion match" 当 permutation

"测试 bikes 类型分布是否符合 [15%, 20%, 40%, 20%, 5%]" —— 这是 standard hypothesis test with TVD（已知 population），不是 permutation test。

陷阱 9：observed_diff 方向和 simulated 不一致

定义 diff = mean_A − mean_B，observed_diff 也得用同样方向。如果 observed 是 mean_B − mean_A，比较时方向就反了。

陷阱 10：groupby 后 .loc[True/False] 大小写

Group label 是 boolean 时用 .loc[True] 和 .loc[False]（Python 关键字，不带引号）。是字符串时用 .loc['Patriots'] 等（带引号）。混了会 KeyError。

练习 8：Permutation vs Bootstrap vs Standard Test 判别

综合判别

下面每个场景，选合适的方法（permutation / bootstrap CI / standard test with TVD / standard test with custom stat / sample size formula）：

  1. 估计 UCSD 学生 sleep 时间的 95% CI
  2. 测 UCSD 男生女生平均身高是否相同
  3. 测一枚硬币是否公平
  4. 测 bike 类型是否符合预期 [15%, 20%, 40%, 20%, 5%]
  5. 预先计算 polling 的 sample size
  6. 测 Patriots 和 Colts 的 pressure drop 是否一样
  7. 用已有 sample 检测 mean 是否 = 98.6
显示答案
  1. Bootstrap CI（或 CLT CI for mean）
  2. Permutation test（两 sample 比较）
  3. Standard hypothesis test（已知 null distribution P=0.5）
  4. Standard test with TVD（categorical 比较）
  5. Sample size formula n ≥ (4·SD/w)²
  6. Permutation test（两 sample）
  7. CI as hypothesis test（CI 法）或 standard test with custom stat（差值绝对值）
