📚 知识点

1. Hypothesis Test 是什么

从 Lec 18 的"Swain jury 故事"形式化而来：

A hypothesis test chooses between two views of how data were generated, based on data in a sample.

注意两件事：

  1. 它在两个 hypothesis 之间选一个，不保证选中的就是真理
  2. 它的结论只基于 sample，所以再小心也只能给可能性

2. Null Hypothesis vs Alternative Hypothesis

Null (H0)Alternative (H1)
定义 一个 well-defined probability model about how data was generated（可以 simulate） 另一种关于 data 怎么来的 view，通常和 null 互斥
语气 "no effect" / "no difference" / "随机抽的" / "param = some value" "there is a difference" / "biased" / "treated 更大" / "param ≠ value"
能否 simulate 必须能 simulate（这就是它叫 null hypothesis 的原因） 不需要能 simulate
结论 reject 或 fail to reject（永远不"accept"） 无直接结论，只能说"data supports alt"或"not"
🔥 高频考点：题目让你"选哪个 null / alt 配对正确"。常见错误：
  • 把 "there is a difference" 当 null（错！这不是 well-defined probability model，没法 simulate）
  • Null 和 alt 互换
  • Alt 包含 causation 描述（"causes taller flowers"）—— 通常 alt 应避免 causation 措辞
口诀：null 是 "no difference / same distribution / equal to X"；alt 是 null 的反面。

例子：硬币是否公平

  • Null：The coin is fair (P(heads) = 0.5)。← well-defined，可以用 np.random.multinomial(400, [0.5, 0.5]) 模拟
  • Alt（双边）：The coin is not fair
  • Alt（单边）：The coin is biased towards tails (P(heads) < 0.5)

例子：Swain v. Alabama

  • Null：Jury panel was selected at random from a population that is 26% Black
  • Alt：Jury panel was not selected at random

例子：Fertilizer & flower height（Wi25 Q4 P2）

  • Null：There is no difference in the average height of treated and untreated flowers
  • Alt：There is a difference in the average height of treated and untreated flowers

注意 Wi25 Q4 P2 的错误选项：

  • ❌ "Null: There is a difference; Alt: no difference"（互换）
  • ❌ "Null: Fertilizer use has no impact; Alt: Fertilizer use causes taller flowers"（causation 措辞 + 单边偏题意）

3. Test Statistic：核心工具

3.1 定义

A test statistic is a single number, computed from a sample, used to decide which hypothesis the data supports.

区分两个相关概念：

  • Observed statistic：test statistic 算在真实 observed data 上的值（只有一个）
  • Simulated test statistic：under null 假设的 simulated sample 上算出的（成千上万个，组成 empirical distribution）

3.2 选 test statistic 的两大原则

  1. 能区分两个 hypothesis：算出来的值要能区分 null 真和 alt 真两种情况
  2. 方向一致：理想情况下，large 值偏一种 hypothesis，small 值偏另一种 —— 不要让"high & low 都偏 alt，中间偏 null"

3.3 ⭐ 决定要不要 abs：取决于 alternative 方向

这是整个 Quiz 4 最高频的陷阱，连续多年考。

Alternative 形式测的方向Test statistic极端值含义
"different / not equal" 双边 带 abs：|A − B| 或 |X − 期望| 大 = 偏 alt（不管 A>B 还是 A<B 都不对）
"A > B"（A 更大） 单边（右尾） 不带 abs：A − B 大 = 偏 alt；小 = 偏 null
"A < B"（A 更小） 单边（左尾） 不带 abs：A − B，or B − A A−B 小 = 偏 alt；或 B−A 大 = 偏 alt
口诀：
"different" → abs(差)，看大值
"> 或 <" → 不带 abs，看一端

3.4 经验：用 absolute value 把"两边都极端"变成"一边极端"

Lec 19 的硬币例子：

  • Null: coin is fair。Alt（双边）: coin is not fair。
  • 如果用 number of heads 作 test stat：太大（接近 400）和太小（接近 0）都 偏 alt，中间偏 null —— 这是双向极端，不好处理
  • 解决：用 |number of heads − 200|。这样 大值统一偏 alt，小值统一偏 null

这个技巧叫 "absolute value trick"。考试代码里经常出现：

def num_heads_from_200(arr):
    return abs(arr[0] - 200)

# observed
num_heads_from_200([188, 212])  # |188 - 200| = 12

3.5 ⭐ 单边 alt 用了 abs 会发生什么？

考试常考 "could we use |A − B| as a test stat for alt: A > B?" —— 答案：不能，因为 |A − B| 大也可能是 A 比 B 小很多，无法区分方向。

4. Simulate Under the Null：标准代码骨架

无论什么场景，simulate under null 的代码总是这个结构：

results = np.array([])
for i in np.arange(N):       # N 通常 = 10000
    # ① 在 null 假设下生成一个 sample
    new_sample = ...         # multinomial / choice / permutation 之一

    # ② 在这个 sample 上算 test statistic
    stat = ...

    # ③ 存起来
    results = np.append(results, stat)

# 现在 results 是 empirical distribution of the test statistic under null

4.1 例：硬币是否公平

# Null: P(heads) = 0.5
# Test stat: |number of heads − 200|

results = np.array([])
for i in np.arange(10000):
    one_sample_heads = np.random.multinomial(400, [0.5, 0.5])[0]
    stat = abs(one_sample_heads - 200)
    results = np.append(results, stat)

# observed
observed = abs(188 - 200)  # = 12

4.2 例：jury 是否随机（Swain）

# Null: panel drawn at random from 26% Black population
# Test stat: number of Black men in panel

counts = np.array([])
for i in np.arange(10000):
    new_count = np.random.multinomial(100, [0.26, 0.74])[0]
    counts = np.append(counts, new_count)

# observed
observed = 8

4.3 例：用 np.random.choice 替代 multinomial（Wi25 Q4 P3.1）

Wi25 给的代码用 np.random.choice，是 multinomial 的等价写法之一：

# Wi25 Q4 P3.1 - fertilizer flower fields
# Null: each field independently 80% fertilized

def one_stat():
    sample_size = flowers.shape[0]
    random_choice = np.random.choice([True, False],
                                      sample_size, p=[0.8, 0.2])
    return abs(0.8 - np.count_nonzero(random_choice) / sample_size)

# Wi25 Q4 P3.2 - 等价的 multinomial 写法
def one_stat_differently():
    multi = np.random.multinomial(flowers.shape[0], [0.8, 0.2])
    return abs(0.8 - multi[0] / flowers.shape[0])

📌 关键观察：np.random.choice([True, False], n, p=[0.8, 0.2]) 抽 n 个 boolean；np.random.multinomial(n, [0.8, 0.2]) 直接给两个 counts。两种写法都正确，看题目代码框架。

5. Draw a Conclusion：把 observed 和 empirical distribution 比

  • 如果 observed 落在 simulated distribution 的常见区域 → null 看起来合理 → fail to reject null
  • 如果 observed 落在罕见区域（尾部）→ null 看起来不合理 → reject null

"罕见"具体多少算罕见？要靠 p-value 量化 —— 那是下一节的内容。本节先建立"看 observed 在 empirical distribution 哪个位置"的直观。

⚠️ 常见陷阱与考点

陷阱 1：Null / alt 互换

Null 必须是"无效应 / 等于某值 / 随机的"那一边（可 simulate 的那一边）。"There is a difference" 不能当 null。

陷阱 2：Alt 写了 causation

"Fertilizer use causes taller flowers" 这种 causation 措辞在 hypothesis test 里通常错。我们只能说"有 difference"，不能说"causes"。

陷阱 3：⭐ Test stat 方向搞错

必背决策树：
  • alt 说 "different" → test stat 必须带 abs；不带 abs 不行，因为正负两端都偏 alt
  • alt 说 "A 比 B 大" → test stat 必须是 A − B 或 -(B − A)；带 abs 不行，因为大值可能来自 A 小很多
  • alt 说 "A 比 B 小" → 用 A − B 看左尾，或 B − A 看右尾；不带 abs

陷阱 4：把 observed statistic 当 simulated

Observed 只有一个，是从真实数据算出来的。Simulated 是从 null 模型生成 N 次得到的 array。这两个不要混。

陷阱 5：Null hypothesis 必须可以 simulate

"Coins are fair" 可以模拟（P=0.5）；"Coins are unfair" 不行（具体多 unfair？0.51？0.03？无法模拟）。所以 "unfair" 只能当 alt。

陷阱 6：高低 both 偏 alt 的 test stat

对双边 alt，number of heads 就有这个问题：太多 和 太少都偏 alt。所以必须加 abs。考试经常给一堆候选 test stat，让你选哪些"valid" —— 注意单边题用 abs 不 valid，双边题不用 abs 也不 valid。

陷阱 7：忘了 [0] 索引

np.random.multinomial(100, [0.26, 0.74]) 返回 array，必须加 [0] 才能得到第一类的 count。这是高频代码扣分点。

陷阱 8：append 没赋值

np.append(arr, x) 不修改 arr，必须写 arr = np.append(arr, x)。

📝 真题 & 练习

真题 1：Fa24 Q4 P2 ⭐⭐⭐（test stat 方向）

Fall 2024 Quiz 4, Problem 2 (Lec 19)

Null: 评分 > 9 的 Comedy 比例 等于 评分 > 9 的 Action 比例。

Alt: 评分 > 9 的 Comedy 比例 > 评分 > 9 的 Action 比例。

设 C9 = Comedy 比例，A9 = Action 比例。下列哪些是 valid test statistic？多选。

  1. C9 − 0.5
  2. A9 − 0.5
  3. A9 · C9
  4. A9 + C9
  5. C9 − A9
  6. A9 − C9
  7. |A9 − C9|
  8. |C9 − A9|
显示答案 & 解析

答案：E（C9 − A9）和 F（A9 − C9）

  • Alt 是 C9 > A9（单边）→ test stat 不该带 abs
  • E. C9 − A9：valid。大值（C 比 A 大很多）→ 偏 alt；小值（甚至负）→ 偏 null。看右尾
  • F. A9 − C9：valid。方向反过来了 —— 小值（甚至负）偏 alt，看左尾。同样能区分
  • A、B：和单独某个比例的偏移有关，不能直接区分两组
  • C、D：A·C 或 A+C 与 alt 不相关
  • G、H：abs 形式 —— alt 是单边，abs 之后无法区分 A>C 还是 A<C，不 valid

⚠️ 关键陷阱：用 abs 对应"different"双边 alt；这里是单边 alt，不能用 abs。

真题 2：Wi25 Q4 P2.1 ⭐⭐（选 null/alt 配对）

Winter 2025 Quiz 4, Problem 2.1 (Lec 19)

研究员想知道 fertilizer 是否影响花的高度（treated vs untreated）。选正确的 null/alt 配对：

  1. Null: There is a difference in average height. Alt: no difference.
  2. Null: no difference. Alt: there is a difference.
  3. Null: no difference. Alt: Treated flowers grow taller, on average.
  4. Null: Fertilizer has no impact. Alt: Fertilizer causes taller flowers.
显示答案 & 解析

答案：B

  • A 错：null 和 alt 互换。Null 必须是 "no difference"（可 simulate）
  • B 对：standard 形式，alt 双边
  • C 不是最佳：题目说"determine whether there is a relationship"（含义是双边）；选 C 把 alt 缩成单边，偏离题意
  • D 错：用了 "causes"，不能从 hypothesis test 推 causation

练习 4：识别 test statistic 的方向（综合）

综合练习

对下面每个 (null, alt) 配对，写出合适的 test statistic 形式 + 看哪一尾。

  1. Null: 硬币公平。Alt: 硬币不公平。
  2. Null: 硬币公平。Alt: 硬币偏 tails。
  3. Null: 红 vs 绿 candy 比例相同。Alt: 红 candy 更多。
  4. Null: Patriots 和 Colts 的 pressure drop 一样。Alt: Patriots 的 pressure drop 更大。
  5. Null: Jeffrey 和 Kate 给的平均分相同。Alt: Jeffrey 给的平均分更低。
显示答案
  1. 双边 → |number_of_heads − 200|，右尾（大值偏 alt）
  2. 单边（偏 tails 即 heads 少）→ number_of_heads，左尾（小值偏 alt）
    或 number_of_tails，右尾
  3. 单边（红 > 绿）→ n_red − n_green，右尾
  4. 单边 → mean_drop_Patriots − mean_drop_Colts，右尾
  5. 单边（Jeffrey 更低）→ mean_Jeffrey − mean_Kate，左尾；或 mean_Kate − mean_Jeffrey，右尾

📌 注意 (5) 是 Wi26 Q4 P2.2 的考点 —— 必须知道方向，否则会选错差的写法。

练习 5：写 null 还是 alt？

综合练习

下面每个 statement，判断它适合做 null 还是 alt（或都不适合）：

  1. "There is no difference between A and B"
  2. "A's mean is higher than B's mean"
  3. "The data is drawn at random from a population that is 26% Black"
  4. "There is some unfairness"
  5. "Treatment causes recovery"
  6. "X follows distribution [0.3, 0.4, 0.3]"
显示答案
  • A: null（可 simulate "same distribution"）
  • B: alt（无法直接 simulate "higher"，不知道高多少）
  • C: null（well-defined model）
  • D: 都不适合。当 null 不行（不能 simulate）；当 alt 太模糊。改成"the coin is unfair"才能当 alt
  • E: 都不适合，"causes" 不该出现在 hypothesis 里
  • F: null（well-defined）
