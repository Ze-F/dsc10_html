📚 知识点

1. Recap：95% CLT-based CI & 宽度公式

来自 Quiz 3 模块（Lec 17）的核心公式 —— 这是样本量推导的起点：

95% CI for population mean
= [ sample_mean − 2·SD/√n ,  sample_mean + 2·SD/√n ]

所以这条 CI 的宽度（右端 − 左端）是：

width = 4 · SD / √n

📌 注意是 4·SD/√n（来自 2·SD/√n 的左右两个 ± 加起来），不是 2·SD/√n。考试时常把这俩混了。

2. 反推 sample size：从 width 解 n

问题："我想要 95% CI 宽度不超过 w，最少要采集多少 sample？"

从 width = 4·SD/√n ≤ w 解不等式：

      4·SD/√n  ≤  w
  ⇔  √n  ≥  4·SD/w
  ⇔  n   ≥  (4·SD/w)²
n ≥ (4 · SD / w)²

实际取满足该不等式的最小整数（向上取整）。

🧮 4× 关系（Wi26 Q4 P4 高频考点）

因为公式里 w 出现在分母平方里，所以：

把 CI 宽度变成原来的一半（精度提高 2 倍）
→ sample size 变成原来的 4 倍（不是 2 倍！）

一般地：宽度变成原来的 1/k，sample size 变成原来的 k² 倍。

直观理解：CLT 告诉我们 SE = SD/√n，所以"精度"和 √n 成正比 —— 想精度翻倍，得 n 翻 4 倍。

3. 当 sample SD 未知怎么办：找 SD 的上界

问题：还没采集 sample，怎么知道 sample SD？

答：找一个upper bound，用它代入。这样能保证 CI 宽度不会超过目标 w。

3.1 一般情况

如果对数据本身一无所知，无法找 SD 上界 —— 这种情况公式用不了，得先做 pilot study 估一个 SD。

3.2 ⭐ 关键特例：0/1 数据（proportions are means）

当数据只有 0 和 1（例如 polling：1 = 用 Duolingo，0 = 不用）时：

  • proportion of 1s = mean of the sample（把 0/1 平均一下就是 1 占的比例）
  • 对 0/1 collection，SD 公式简化为：
SD of collection of 0s and 1s = √(prop_of_0s · prop_of_1s) = √(p · (1−p))

这个表达式在 p = 0.5 时取最大值：

max SD of 0/1 collection = √(0.5 · 0.5) = 0.5

所以对 polling proportion，我们总能用 SD = 0.5 作为兜底：

n ≥ (4 · 0.5 / w)² = (2 / w)²

这是 polling 类问题的速算公式。背下来：n ≥ (2/w)²。

3.3 三组速算（背在小本本上）

目标 CI 宽度 wn ≥ (2/w)²最小 n（向上取整）
0.10(20)² = 400400
0.06(33.33)² ≈ 1111.111112（Lec 18 Duolingo 例子）
0.04(50)² = 25002500
0.03(66.67)² ≈ 4444.444445
0.02(100)² = 1000010000

4. np.random.multinomial —— Quiz 4 的核心武器

⚠️ 这个函数会在后续每一节出现：jury panel 模拟、hypothesis testing simulation 都靠它。务必掌握。

4.1 签名 & 行为

np.random.multinomial(sample_size, pop_distribution)
  • sample_size：要抽几个样本
  • pop_distribution：每个 category 的概率（数组），必须近似和为 1
  • 返回：长度等于 category 数的 counts array（不是 proportion！）
  • with replacement（每次抽都独立，不放回也叫"放回采样"）

4.2 例子：万圣节糖果（Lec 18 原例）

35 户人家，每家糖盒：30% Starbursts、30% Sour Patch Kids、40% Twix。每家抓一颗。

np.random.multinomial(35, [0.3, 0.3, 0.4])
# 可能输出：array([ 8, 14, 13])
#                  ↑   ↑   ↑
#         Starbursts  SPK  Twix

每次运行结果都不同（随机），但加起来一定 = 35。

4.3 例子：Swain jury panel

人口 26% Black，74% non-Black，panel 100 人：

demographics = [0.26, 0.74]
np.random.multinomial(100, demographics)
# 可能输出：array([28, 72])
# 第 0 个 = Black 人数；第 1 个 = non-Black 人数

# 只要 Black 人数：
np.random.multinomial(100, demographics)[0]
# 28

4.4 模拟 N 次 jury panel：标准模板

counts = np.array([])
for i in np.arange(10000):
    new_count = np.random.multinomial(100, demographics)[0]
    counts = np.append(counts, new_count)

# 现在 counts 是 10000 次模拟下的 Black 人数 array
counts.min()  # 比如 11 → 说明历史最少都有 11 人，8 极罕见

📌 这就是 hypothesis testing 的骨架代码。下一节我们把"Black 人数"换成"test statistic"，就完全是一样的循环。

4.5 想要 proportion 而非 count：除以 sample_size

# Counts
np.random.multinomial(1453, eligible)
# array([222, 244, 165, 798, 24])

# Proportions（除以 sample size）
np.random.multinomial(1453, eligible) / 1453
# array([0.153, 0.168, 0.114, 0.549, 0.017])

5. Statistical Models —— Quiz 4 的概念引子

从这里开始，Lec 18 进入 Quiz 4 的核心思维。

5.1 Statistical inference 的两类问题

类型例子对应技术
Parameter estimation（参数估计） "美国成年人平均身高是多少？" bootstrap CI / CLT CI（Quiz 3 范围）
Yes/no questions（是非题） "这个 jury panel 是从这个 population 随机抽的吗？"
"这枚硬币公平吗？"		 Hypothesis testing（Quiz 4 范围！）

5.2 "Model" 的定义

A model is a set of assumptions about how data was generated.

例子：

  • "This jury panel was randomly drawn from a population that is 26% Black."
  • "This coin lands heads with probability 0.5 each flip, independently."
  • "Section C 的成绩是从全班分数分布中随机抽 108 个得到的。"

注意 model 必须是可以拿来模拟的（well-defined probability model）—— 否则没法跑 simulation。这一点在下一节会变成 null hypothesis 的硬性要求。

5.3 评估 model 的标准套路（Lec 18 → Lec 19 桥梁）

  1. 假设 model 为真
  2. 在这个假设下，反复 simulate 生成 statistic 的 empirical distribution
  3. 看 observed statistic 落在哪 —— 如果非常罕见，说明 model 可能不对

💡 记住这个流程：Lec 19 起的 hypothesis testing 就是把它形式化 —— "model"变成"null hypothesis"，"罕见程度"用"p-value"量化。

6. 完整跑一遍：Swain v. Alabama jury panel

这是 Lec 18 的高潮，也是 Lec 19 的开场。完整背下来。

背景

  • 1965 年 Talladega County, Alabama
  • 合格陪审员人口：men 21+，其中 26% 是 Black
  • Robert Swain 的 jury panel 100 人，只有 8 人是 Black
  • 问题：jury panel 真的是 random 抽的吗？

三要素

ModelJury panel 是从 26% Black 的人口中随机抽 100 人
Observation实际 panel 100 人里只有 8 个 Black
Statisticpanel 里 Black 人数
QuestionModel 能解释这个 observation 吗？

三步模拟

# Step 1：一次实验生成一个 statistic
demographics = [0.26, 0.74]
np.random.multinomial(100, demographics)[0]  # → 比如 26

# Step 2：重复 10000 次
counts = np.array([])
for i in np.arange(10000):
    new_count = np.random.multinomial(100, demographics)[0]
    counts = np.append(counts, new_count)

# Step 3：看分布、和 observed = 8 比较
counts.min()  # 比如 11 → 历史最少 11，远大于 8

结论

  • 在 10,000 次随机模拟里，从未出现 ≤ 8 个 Black（最小值 11）
  • 所以 "random selection" 这个 model 极难解释 observation
  • 结论：jury panel 很可能不是 random，存在其他因素（种族偏见）

⚠️ 注意：我们没有证明 bias 存在，只是说 random model 解释不了。可能还有其他模型（虽然这里直觉上 bias 是唯一合理解释）。这种谨慎的措辞会反复在 Quiz 4 出现 —— 永远不要从 reject 推 causation。

7. 三个 np.random 函数的对比表（高频考点）

函数用途典型 Quiz 4 场景放回？
np.random.choice(arr, n) 从 array 里抽 n 个元素 已知所有可能取值的 list（如 ["H","T"]、所有学生） 默认放回；replace=False 不放回
np.random.multinomial(n, probs) 按概率分布抽 n 个，返回 counts 已知 population 的类别比例，不知道总人数（jury, 民意调查） 始终放回
np.random.permutation(arr) 把 array 重新洗牌 Permutation test（Lec 22）shuffle group labels —（不放回的"重排"）
🔥 高频考点：考试给你一个场景，问该用哪个？口诀：
  • "知道有多少人，每个人具体取值" → choice
  • "只知道比例，要按比例抽 n 个" → multinomial
  • "已有两个 sample 要打散" → permutation

⚠️ 常见陷阱与考点

陷阱 1：w 是总宽度，不是 margin

  • "95% CI 宽度不超过 0.06" → w = 0.06，不是 0.03
  • 公式 n ≥ (4·SD/w)² 里的 w 就是右端 − 左端
  • 但有时题目说 "margin of error ≤ 0.03"，那时 w = 2 × 0.03 = 0.06

陷阱 2：sample size 一定向上取整

如果算出 1111.11，答案是 1112，不是 1111 也不是 1111.11。因为 1111 不满足不等式。

陷阱 3：multinomial 返回 counts，不是 proportion

# ✗ 错误：以为返回比例
sample_dist = np.random.multinomial(100, [0.26, 0.74])  # array([28, 72])

# ✓ 想要 proportion 要除以 sample size
sample_dist = np.random.multinomial(100, [0.26, 0.74]) / 100  # array([0.28, 0.72])

陷阱 4：multinomial 第二个参数必须近似和为 1

# ✗ 错误：概率和 ≠ 1
np.random.multinomial(100, [0.26, 0.50])  # 概率和 = 0.76，结果不可靠

# ✓ 正确
np.random.multinomial(100, [0.26, 0.74])  # 概率和 = 1.00

陷阱 5：用错 np.random 系列函数

  • "模拟 400 次抛硬币" → np.random.multinomial(400, [0.5, 0.5])[0]（heads 的次数）或 np.random.choice(['H','T'], 400) 都行；后者得手动 count
  • "从 1453 个 Alameda County 居民里按比例抽" → multinomial（按比例）
  • "shuffle smoker labels" → permutation

陷阱 6：SD = 0.5 只对 0/1 数据

不要对任意 polling 都用 0.5。题目里如果出现连续值（如身高、温度），SD 就不是 0.5 —— 通常会给你 sample SD 或要你假设一个值。

陷阱 7：multinomial 是"with replacement"

真实 jury 抽签理论上是 without replacement（同一人不会被抽两次）。但 Lec 18 指出：当 sample 规模相对 population 很小时，with vs without replacement 几乎没差别，所以放心用 multinomial。考试不要为这个纠结。

陷阱 8：Model ≠ "事实"

Model 是一个 假设。结论"reject model"不等于"证明了反面"。这个谨慎措辞在 Quiz 4 的 hypothesis testing 题里反复考。

📝 真题 & 练习

真题 1：Wi26 Q4 Problem 4 ⭐⭐⭐（必做）

Winter 2026 Quiz 4, Problem 4 (Lec 18)

Detective Peter 想估计 Jeffrey 评分中 score 低于 70 的比例。他构造 95% CLT-based confidence interval。

(a) 他希望 CI 宽度不超过 0.08，最少需要多少条 Jeffrey 的 submission？

(b) 假如改要宽度不超过 0.04，最少 sample size 是 (a) 的几倍？

显示答案 & 解析

(a)

  • 这是 polling proportion 问题（0/1：score < 70 或不 < 70）→ 用 SD 上界 = 0.5
  • 公式：n ≥ (4 · 0.5 / 0.08)² = (2/0.08)² = 25² = 625
  • 答案：625

(b)

  • w 从 0.08 → 0.04，除以 2 → n 变成原来 4 倍
  • 验算：n ≥ (2/0.04)² = 50² = 2500 = 4 × 625 ✓
  • 答案：4 倍

💡 这道题就是为本节高频陷阱 1+2 设计的：(1) 总宽度不是 margin；(2) w 减半 → n × 4。两点都得脱口而出。

练习 2：Multinomial 输出预测

综合练习

下面的代码各返回什么 shape / 含义？写出你的预测。

(A) np.random.multinomial(50, [0.4, 0.3, 0.2, 0.1])
(B) np.random.multinomial(50, [0.4, 0.3, 0.2, 0.1])[0]
(C) np.random.multinomial(50, [0.4, 0.3, 0.2, 0.1]) / 50
(D) np.random.multinomial(50, [0.4, 0.3, 0.2, 0.1]).sum()
(E) np.random.multinomial(1, [0.5, 0.5])
显示答案
  • (A)：长度 4 的 array，每个元素是该 category 的 count；和必等于 50。例如 array([19, 17, 9, 5])
  • (B)：第一个 category 的 count（一个 int / numpy 整数）。例如 19
  • (C)：长度 4 的 array，每个元素是 proportion；和必等于 1。例如 array([0.38, 0.34, 0.18, 0.10])
  • (D)：恒等于 50（counts 总和 = sample size）
  • (E)：长度 2 的 array，每个 0 或 1，和等于 1。例如 array([0, 1])。这是"抽一次"的模拟，相当于抛一枚 biased coin

练习 3：选对 np.random 函数

综合练习

下列每个场景，选 np.random.choice / np.random.multinomial / np.random.permutation 之一：

  1. "DataFrame students 有 393 行，要随机抽 108 个作为 Section C 的模拟"
  2. "已知 Alameda County 人口中 Asian 15%、Black 18%、Latino 12%、White 54%、Other 1%，要按比例抽 1453 人作为一个模拟 panel"
  3. "smoking 数据 1174 行，要 shuffle Maternal Smoker 列以做 permutation test"
  4. "模拟抛公平硬币 400 次，得到 heads 数量"
  5. "已知一个游戏每局赢的概率 0.3，模拟 100 局看赢几局"
显示答案
  1. df.sample(108, replace=False)（DataFrame 直接 sample）。用 np.random.choice 也可以，但题目说"DataFrame"通常用 .sample
  2. np.random.multinomial(1453, [0.15, 0.18, 0.12, 0.54, 0.01])。已知比例而非具体名单 → multinomial

  4. np.random.multinomial(400, [0.5, 0.5])[0]。或 np.random.choice(['H','T'], 400) 然后数 H 的个数
  5. np.random.multinomial(100, [0.3, 0.7])[0]。本质是 binomial，但课程里用 multinomial

练习 4：完整跑一遍 jury 模拟

基于 Lec 18 原例 / Lec 19 起点

Robert Swain 案：合格人口 26% Black，panel 100 人，observed 8 Black。完整写出三步模拟代码，并描述结论。

显示答案 
# Setup
demographics = [0.26, 0.74]
observed = 8

# Step 1: one simulation
np.random.multinomial(100, demographics)[0]  # → e.g. 28

# Step 2: 10,000 simulations
counts = np.array([])
for i in np.arange(10000):
    new_count = np.random.multinomial(100, demographics)[0]
    counts = np.append(counts, new_count)

# Step 3: compare
counts.min()  # ≈ 11（远大于 8）
np.count_nonzero(counts <= observed) / 10000  # ≈ 0.0
# → p-value ≈ 0：极罕见

结论：模拟下从未出现 ≤ 8 Black（最少 ≈ 11），所以 "random panel" 这个 model 解释不了 observation。Quiz 4 的措辞应当是 "reject" 而不是 "证明 bias"。

练习 5：综合判断（Quiz 4 风格）

综合判断 · Tier 2 整合题

下面哪些陈述正确？（多选）

  1. 对 polling proportion，无论数据怎样，把 sample SD 取为 0.5 总能保证 CI 宽度满足要求
  2. 如果一开始计算得 n = 624.99，可以四舍五入到 625
  3. 把目标 CI 宽度从 0.10 缩到 0.05，sample size 也变为原来一半
  4. np.random.multinomial(100, [0.4, 0.6]) 返回长度为 2 的 array，且两个元素之和恒等于 100
  5. 一个 "model" 必须能用代码模拟才能用于做 hypothesis testing
显示答案
  • A ✓：0.5 是 √(p(1−p)) 在 [0,1] 上的最大值，用它一定够。
  • B ✓：624.99 向上取整就是 625（等价于"不小于 624.99 的最小整数"）。注意如果是 625.01 就得取 626。
  • C ✗：宽度减半 → n 变 4 倍，不是 1/2 倍。
  • D ✓：multinomial 返回 counts，和 = sample size。
  • E ✓：这就是下一节 null hypothesis 的核心要求 —— "well-defined probability model"。
